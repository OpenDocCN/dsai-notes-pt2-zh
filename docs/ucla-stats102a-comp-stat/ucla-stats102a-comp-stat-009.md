## 矩阵与数组的子集选取

![](img/d7c466410c075298cefd223d42aad9ba_6.png)

上一节我们介绍了一维结构（原子向量和列表）的子集选取。本节中，我们来看看二维结构，首先从矩阵或数组开始。

![](img/d7c466410c075298cefd223d42aad9ba_8.png)

最常见的矩阵子集选取方法是使用多个向量，每个维度对应一个向量，用逗号分隔。如果某个维度留空，则会返回该维度的所有元素。

例如，我们有一个3x3的矩阵 `A`：
```r
A <- matrix(1:9, nrow=3, ncol=3, dimnames = list(NULL, c("A", "B", "C")))
#      A B C
# [1,] 1 4 7
# [2,] 2 5 8
# [3,] 3 6 9
```

![](img/d7c466410c075298cefd223d42aad9ba_10.png)

以下是几种选取方式：
*   `A[1:2, ]`：选取第1、2行，所有列。
*   `A[c(TRUE, FALSE, TRUE), c("B", "A")]`：使用逻辑和字符向量组合选取第1、3行的B列和A列。
*   `A[0, -2]`：选取0行（即无行），并排除第2列，结果是一个仅有列名A和C的空结构。

![](img/d7c466410c075298cefd223d42aad9ba_12.png)

![](img/d7c466410c075298cefd223d42aad9ba_14.png)

默认情况下，对矩阵进行子集选取会进行“简化”。例如，`A[1, ]` 返回的是一个命名向量，而不是一个1行的矩阵。

![](img/d7c466410c075298cefd223d42aad9ba_16.png)

![](img/d7c466410c075298cefd223d42aad9ba_17.png)

由于矩阵本质上是带有维度属性的原子向量，因此也可以使用单个向量进行选取。此时，R会将矩阵视为一个长向量，按列优先的顺序进行索引。
```r
vowels <- matrix(letters[1:25], nrow=5)
vowels[8:9] # 返回第8和第9个元素，即 "h" 和 "i"
```

![](img/d7c466410c075298cefd223d42aad9ba_18.png)

你还可以用一个坐标矩阵来选取子集。矩阵的每一行代表你想要选取的元素的坐标。
```r
# 创建一个坐标矩阵
coord <- matrix(c(1,5, 3,1, 2,3, 1,1), ncol=2, byrow=TRUE)
vowels[coord] # 返回 (1,5), (3,1), (2,3), (1,1) 位置上的元素
```

![](img/d7c466410c075298cefd223d42aad9ba_20.png)

![](img/d7c466410c075298cefd223d42aad9ba_22.png)

对于更高维度的数组，原理相同。你需要提供一个列数与数组维度数相同的坐标矩阵。
```r
arr <- array(1:12, dim=c(2,3,2))
coord_arr <- matrix(c(1,2,1, 2,3,2), ncol=3, byrow=TRUE)
arr[coord_arr] # 返回指定坐标的元素
```

## 数据框的子集选取

数据框在内部以列表形式存储，但同时也具有二维结构。因此，你可以像子集列表一样选取数据框，也可以像子集矩阵一样选取。

让我们创建一个简单的数据框 `df`：
```r
df <- data.frame(x=1:4, y=4:1, z=letters[1:4])
```

像矩阵一样选取（使用逗号）：
```r
df[df$y %% 2 == 0, ] # 选取y列为偶数的所有行，所有列
```

![](img/d7c466410c075298cefd223d42aad9ba_26.png)

像列表一样选取（不使用逗号）：
```r
df[c("x", "z")] # 选取名为"x"和"z"的列（返回数据框）
```
这与 `df[, c("x", "z")]` 的结果类似。

![](img/d7c466410c075298cefd223d42aad9ba_28.png)

**关键区别在于逗号的使用**：
*   `df[1:3, ]`：选取第1到3行，所有列。
*   `df[1:3]`：选取第1到3列。

![](img/d7c466410c075298cefd223d42aad9ba_30.png)

当只选取一列时，不同方法的结果形式不同：
*   `df["x"]` 或 `df[1]`：返回一个单列的数据框（**保留**结构）。
*   `df[, "x"]` 或 `df[, 1]`：返回一个原子向量（**简化**结构）。
*   `df$x` 或 `df[["x"]]`：同样返回一个原子向量（**简化**结构）。

## 简化与保留

![](img/d7c466410c075298cefd223d42aad9ba_36.png)

在子集选取时，理解结果是“简化”了原始结构还是“保留”了原始结构非常重要。

![](img/d7c466410c075298cefd223d42aad9ba_38.png)

以下是控制简化行为的总结：
*   **原子向量**：`[` 保留名称；`[[` 丢弃名称。
*   **列表**：`[` 总是返回列表；`[[` 提取列表中的元素。
*   **因子**：`[` 默认保留未使用的水平；可以使用 `drop=TRUE` 参数来丢弃未使用的水平。
*   **矩阵**：默认会简化（例如，单行/单列变为向量）。使用 `drop=FALSE` 参数可以保留矩阵结构。
    ```r
    a <- matrix(1:4, nrow=2)
    a[1, ]          # 简化成向量
    a[1, , drop=FALSE] # 保留为 1x2 的矩阵
    ```
*   **数据框**：`[` 选取单列时返回数据框（保留）；`[[` 或 `$` 选取单列时返回向量（简化）。

## 常见错误与示例

![](img/d7c466410c075298cefd223d42aad9ba_46.png)

![](img/d7c466410c075298cefd223d42aad9ba_48.png)

![](img/d7c466410c075298cefd223d42aad9ba_50.png)

让我们通过 `mtcars` 数据集来看一些常见的子集选取错误。

![](img/d7c466410c075298cefd223d42aad9ba_52.png)

**错误1：遗漏逗号**
```r
mtcars[mtcars$cyl <= 5] # 错误：未定义列被选中
```
原因：没有逗号，逻辑向量 `mtcars$cyl <= 5` 被解释为要选取的**列索引**，而不是行索引。由于数据框没有那么多列，所以报错。
**正确做法**：
```r
mtcars[mtcars$cyl <= 5, ] # 正确：选取气缸数<=5的所有行
```

![](img/d7c466410c075298cefd223d42aad9ba_54.png)

![](img/d7c466410c075298cefd223d42aad9ba_55.png)

**错误2：逻辑运算错误**
```r
mtcars[mtcars$cyl == 4 | 6, ] # 结果不正确，可能选中了所有行
```
原因：`| 6` 中的数字6被强制转换为逻辑值 `TRUE`，因此整个条件对所有行都成立。
**正确做法**：
```r
mtcars[mtcars$cyl == 4 | mtcars$cyl == 6, ]
```

![](img/d7c466410c075298cefd223d42aad9ba_56.png)

![](img/d7c466410c075298cefd223d42aad9ba_57.png)

**错误3：选取行时遗漏逗号**
```r
mtcars[1:13] # 错误：尝试选取第13列，但可能不存在
```
**正确做法**：
```r
mtcars[1:13, ] # 正确：选取前13行
```
