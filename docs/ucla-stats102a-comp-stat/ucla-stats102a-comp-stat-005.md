## 因子（Factors）

![](img/2a356d4435118b8358a019b4a87d762c_5.png)

![](img/2a356d4435118b8358a019b4a87d762c_6.png)

上一节我们介绍了向量和列表。本节中，我们来看看因子。因子用于表示分类变量。从内部看，因子本质上是一个**整数向量**，但它带有属性。最重要的属性是**水平**，它定义了因子可以取哪些类别值，并且其类被标记为“因子”。

![](img/2a356d4435118b8358a019b4a87d762c_8.png)

![](img/2a356d4435118b8358a019b4a87d762c_10.png)

![](img/2a356d4435118b8358a019b4a87d762c_12.png)

![](img/2a356d4435118b8358a019b4a87d762c_14.png)

例如，我们有一个字符向量 `gender`：
```r
gender <- c("M", "F", "F", "X", "M", "F")
```
使用 `factor()` 函数可以将其转换为因子：
```r
gender_factor <- factor(gender)
```
打印 `gender_factor` 会显示值 `M F F X M F` 以及水平 `F M X`。注意，打印因子时默认没有引号。

以下是关于因子的几个关键点：

![](img/2a356d4435118b8358a019b4a87d762c_16.png)

![](img/2a356d4435118b8358a019b4a87d762c_18.png)

*   **内部存储机制**：因子内部存储的是整数索引，而非原始字符。例如，水平 `F M X` 分别对应整数 `1 2 3`。原始向量 `c("M", "F", "F", "X", "M", "F")` 在内部被存储为 `c(2, 1, 1, 3, 2, 1)`。可以使用 `as.integer(gender_factor)` 查看。
*   **水平的顺序**：默认情况下，R会提取字符向量中的所有**唯一值**，并按**字母顺序**将它们设置为水平。因此，`F` 排在 `M` 之前，`M` 排在 `X` 之前。
*   **处理数值型因子时的陷阱**：如果将数值向量转换为因子，直接进行数值运算可能会得到意外结果。因为R将其视为分类变量，而非数值。
    ```r
    x <- c(0, 1, 10, 5)
    x_factor <- factor(x)
    mean(x_factor) # 返回 NA
    ```
    如果对因子直接使用 `as.numeric()`，返回的是其内部整数索引（1, 2, 3...），而非原始数值。要计算原始数值的均值，需要先转换为字符，再转换为数值：
    ```r
    mean(as.numeric(as.character(x_factor))) # 返回正确结果 4
    ```
*   **赋值限制**：尝试为因子分配一个不在其水平列表中的值，会导致产生 `NA`。
    ```r
    gender_factor[1] <- "Mail" # 会生成 NA
    ```

## 矩阵（Matrices）

![](img/2a356d4435118b8358a019b4a87d762c_27.png)

现在，让我们从一维结构转向二维结构。R中的矩阵本质上是一个带有**维度属性**的原子向量。维度属性是一个包含行数和列数的向量。

![](img/2a356d4435118b8358a019b4a87d762c_29.png)

![](img/2a356d4435118b8358a019b4a87d762c_30.png)

![](img/2a356d4435118b8358a019b4a87d762c_32.png)

![](img/2a356d4435118b8358a019b4a87d762c_34.png)

创建矩阵的一种方法是先创建向量，然后为其设置维度属性：
```r
m <- 1:10 # 创建一个整数向量
attr(m, "dim") <- c(2, 5) # 设置维度属性为2行5列
```
现在，`m` 变成了一个2x5的矩阵，数据按**列优先**的顺序填充。此时，`class(m)` 会返回 `“matrix”`。

![](img/2a356d4435118b8358a019b4a87d762c_36.png)

要移除矩阵结构，只需将维度属性设为 `NULL`：
```r
attr(m, “dim”) <- NULL
```
这样，`m` 就又变回了一个一维的原子向量。

## 数组（Arrays）与数据框（DataFrames）

![](img/2a356d4435118b8358a019b4a87d762c_45.png)

![](img/2a356d4435118b8358a019b4a87d762c_46.png)

![](img/2a356d4435118b8358a019b4a87d762c_47.png)

矩阵是二维数组。数组是更一般的概念，其维度属性的长度可以大于2，从而形成三维或更高维的数据结构。
```r
a <- 1:12
attr(a, “dim”) <- c(2, 3, 2) # 创建一个2行、3列、2层的三维数组
```
数据填充顺序是：先填满第一层的行，再填列，然后移动到下一层。

![](img/2a356d4435118b8358a019b4a87d762c_49.png)

最后，我们介绍数据框。数据框是R中最常用的数据结构之一，用于存储表格数据。与矩阵不同，数据框的**每一列可以包含不同类型的数据**（如数值、字符、逻辑值）。

![](img/2a356d4435118b8358a019b4a87d762c_51.png)

从内部看，数据框被存储为一个**列表**，列表中的每个元素都是一个**等长的向量**，对应数据框的一列。

![](img/2a356d4435118b8358a019b4a87d762c_53.png)

R自带了一个示例数据集 `trees`。我们可以检查它的结构：
```r
class(trees) # 返回 “data.frame”
typeof(trees) # 返回 “list”，表明其内部是列表
str(trees) # 显示数据结构：一个包含3列（Girth, Height, Volume）、每列31个数值向量的数据框
```
数据框的属性包括列名、类（`data.frame`）和行名（默认为1到n的整数序列）。

![](img/2a356d4435118b8358a019b4a87d762c_55.png)

![](img/2a356d4435118b8358a019b4a87d762c_57.png)

![](img/2a356d4435118b8358a019b4a87d762c_59.png)

本节课中我们一起学习了R语言中三种重要的数据结构：用于处理分类数据的**因子**，用于存储同类型二维数据的**矩阵**，以及用于存储表格型异质数据的**数据框**。理解这些结构的内部原理和特性，是高效进行数据操作和统计分析的关键。
