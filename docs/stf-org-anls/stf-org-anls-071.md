#  071：论坛改进方案

在本节中，我们将探讨如何改进课程论坛的组织方式，以更好地保存和传递历年积累的知识与讨论。我们将回顾学生提出的建议，并讨论可行的实施方案。

上一节我们介绍了组织学习的重要性，本节中我们来看看如何将论坛中的宝贵对话转化为可持续的组织记忆。

## 课程背景与挑战

课程计划在未来多次重开，可能多达十次，目标是让尽可能多的人受益。首次开设时，斯坦福大学提供了教学学分和少量资金支持，用于准备材料和录制。如今课程依靠自有预算和时间运行。团队需要找到自动化管理的方法，因为团队成员还有本职工作及其他项目。目标是以低成本持续免费提供课程，避免投入过多持续管理精力，因为这非常耗费人力。

论坛中存在大量宝贵资源，去年的讨论同样精彩。团队一直在思考如何将其转化为可用的组织记忆。这在各种邮件列表和论坛中很常见。例如，Stack Overflow 是一个流行的编程问答论坛，用户提问、解答、投票排序。但本课程的许多问题不同，它们值得反复回顾并从不同角度深化。同时，团队也希望快速访问更具洞察力的评论和主题，因为师生时间都很有限。如果能找到方法让所有课程都具备这些特质，将非常有益。

## 学生建议汇总

以下是学生们为保存当前输入并访问往年讨论所提出的主要解决方案。

*   **维基（Wiki）**：多位学生如Milroy和Kathy建议使用维基。实际上，页面底部已有一个维基链接，去年就已存在并鼓励使用，但使用者寥寥。或许本届学员可以更多利用并推广它。
*   **总结报告**：学生如Sharif建议，可以让学员在课程结束时总结自己的经验，形成类似期刊或文献的资料保存下来，作为课程要求的一部分。今年可能不会实施，但未来课程可考虑。这可以作为未来课程组织者了解学员普遍经验的途径。
*   **主题摘要**：学生如Susanna Hon Ruia提出，为每个讨论主题添加标签并发布摘要会非常有用。教学助理（CTA）或许可以帮助将这些摘要置顶。学员可能没有此权限，但教学助理可以通过代码实现。这能引导兴趣，并帮助后续高效回顾整个主题，而无需阅读全部几十甚至上百条回复。

以上都是很好的想法。

## 已实施与计划中的改进措施

团队已对此做出回应，并为一些错过截止日期的学员提供补交机会以获取额外学分。这是一种快速、低成本的补救和激励方式。如果教学助理有时间，可以标记并置顶这些总结，这将成为有用的资源。它成本低，且无需对现有系统做大改动，短期内实施是合理的。

其他合理的改进包括：由于“点赞”投票功能并不总能凸显有趣的主题，可以考虑将部分责任更多分配给教学助理。教学助理功能对团队和极具耐心与奉献精神的教学助理们来说都是新事物，双方都在探索如何最佳利用，并使他们的体验更有回报。这可以是一个好方法，例如让教学助理标注、置顶优秀的总结，或高亮他们认为有洞察力的帖子以引导关注。此外，这也能提醒教师哪些主题值得制作“屏边聊天”视频。

请注意，教师现在也开始复用往年的“屏边聊天”视频作为回应。因为去年的帖子和教师的回答与今年必然不同，其中蕴含的知识和见解能以高效的方式使当前学员受益。团队希望逐步融入所有这些过去的聊天内容。

## 未来可能性展望

有人提到了在线直播环节。许多课程确实为部分学员提供直播，但参与机会有限。教师对此有些顾虑，但希望未来能更开放地实现。小组聊天对参与其中的学员很有用，团队也在熟悉相关技术，希望未来能更好地实施，或许还包括小组项目。

在未来版本的课程中，可能会引入斯坦福本校的做法：让学生进行更多小组项目，综合多种理论并将其应用于自选的案例。或者，设计更多设计学院式的练习，让学员以小组形式识别组织中的实际问题，设计解决方案并实施，最后在论坛中汇报。这些成果都可以作为课程记忆保存下来。

每次尝试新事物时，都可能出现技术故障，需要排查。微小的改动也可能在系统中产生连锁反应。团队很幸运有优秀的员工协助解决，但这确实增加了额外的工作量。目前，至少对于这门慕课，相关资源正在减少。因此，必须思考自动化方案，以便在兴趣减退前持续提供课程，或让其他版本、其他资源以更精良的方式呈现。

团队将持续改进，倾听反馈，并渴望采纳许多建议。虽然无法立即实施所有想法，但会进行尝试。正如鸟巢中叫声最大的雏鸟得到最多的虫子，如果学员们能以友好愉快的方式组织、推动并提出建议，团队很可能会朝那个方向努力。

非常感谢大家的意见，也感谢Loie发起这个主题。希望上述讨论已部分回应了这个问题。

本节课中，我们一起探讨了改进课程论坛以保存和传递知识的多种方案，包括维基、总结报告、主题摘要等具体建议，以及已实施和未来可能的改进措施。关键在于在资源有限的情况下，找到低成本、可持续的自动化或半自动化解决方案，以丰富所有学员的学习体验。
