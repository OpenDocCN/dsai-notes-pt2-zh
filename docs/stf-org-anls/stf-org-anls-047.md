#  047：从失败中学习 📚

在本节中，我们将探讨组织如何从失败中学习，以及如何平衡对成功与失败的分析，以促进持续改进和创新。

上一节我们讨论了组织学习中的一些潜在陷阱。本节中，我们来看看一个核心问题：组织是否应该以及如何从失败中汲取经验。

---

一位名叫斯蒂芬·道威的学员提出了一个很好的问题。他写道：“麦克法兰教授提到了过度关注失败的危险，但从失败中不是可以学到很多东西吗？为什么航空如此安全？一个原因是它通过程序和文化，将‘从失败中学习’制度化了。我在潜艇部队的经验也类似，我们发展了一种文化：错误不被掩盖，而是被公开以供他人学习。我们尽可能避免指责文化。未遂事件报告提供了在他人的错误演变为失败之前，就从中学习的方法。或许我们从失败中学到的比从成功中学到的更多。”

许多学员赞同这个观点。这确实是一个值得我们深入思考的重要话题，它反映了我们在课程中遇到的许多核心问题。

以下是学员们的一些主要观点：

*   **支持从失败中学习**：许多学员，如安吉·格林哈、朱莉娅·乌利亚诺、马丁、乌拉尼亚和约瑟菲，都同意斯蒂芬的观点，认为从失败中学习具有重要价值，甚至可能比从成功中学到更多。
*   **警惕过度关注失败**：也有学员，如萨达德什·基克，提出可能存在“矫枉过正”的风险。如果我们总是聚焦于失败并不断分析，可能会形成一种批判或问题导向的文化，这本身也可能带来问题。
*   **主张平衡分析**：因此，像法比奥·艾瓦这样的学员提出，我们应该同时分析失败与成功。这引出了“欣赏式探询”的观点，即我们不应只盯着失败，也应分析那些进展顺利的事情。
*   **强调迭代发展**：艾迪生·普莱斯则指出，批判与迭代发展相结合是积极的学习方式。通过持续的批判和迭代开发来调整我们的目标，这也是反思失败的一种积极途径。

诚然，我们需要避免灾难性的重大失败。但课程阅读材料的重点并非否定这种学习过程。事实上，这种学习是被鼓励的，对实践的反思应成为组织的核心。你的技术核心，即你试图做好的关键任务（将输入转化为输出），需要你不断反思如何改进这些实践。找到改进的杠杆，才能实现学习曲线，提升生产效率和公司绩效。

然而，就阅读材料而言，我想表达的真实意图是：在大多数公司里，人们常常陷入某些**学习陷阱**。这些陷阱主要有两种：

1.  **过早放弃陷阱**：当我犯错或某件事失败时（例如尝试新课程、新的同伴评估方式或评分标准），如果结果不完美，许多公司的倾向是彻底放弃这个产品或方案，转而去做别的事，而不去反思失败的原因。这是一种学习陷阱，因为你可能放弃了一个原本通过迭代改进就能变得非常出色、成为“全局最优解”的方案。
2.  **路径依赖陷阱**：反之，在“欣赏式探询”中，如果总是只关注积极面，也可能导致自满的叙事。但更常见的是，阅读材料中我想论证的是：**当你某件事做成功了，即使它可能不是最优的活动或程序，因为这一次成功了，你就会重复它。再次重复时，通常会做得稍好一点，形成一条学习曲线。但你不知道，如果尝试另一种方法，其学习曲线是否能在最终高度上远超当前方法。** 这就是“积极学习陷阱”的问题：你从不尝试新事物，陷入一种循环，自以为做得很好，但实际上可能存在更好的方法。

我们常常倾向于对问题做出反应。而组织学习的目标，是希望推动你的公司不仅仅思考问题和成功，更要反思你的核心技术、核心实践，并将其作为一个**实践社群**来讨论。这个社群协作并持续尝试改进他们的工作。

而**实践网络**则更进一步，延伸到更广阔的世界，从其他大学、公司或国家汲取截然不同的想法，带来你可能在当地社群中无法发现的创造性方法，并将其适应当地环境。实践网络和实践社群是相辅相成的。

另一个需要考虑的要点是，相关理论实际上提出了**探索与利用的平衡**。这引出了以下问题：你何时应该坚持不懈地让某个方案生效？何时应该探索新的解决方案、反思并持续调整或创新？何时应该专注于高效、有效地执行一件事并将其做到极致？这种探索与利用的平衡，与组织内关于实践的成功、失败及反思过程密切相关。

理论通常认为，你需要一种平衡和适度。反思过程很重要，但有时你必须采取行动（这属于利用模式）。反思则更多关乎探索和考虑改进的替代方案。

理想的情况是，围绕你的核心技术（核心的输入到输出任务），你持续思考改进的方法，在搜索与利用模式之间保持平衡。你既考虑失败之处并从中学习，也快速推进成功并反思其有益的原因。

因此，**对实践进行持续反思，是组织学习的关键**。你们敏锐地捕捉到了这一点。我希望我没有贬低从失败中学习的重要性，因为它显然是一个重要的学习途径。感谢斯蒂芬提出这个问题。

---

本节课中，我们一起学习了组织从失败中学习的价值与潜在陷阱。我们探讨了平衡分析成功与失败的重要性，理解了“实践社群”与“实践网络”在促进学习中的作用，并最终认识到，**持续反思核心实践，并在探索新方法与高效利用现有方法之间保持动态平衡，是组织实现有效学习和持续改进的关键**。
