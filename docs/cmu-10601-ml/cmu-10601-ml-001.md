# 001：课程介绍与核心概念

在本节课中，我们将要学习机器学习的基本定义、核心要素以及一些实际应用案例。我们将从一个清晰的定义出发，了解构成一个机器学习问题所必需的三个部分，并通过常见的例子来加深理解。

## 概述

机器学习研究如何构建能够通过经验自动改进其表现的计算机程序。要定义一个明确的机器学习问题，需要指定三个关键要素：任务、性能度量和经验类型。

## 核心要素

一个定义明确的机器学习问题包含以下三个部分：

1.  **任务**：计算机需要执行的具体功能。
2.  **性能度量**：用于衡量任务执行好坏的标准。
3.  **经验**：程序用于学习和改进的数据或信息。

## 实例解析：垃圾邮件过滤器

上一节我们介绍了机器学习的核心要素，本节中我们来看看一个具体的例子：垃圾邮件过滤器。

*   **任务**：接收任意电子邮件作为输入，并输出一个标签：“垃圾邮件”或“非垃圾邮件”。
*   **性能度量**：程序预测的标签与人工标注的“黄金标准”标签的匹配程度。例如，我们可以追求最小化分类错误的数量。性能度量也可以根据需求调整，例如，对“将正常邮件误判为垃圾邮件”这一错误施加更大的惩罚。
*   **经验**：程序用于学习的数据集。最常见的是**有标签的示例**，即一组已被人工标记为“垃圾邮件”或“非垃圾邮件”的电子邮件。此外，一些算法也能利用**无标签的示例**（即未标记的邮件）来提升学习效果。

## 更多应用示例

理解了垃圾邮件过滤器的例子后，我们可以将其框架应用到其他领域。

以下是几个不同类型的机器学习问题实例：

*   **自动驾驶车辆**：
    *   **任务**：根据摄像头和其他传感器的输入数据，输出建议的转向和刹车控制指令。
    *   **性能度量**：例如，成功准时抵达目的地的次数比例。
    *   **经验**：可以通过人类驾驶员驾驶车辆，让程序同时观察传感器数据和驾驶员的操作指令，从而获得大量有标签的训练数据。

*   **医疗数据挖掘**：
    机器学习常用于从历史数据库中发现模式。例如，在一个新生儿医疗记录数据库中，程序可以学习母亲怀孕期间的症状与分娩方式（自然分娩或剖腹产）之间的关联规则。这属于对**结构化数据**（由具有值的字段描述的数据）的挖掘。

## 总结

本节课中我们一起学习了机器学习的基本概念。我们明确了机器学习的目标是构建能通过经验自动改进的计算机程序。一个定义明确的问题必须包含**任务**、**性能度量**和**经验**这三个要素。我们通过垃圾邮件过滤、自动驾驶和医疗数据分析等实例，具体了解了这些要素如何在不同场景中体现。
