# 002：决策树与函数逼近

在本节课中，我们将完成对决策树学习的讨论，并以此作为案例研究，引出关于函数逼近的一些更普遍的见解。

## 概述

上一节我们介绍了函数逼近的基本框架。在函数逼近中，我们的目标是学习一个未知的目标函数 **F**，它从输入空间 **X** 映射到输出空间 **Y**。例如，**X** 可以是描述天气的属性（如湿度、温度），**Y** 可以是朋友当天是否打网球（是/否）。学习算法接收一组训练样本（即 **X-Y** 对），并输出一个能最佳逼近目标函数的假设（即一个分类器）。

在决策树学习的语境下，我们可以具体化这个框架。决策树处理由离散值特征组成的输入向量，并预测离散的输出值。每个假设就是一个决策树。决策树的一个优点是，它们足以表示任何定义在离散值特征向量上的离散值函数。

## 决策树学习算法回顾

上一节我们讨论了一个自顶向下生长决策树的算法。以下是该算法的核心步骤：

1.  **选择根节点属性**：算法从选择在决策树根节点进行测试的属性开始。它使用基于熵的互信息统计量来进行选择，旨在挑选出能提供最多信息的属性。
2.  **划分数据**：假设我们选择了“湿度”作为根节点。算法将根据湿度值（如高、低）将训练样本划分到不同的分支（或“桶”）中。
3.  **判断终止或递归**：如果某个分支下的所有样本都具有相同的标签（全是“是”或全是“否”），则停止在该分支生长，并将其标记为叶节点。如果分支内样本标签混杂（既有“是”也有“否”），则在该分支上递归地应用相同策略，选择新的属性进行进一步划分。
4.  **终止条件**：算法持续生长树，直到每个叶节点都只包含纯正例或纯负例，即完美分类训练数据。

你可以将这个算法视为在**所有可能决策树组成的假设空间**中进行搜索。每一步，它都通过添加一个节点来扩展当前的树，并使用互信息度量来决定下一步扩展哪个分支能带来最大收益。

## 函数逼近的通用视角

这个“在假设空间中搜索”的视角，对于我们理解本课程中将讨论的许多监督学习算法都非常有用。事实上，许多函数逼近算法都可以被概括为：**在可能的假设空间中搜索最佳假设**。

为了更清晰地理解这一点，让我们引入一个有助于思考机器学习领域的通用图示。

### 假设空间与实例空间

想象有两个维恩图：

*   **假设空间 (H)**：这个空间包含了所有可能的假设。在我们的例子中，就是所有可能的决策树函数。空间中的每一个点都代表一个具体的决策树（一个具体的假设）。
*   **实例空间 (X)**：这个空间包含了函数所有可能的输入实例。每一个点代表一个具体的输入向量（例如，`湿度=高, 温度=低, ...`）。

这两个空间通过假设函数联系起来。**每一个具体的假设（决策树）都会为实例空间中的每一个点分配一个标签**（例如，“打网球”或“不打网球”）。因此，每个假设都对应着实例空间的一个划分：一些实例被标记为正例，另一些被标记为负例。

### 思考题

为了加深理解，请思考以下问题：假设我们有 **20 个布尔特征**（即每个特征值为真或假）。

*   实例空间 **X** 有多大？（可能有多少种不同的输入？）
*   假设空间 **H**（所有可能的布尔函数决策树）又有多大？
