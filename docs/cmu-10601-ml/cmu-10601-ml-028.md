# 028：决策树与假设空间

在本节课中，我们将继续学习函数逼近这一核心机器学习任务，并深入探讨决策树算法。我们将从假设空间的角度重新审视决策树学习过程，理解学习算法本质上是一种在假设空间中的搜索。

## 假设空间：学习算法的核心图景

上一节我们介绍了决策树及其贪心生长算法。本节中，我们来看看如何从更宏观的“假设空间”视角来理解学习过程。这个视角是理解所有机器学习算法的根本。

我们可以将学习过程想象为在一个巨大的“假设空间”中进行搜索。这个空间包含了学习者可能考虑的所有假设（即函数）。

*   **假设空间（H）**：这是一个包含所有可能假设的集合。在决策树的例子中，每一个可能的决策树都是这个空间中的一个点。这个空间由学习者使用的**表示方法**（例如，决策树这种形式）所定义。
*   **实例空间（X）**：这是所有可能输入（例如，不同的天气特征组合）的集合。
*   **映射关系**：每一个假设 `h ∈ H` 都是一个从实例空间 `X` 到输出 `Y` 的函数。对于布尔分类任务，`Y = {正例, 负例}` 或 `{0, 1}`。给定一个假设（如一个决策树），我们可以将其应用于 `X` 中的每一个点，从而得到该点的标签。

因此，对于空间中的每一个假设（决策树），它都会将实例空间 `X` 划分为两个区域：一个被标记为正例的区域，其余部分被标记为负例。

## 学习即搜索：在假设空间中寻找匹配

那么，我们如何进行学习呢？关键在于训练数据。

*   **训练数据的作用**：外界提供一组输入-输出对（训练样本），例如“这个实例是正例”、“那个实例是负例”。
*   **搜索目标**：学习算法的任务就是在假设空间 `H` 中，找到一个能**正确标记所有已知训练样本**的假设。
*   **可能存在多个解**：通常，会有多个不同的假设（例如，不同的决策树）都能完美拟合提供的训练数据。它们对未知实例的预测可能不同，但在训练集上表现一致。
*   **算法即搜索策略**：我们之前讨论的决策树生长算法，本质上就是一种**启发式搜索策略**。它从最简单的树（空树）开始，基于信息增益等启发式规则，贪婪地添加属性测试，在假设空间中进行搜索，直到找到一个能拟合数据的树。

需要明确的是，尽管信息增益是一个优美的启发式准则，但使用它进行贪心搜索**并不能保证**一定找到能完美拟合数据的最简单树（如果存在的话）。这只是一种高效的启发式方法。然而，理论上，如果我们不惜一切代价持续生长树（例如，生长出完整的树，每个叶节点对应一个训练样本），只要存在一个能拟合数据的树，我们最终总能到达它。

## 总结

本节课中我们一起学习了：
1.  从**假设空间**的宏观视角来理解机器学习：学习是在由所有可能假设构成的空间中，寻找与训练数据匹配的点的过程。
2.  决策树学习算法是**假设空间**中的一种**启发式搜索**过程。
3.  训练数据用于指导搜索，目标是找到能正确标注所有训练样本的假设。
4.  可能存在多个能完美拟合训练数据的假设，学习算法输出的只是其中之一。
