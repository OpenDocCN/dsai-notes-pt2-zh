# 20：在线学习

在本节课中，我们将学习在线学习。这是一个与我们之前所学内容在表面上非常不同的主题，但它与我们第一节课中关于感知机的统计学习内容有着深刻的联系。

## 概述

在线学习是一种预测方法，其特点在于数据以序列形式逐个到达，算法必须在观察到真实结果之前做出预测。与传统的统计学习不同，在线学习的结果通常对任意数据序列都成立，而不需要对数据生成过程做任何假设。

## 从感知机到在线学习

让我们回顾一下感知机算法。我们曾以一种非常简单的方式证明了一个关于感知机在数据序列上犯错次数的上界。这个结果对任意数据序列都成立，这是一个非常强的结论，因为它不依赖于任何关于数据的假设。

随后，我们假设数据是独立同分布的，并将这个结果转化为关于独立同分布数据学习的陈述。事实上，仅仅通过几行推导，我们就为感知机证明了其样本外性能的保证。令人惊讶的是，我们可以将一个适用于任意序列的结果，转化为我们想要的形式。

## 在线学习的基本设定

今天，我们将讨论这种适用于任意序列的结果类型，这个领域我们称之为在线学习。有时我们会对序列做一些假设，但大多数结果对所有序列都成立。

那么，为什么我们需要一个对所有序列都成立的预测结果呢？

考虑一个简单的例子：石头剪刀布游戏。如果你完全随机地出拳，我无法做得比随机猜测更好。然而，当我们实际玩游戏时，我们试图通过观察对手过去的出拳模式来预测其下一步行动。这表明，如果序列不是完全随机的，我们就有可能利用其结构获得优势。

## 形式化协议

让我们形式化这个问题。假设有一个长度为 `n` 的序列 `y_1, ..., y_n`，其中每个 `y_t` 取值于 `{+1, -1}`。我们的目标是因果地预测这个序列，即：
1.  在观察到 `y_t` 之前，我们做出预测 `ŷ_t`（也取值于 `{+1, -1}`）。
2.  然后我们观察到真实的 `y_t`。

这个过程对 `t = 1` 到 `n` 重复进行。

我们关心的是平均错误次数：
```
L = (1/n) * Σ_{t=1}^{n} I(ŷ_t ≠ y_t)
```
其中 `I(·)` 是指示函数。

## 随机化策略与性能保证

为了获得理论保证，我们通常允许预测策略是随机化的。我们不是直接输出一个确定的 `ŷ_t`，而是输出一个分布 `Q̂_t`，其期望值在 `[-1, +1]` 之间，然后从这个分布中采样得到 `ŷ_t`。

我们希望得到一个保证：对于**任意**序列 `y_1, ..., y_n`，我们的（期望）平均错误次数 `L` 都能被某个函数 `Φ(y_1, ..., y_n)` 所上界。即：
```
E[L] ≤ Φ(y_1, ..., y_n)
```
这里的期望是针对我们算法自身的随机性。

函数 `Φ` 编码了我们希望利用的序列结构。例如，如果我们希望利用序列的不平衡性，可以定义 `Φ` 为序列中少数类标签的频率。

## 一个关键例子：伯努利序列

假设序列 `y_1, ..., y_n` 是独立同分布的伯努利序列，其偏倚为 `p`（即 `E[y_t] = p`）。如果我们知道这个假设，一个自然的策略是“多数投票”：根据已见数据中 `+1` 的频率是否超过 `1/2` 来预测。

在这种情况下，随着 `n` 增大，我们的错误率会收敛到 `min(p, 1-p)`，即少数类的频率。这是一个在随机性假设下可达到的性能。

在线学习的惊人之处在于：**存在一种策略，可以在不假设序列是伯努利分布的情况下，对任意序列达到接近 `min(频率(+1), 频率(-1))` 的错误率**。这意味着，只要对手产生的序列不平衡，我们就能利用这一点获得优于50%随机猜测的表现。

## 性能的“守恒定律”

一个重要的观察是性能的“守恒定律”。考虑所有可能的 `2^n` 个 `{+1, -1}` 序列。对于任何预测算法，当序列在这个超立方体上均匀随机选取（即每个比特独立以 `1/2` 概率为 `+1` 或 `-1`）时，其期望错误率必然是 `1/2`。

这意味着，**没有任何算法能在所有序列上都表现良好**。但是，我们可以设计算法，使其在我们“偏爱”的某些序列（如有不平衡性的序列）上表现优异，而这必然以在其他序列上表现更差为代价。函数 `Φ` 就体现了这种偏爱：它在我们偏爱的序列上值较小（即允许的错误少），在不偏爱的序列上值较大。

## 构建通用策略：势函数法

那么，如何为给定的势函数 `Φ` 构建一个满足保证的策略呢？一个核心结果是：如果 `Φ` 满足稳定性条件（即改变序列的任何一个坐标，`Φ` 值的变化不超过 `1/n`），并且在整个超立方体上的期望值至少为 `1/2`，那么就存在一个策略可以实现 `E[L] ≤ Φ(序列)`。

这个策略可以通过一个简单的“随机向前看”算法来计算。在时间 `t`，算法评估：如果我现在预测 `+1`，然后未来序列随机生成，`Φ` 的期望值是多少？如果预测 `-1` 呢？然后根据这两个期望值的差异来设置随机预测的偏倚 `Q̂_t`。这类似于博弈论中的平滑虚拟对局或强化学习中的蒙特卡洛树搜索思想。

## 通过子集定义势函数

一种通用且强大的定义 `Φ` 的方法是选择一个我们“偏爱”的序列集合 `F`（`F` 是 `{+1, -1}^n` 的一个子集）。然后定义 `Φ(y)` 为序列 `y` 到集合 `F` 的归一化汉明距离，再加上一个常数 `C`：
```
Φ(y) = C + (1/n) * min_{f in F} Σ_{i=1}^{n} I(y_i ≠ f_i)
```
这个 `Φ` 天然满足稳定性条件。常数 `C` 需要足够大，以确保 `Φ` 在整个超立方体上的期望至少为 `1/2`。理论分析表明，所需的最小 `C` 与集合 `F` 的**Rademacher复杂度**密切相关，这建立了在线学习与统计学习理论之间的深刻联系。

例如，如果我们想利用不平衡性，可以令 `F` 包含两个序列：全 `+1` 序列和全 `-1` 序列。此时，`Φ(y)` 就与 `min(频率(+1), 频率(-1))` 相关。

## 与统计学习的联系

上一节我们介绍了通过子集定义势函数的方法，本节我们来看看它如何与统计学习联系起来。如果我们引入协变量 `x_t`，这个框架可以扩展到更丰富的场景。此时，我们偏爱的对象可能不再是固定的序列，而是由某个函数类 `F` 生成的预测规则。相应的在线学习保证中，Rademacher复杂度将被其序列版本——**序列Rademacher复杂度**所取代，后者能处理协变量也依赖于历史的情况。

将在线学习保证转化为离线（统计学习）保证的过程是直接的：如果我们为适应性的、鞅差序列证明了某个结果，那么当数据是独立同分布时，该结果当然也成立。这揭示了在线分析与离线泛化界之间的对偶性。

## 总结

本节课中，我们一起学习了在线学习的基本框架。我们了解到：
1.  在线学习关注对任意数据序列的预测保证，无需随机性假设。
2.  性能通过一个势函数 `Φ` 来界定，`Φ` 编码了算法希望利用的序列结构。
3.  存在一个“守恒定律”：没有算法能在所有序列上表现都好，但可以针对特定结构进行优化。
4.  通过势函数法和“随机向前看”算法，可以构建满足保证的策略。
5.  选择偏爱序列的子集 `F` 来定义 `Φ`，能够自然地与统计学习中的复杂度度量（如Rademacher复杂度）联系起来。
6.  在线学习与统计学习理论之间存在深刻的对偶关系，在线分析可以为离线泛化提供保证。

在线学习是一个丰富而强大的框架，为我们理解序列预测和对抗环境下的学习提供了基础工具。
