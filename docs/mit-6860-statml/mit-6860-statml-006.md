# 6：在线学习算法

在本节课中，我们将学习在线学习算法的基本概念。我们将从经典的递归最小二乘法入手，理解如何增量式地更新模型参数，而无需在每次获得新数据时从头开始重新计算。接着，我们将探讨更通用的随机梯度下降法，了解其在不同场景下的应用和优势。

到目前为止，我们主要研究了基于几种不同思想的学习算法。

第一种思想是基于给定的损失函数，最小化数据的经验风险，并可能进行正则化。我们使用了一些技术，例如在线性系统情况下的最小二乘法，以及针对更一般损失函数的一阶方法。这些可以被视为批量学习算法，它们一次性处理所有数据。因此，如果获得新数据，就必须重新训练模型。

今天我们要探讨的是希望算法在获得更多数据时能够自适应更新，而无需从头重新计算的情况。我们将讨论几种可能对此感兴趣的不同场景。

第一种情况是数据以流的形式实时到达。你一次获得一批数据，但不想总是从头重新计算或训练所有内容。许多现实生活中的数据都属于这种情况，它们可能不满足我们之前一直使用的独立同分布假设。

另一种情况是数据不一定以流的形式到达，它们可能存储在笔记本电脑、台式机或GPU服务器上，但数据量非常大，无法一次性装入可用内存。此时，重点在于你需要以某种方式流式处理数据，分块读取，因为你无法一次性将所有数据放入内存。这在处理当今许多大型数据集时，至少与第一种情况同等重要。

今天的主题是**在线学习**。你可以用这个词来指代多种不同的情况。特别是，你可以强调问题本身或算法本身。问题可以是“在线”的，即数据一次到达一些。数据流的概念可以通过多种不同的方式精确化：可以假设数据是独立同分布并按序列到达；可以假设像时间序列模型那样，数据按序列到达，是平稳的但非独立的；也可以考虑一种情况，即在某种意义上存在一个有限的时间范围，在此期间你接收数据序列，但对数据如何产生不做任何建模假设，这更类似于博弈论的观点，有时被称为序列预测。我们不会过多讨论这种建模视角，而是更多地从算法角度出发。

因此，重点是算法如何处理数据。我们可以加上这个副标题：**在线学习算法**。算法本身无论何时获得数据，都一次处理一点。我们目前看到的算法并非如此。我们看到的本质上是基于最小化由某个损失函数（可能正则化）给出的经验风险的ERM方法。

我们之前看到的方法，可以称之为梯度方法。后者通过迭代计算解，即 **W_t = W_{t-1} - γ_t * G_t**，其中 **G_t** 是一个向量。我们见过几种不同的情况。如果损失函数可微，那么我们可以取 **G_t** 等于正则化经验风险在当前迭代点的梯度。这是基本的梯度下降迭代，此时我们可以取 **γ_t** 为常数。另一种情况是损失函数仅为凸函数但不可微，那么我们可以取 **G_t** 为正则化经验风险的一个次梯度。在这种情况下，我们需要取 **γ_t** 为一个衰减序列，例如随 **1/√t** 递减。

这些是我们上次介绍的主要思想。我们只处理了线性模型，然后通过特征映射或核函数跳转到非线性扩展。在这个过程中，我们在习题集中证明了多个表示定理。你们将被要求证明一个更常见的表示定理。我们通常将所有表明 **W** 可以写成输入点的线性组合的结果称为表示定理，因为这直接给出了 **W** 的重新参数化，为使用核函数铺平了道路。在讨论过程中，我们证明了几个这样的结果：**W = Σ_i c_i X_i**，其中 **X_i** 是输入训练点，**c_i** 是合适的系数，并且我们实际上提供了一种基于此计算它们的方法。

现在，我们要考虑一种情况，即我们希望一次处理一点数据，而不是一次性处理所有数据。从我们之前的讨论来看，在这些方法的每一步中，我们都使用了所有数据。如果我给你一个额外的点，你该怎么办？至少在这个框架下，你似乎需要从头重新计算所有内容：必须将额外的点加入，然后计算新的梯度。当然，你可以用当前计算的结果来初始化新的梯度迭代，但这并非标准做法，也不清楚能对此解给出何种保证。因此，我们希望更系统地研究在数据到达时更新解的想法。和之前一样，我们将首先使用最小二乘法作为一个简单的试验场，推导出一个简单的方程。我们将介绍经典思想中的基本算法之一——递归最小二乘法，这主要是为了使用几个线性代数技巧和方程。然后，我们将继续介绍随机梯度下降，并试图理解为什么随机梯度下降有用，以及在何种情况下使用它，因为关于如何以及何时使用随机梯度下降存在多种不同的观点。

## 递归最小二乘法

我们想要考虑递归最小二乘法。让我们称 **Ŵ_λ** 为正则化最小化器。我们之前已经看到，这可以写成 **Ŵ_λ = (X^T X + λI)^{-1} X^T y**。其中，**X** 是 m×d 的数据矩阵，**y** 是 m×1 的输出向量。我们引入以下紧凑符号：矩阵 **Σ_n = X^T X + λI**，这是数据的协方差矩阵或二阶矩矩阵，偏移了 λI。显然它依赖于 λ，但如果我们不关心这种依赖关系，我们想强调其对数据集大小的依赖，因为我们希望从 n 点过渡到 n+1 点，依此类推。

递归最小二乘算法的推导如下。它看起来有点像最小二乘的梯度下降迭代，但有几个关键区别。首先，这里你没有使用完整的梯度（即 **X^T (XW - y)**），而是用单点数据（矩阵的一行）代替了矩阵。其次，在经典梯度下降中，你在这里使用一个标量步长，而在这里，步长是一个矩阵，我们为其提供了一个更新规则。

这是一个在线学习算法的例子，因为一旦你给我一个新点，我就可以通过将其代入这个方程来计算一个新解。你不给我任何点，我就不做任何事。这就是我的解。你给我一个新点，我就更新解。与批量梯度下降相比，区别非常明显：这是一个在线梯度或随机梯度，因为你只访问总梯度的一小部分。

这个算法的内存成本是多少？时间成本是多少？主要的内存成本是存储这里的矩阵。注意它是常数大小，你添加更多数据时它不会改变。这是一个常数时间算法。成本本质上是这些向量和矩阵乘法，次数与你执行的步数一样多。你可以检查，这基本上与执行批量解的计算成本相同。批量解的内存成本是 O(nd) 来初始化矩阵，然后计算这个和那个，此时成本将是 O(nd + d²)，然后你必须求解这个系统，这将是 O(d³)。递归最小二乘法的成本在内存上是 O(d²)，在时间上是 O(nd²)。区别在于，如果我想从 n 点过渡到 n+1 点，除非我使用一些技巧，否则我必须重建矩阵并重新求逆，而在这里，我只需从这一步走到下一步。

关于初始化，你可以取这里的矩阵为单位矩阵，向量为零向量。我们将通过推导看到这一点。

递归最小二乘法的推导是初等的，只是一系列的观察和一些线性代数等式。第一个观察是，**Ŵ_n** 的解依赖于数据量 n。我们可以写出 n 点解和 n-1 点解的表达式，然后计算它们的差，并详细阐述这个差。最终，通过应用 Sherman-Morrison-Woodbury 公式来更新逆矩阵，我们可以得到递归更新规则。Sherman-Morrison-Woodbury 恒等式允许我们在已知矩阵 **A** 的逆的情况下，计算 **A + B C D** 的逆，其中 **B, C, D** 是适当维度的矩阵。在我们的情况下，这是一个秩为一的更新。

通过这个推导，我们得到了一个递归公式，它允许我们在获得新点时增量式地更新解，而无需重新计算所有内容。这个算法在每一步 n 都计算出与批量最小二乘法完全相同的解。因此，研究这个算法的任何性质，与研究精确的岭回归经验风险最小化估计器的性质是相同的。

## 随机梯度下降

现在，我们转向更通用的随机梯度下降。在递归最小二乘法中，我们使用了一个矩阵步长 **Γ_t**。在随机梯度下降中，我们将其替换为一个标量序列 **γ_t**。当然，现在我们不能再期望它在有限步内精确收敛到解；我们必须研究其收敛性。

我们可以为任何损失函数写下类似的更新规则。在每一步，你可以访问单点损失函数的梯度，然后更新你的解。如果损失函数不可微，你可以用次梯度代替。与经典梯度下降或随机梯度下降迭代相比，关键区别在于：在经典方法中，你计算所有梯度，然后更新解；而在这里，每次收到一个梯度（一个点），你就更新解。

这就是通常所说的随机梯度下降。更准确地说，如果损失函数不可微，它应该被称为随机次梯度方法，但通常简称为 SGD。

这个更新只使用常数内存。每一步的时间成本只是一个向量乘法，存储成本是计算梯度所需的内存。这让你感觉不需要将所有数据存储在内存中。

如何选择步长 **γ_t**？你可以通过研究其收敛性来决定目标，并推导出确保收敛到某处的选择。既然我们用小步长 **γ_t** 代替了大矩阵 **Γ_t**，我们甚至不知道它是否收敛。

在实践中，你很少一次只处理一个点，除非数据真的是一个接一个地到达。你通常做的是**小批量**处理。你不是选取一个点或所有点，而是选取你能快速处理的一批点，称为小批量，大小为 B。更新规则变为使用该批次中所有点的梯度平均值。这样做有几个原因：更好地利用内存；梯度估计的方差更小；以及便于并行化计算。

另一种流行的变体是**带动量的随机梯度下降**，它受到 Nesterov 加速方法的启发。基本思想是不仅使用前一次迭代，还使用更早的迭代，通过组合它们来获得更好的效果。这可以看作是一种惯性或加速项。更新规则在梯度计算中使用了一个组合向量 **V_t**，而不仅仅是前一次迭代 **W_{t-1}**。**V_t** 是 **W_{t-1}** 和 **W_{t-2}** 的组合。这是一种常见且实用的方法。

## 随机梯度的视角

现在，让我们更深入地理解随机梯度下降的“随机”部分。我们可以从两个角度来看待梯度项 **G_t**。

第一个角度是将其视为真实风险梯度的无偏估计。假设数据点 **(X_t, Y_t)** 来自真实的分布 **P(X,Y)**。那么，在固定参数 **W** 的情况下，**G_t** 的期望值就是真实风险 **R(W)** 的梯度。当我们实际运行算法时，**W_t** 依赖于过去的数据，而新点 **(X_t, Y_t)** 是新鲜的。在数据独立同分布的假设下，以过去为条件，**G_t** 仍然是真实风险梯度的无偏估计。因此，SGD 可以被视为求解一个随机优化问题，其中目标函数是一个期望，而我们只能访问其梯度的随机估计。

第二个角度是将其视为经验风险梯度的估计。经验风险可以看作是在训练数据索引集上的均匀分布下的期望。如果你从 {1, ..., n} 中均匀随机地抽取一个索引 **J**，那么对应数据点损失的梯度的期望，正好是经验风险 **R̂(W)** 的梯度。在这种视角下，如果你进行多次迭代（多次遍历数据），算法可能会收敛到经验风险的最小值。

这两种视角导致了不同的使用模式。第一种有时被称为**单遍SGD**，你每个数据点只使用一次，目标是真实风险。第二种被称为**多遍SGD**，你会多次查看每个数据点，目标是经验风险。在多遍SGD中，你还可以选择不同的数据访问顺序，例如随机洗牌后顺序访问，或者固定顺序循环访问，这有时被称为增量梯度法。

## 收敛性证明概述

最后，我们简要提一下随机梯度下降的收敛性证明。假设我们的目标函数 **F(W)** 是凸函数，并且其次梯度有界。我们选择衰减步长 **γ_t = C / √t**。那么，对于迭代序列 **W_t**，我们可以考虑其平均值 **Ŵ_n = (1/n) Σ_{t=1}^n W_t**。可以证明，平均解的目标函数值与最优值 **F(W^*)** 的差距以 **O(1/√n)** 的速率收敛。证明思路与经典次梯度方法的证明类似，在两边取期望后，利用次梯度的有界性即可完成。需要注意的是，对于SGD，经典结果通常针对平均迭代值，而非最后一次迭代值，因为该方法不是严格的下降法。

关于数据采样方式（有放回、无放回、洗牌）对收敛速度的影响，也存在相关的研究结果。

本节课中，我们一起学习了在线学习算法的核心思想。我们从递归最小二乘法开始，看到了如何通过线性代数技巧实现精确的增量更新。然后，我们介绍了更通用的随机梯度下降法，理解了它作为随机优化工具的本质，以及其小批量、带动量等实用变体。最后，我们概述了其收敛性理论。这些方法对于处理大规模或流式数据至关重要。
