在本节课中，我们将学习一种解决离线强化学习中“分布外动作”问题的新方法。与之前试图约束策略（Actor）的方法不同，本节介绍的方法直接修正Q函数中的过高估计问题。

![](img/fe7fa072ba65b87d0244db21544a2a41_5.png)

## 问题回顾：Q函数过高估计

![](img/fe7fa072ba65b87d0244db21544a2a41_7.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_9.png)

上一节我们介绍了离线强化学习中的核心挑战。本节中我们来看看另一种思路。

![](img/fe7fa072ba65b87d0244db21544a2a41_11.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_13.png)

正如之前讨论的，问题的根源在于贝尔曼备份（Bellman backup）过程中，我们选择了价值最大的动作。如果这些动作不在数据分布内，那么被选中的往往是那些被错误地、正向高估的动作。

![](img/fe7fa072ba65b87d0244db21544a2a41_15.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_17.png)

为了直观理解，假设绿色曲线代表真实的Q函数，蓝色曲线是我们的拟合函数。尽管蓝色曲线在大部分区域拟合良好，但如果我们选择蓝色曲线最大化的点，它很可能是在正方向上误差最大的那个点。

![](img/fe7fa072ba65b87d0244db21544a2a41_19.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_21.png)

## 解决方案：修正Q函数的目标

![](img/fe7fa072ba65b87d0244db21544a2a41_23.png)

因此，一个想法是：修改训练Q函数的目标函数。新的目标函数包含两部分。

![](img/fe7fa072ba65b87d0244db21544a2a41_25.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_27.png)

以下是目标函数的核心组成部分：

![](img/fe7fa072ba65b87d0244db21544a2a41_29.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_31.png)

1.  **标准贝尔曼误差项**：`L_贝尔曼 = E_{(s,a)~D} [ (Q(s,a) - (r + γ * max_{a'} Q(s', a')) )^2 ]`
    这部分与传统的Q学习相同，旨在最小化时序差分误差。

![](img/fe7fa072ba65b87d0244db21544a2a41_33.png)

2.  **惩罚高估项**：`L_惩罚 = α * E_{s~D} [ max_{μ(a|s)} E_{a~μ} [Q(s,a)] ]`
    这部分引入了一个新的分布μ。我们选择μ，使其在给定状态下期望的Q值尽可能大。然后，我们训练Q函数来**最小化**这些被μ选中的高Q值。直觉上，这个程序会找到那些错误的“峰值”并将它们向下推。

实际上，如果惩罚系数α选择得当，最终学到的Q函数将是真实Q函数的一个下界。然而，这个目标可能过于悲观，因为它会压低所有的Q值，导致无论有多少数据，恢复出的Q函数都永远不会是正确的。

![](img/fe7fa072ba65b87d0244db21544a2a41_35.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_37.png)

## 改进方案：平衡的Q函数学习

![](img/fe7fa072ba65b87d0244db21544a2a41_39.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_41.png)

为了解决过度悲观的问题，我们可以在目标函数中增加第三个项。

![](img/fe7fa072ba65b87d0244db21544a2a41_43.png)

以下是完整的目标函数：

![](img/fe7fa072ba65b87d0244db21544a2a41_45.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_47.png)

1.  **标准贝尔曼误差项**：`L_贝尔曼`（同上）。
2.  **惩罚高估项**：`L_惩罚`（同上）。
3.  **数据内奖励项**：`L_数据 = - E_{(s,a)~D} [ Q(s,a) ]`
    这项旨在**最大化**数据集中（即分布内）动作的Q值。

![](img/fe7fa072ba65b87d0244db21544a2a41_49.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_51.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_53.png)

这个组合最初看起来可能很奇怪，因为我们本意是防止高估。但让我们想象一下这个程序的实际效果：
*   如果所有高Q值对应的动作都在数据集中，那么惩罚项（推低μ选中的动作）和奖励项（推高数据集中的动作）会大致平衡，净效果很小。
*   如果高Q值对应的是与数据集差异很大的动作（分布外动作），惩罚项会将其Q值推低，而奖励项则会推高数据集内动作的Q值。这意味着下一次μ选择动作时，会更倾向于接近数据集的动作。

从某种意义上说，这是一个反馈过程：Q值越偏离数据分布，这两项合力就越会将其“推回”分布。一旦它们被完全推回，两项的作用又会相互抵消。

![](img/fe7fa072ba65b87d0244db21544a2a41_55.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_57.png)

这个更精细的目标函数不再保证学到的Q函数是所有状态-动作对真值的下界，但可以证明，**在策略的期望值上**，它仍然能得到保证，而这正是我们最终关心的。

![](img/fe7fa072ba65b87d0244db21544a2a41_59.png)

使用这一概念的算法（例如SQL - Soft Q-Learning）的一般结构如下：

![](img/fe7fa072ba65b87d0244db21544a2a41_69.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_71.png)

以下是算法的主要步骤：

![](img/fe7fa072ba65b87d0244db21544a2a41_73.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_75.png)

1.  **更新评论家（Critic - Q函数）**：
    *   使用包含上述三项（贝尔曼误差、惩罚项、数据奖励项）的目标函数来更新近似的Q函数。
    *   唯一改变的是评论家的损失函数。
2.  **更新演员（Actor - 策略）**：
    *   按照常规的演员-评论家方法更新策略π_θ，以最大化Q值的期望。
    *   对于离散动作，策略就是简单的`argmax_a Q(s,a)`，这与Q学习完全相同。
    *   对于连续动作，则需要一个显式的演员网络，并通过梯度上升来更新。

## 实践技巧：处理分布μ

![](img/fe7fa072ba65b87d0244db21544a2a41_79.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_81.png)

在实践中，我们需要为分布μ选择一个正则化项R(μ)。一个非常常见且方便的选择是使用**熵正则化**，即我们希望μ的熵较高。这本质上是一种最大熵正则化。

![](img/fe7fa072ba65b87d0244db21544a2a41_83.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_85.png)

结果证明，如果这样做，最优的μ与Q值的指数成正比：`μ(a|s) ∝ exp(Q(s,a))`。

![](img/fe7fa072ba65b87d0244db21544a2a41_87.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_89.png)

此外，在μ下Q函数的期望值，等于Q值的“对数-指数和”（Log-Sum-Exp）：`E_{a~μ}[Q(s,a)] = log( Σ_a exp(Q(s,a)) )`。

![](img/fe7fa072ba65b87d0244db21544a2a41_91.png)

这个性质非常方便：
*   **对于离散动作**：你甚至不需要显式构造μ，可以直接在损失函数中计算每个状态下的`log( Σ_a exp(Q(s,a)) )`项。
*   **对于连续动作**：你可以使用重要性采样来估计这个期望。例如，从当前策略或均匀分布中采样一批动作，然后利用`μ(a|s) ∝ exp(Q(s,a))`的事实，用指数Q值对它们进行重新加权。虽然这是重要性采样，但由于只涉及单步（而非多步累积），它通常效果很好且方差可控。

![](img/fe7fa072ba65b87d0244db21544a2a41_93.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_95.png)

## 总结

![](img/fe7fa072ba65b87d0244db21544a2a41_97.png)

![](img/fe7fa072ba65b87d0244db21544a2a41_99.png)

本节课中我们一起学习了处理离线RL中分布外动作的另一种核心方法。我们了解到，可以通过修改Q函数的学习目标，直接惩罚可能导致高估的Q值峰值，同时奖励数据分布内的动作。这种方法在SQL等算法中得以实现，并通过熵正则化等技巧变得易于计算。对于离散和连续动作空间，我们分别讨论了高效的计算策略。
