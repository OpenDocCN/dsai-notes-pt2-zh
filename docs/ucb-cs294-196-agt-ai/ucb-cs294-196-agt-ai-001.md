# 1：LLM 智能体概述

![](img/059af7517699ed545cde957ec87eab02_1.png)

在本节课中，我们将学习如何训练用于AI智能体的大型语言模型。我们将从宏观视角了解整个训练流程，包括预训练、后训练等核心阶段，并探讨每个阶段的目标、数据、计算成本以及关键挑战。

---

## 1.1：LLM 训练流程概览

上一节我们介绍了课程的整体目标，本节中我们来看看训练一个大型语言模型通常包含哪些主要阶段。

![](img/059af7517699ed545cde957ec87eab02_3.png)

训练一个现代LLM的流程通常包含三个主要部分：

1.  **预训练**：这是训练流程的第一步。其核心思想是让模型通过预测互联网文本中的下一个词，来学习关于世界的广泛知识。你可以将其理解为模型在“阅读”整个互联网。
    *   **数据规模**：非常庞大，通常超过10万亿个词元。
    *   **计算成本**：高昂，通常需要数百万美元和数月的训练时间。
    *   **目标**：获得一个“知识渊博”但尚未学会与人类交互的基础模型。例如，Llama 3就是一个预训练模型。

![](img/059af7517699ed545cde957ec87eab02_5.png)

![](img/059af7517699ed545cde957ec87eab02_6.png)

2.  **经典后训练**：这是在预训练之后进行的步骤，目的是让模型学会遵循指令并与人类有效交互。你可以将其理解为教一个知识渊博但不懂社交的人如何回答问题。
    *   **数据规模**：小得多，大约在10万到50万个示例之间。
    *   **计算成本**：相对较低，可能只需几天时间和约10万美元。
    *   **目标**：优化模型与人类的互动，确保它能理解并执行用户的请求。例如，`Llama-3-Instruct` 就是经过此类训练的模型。

3.  **推理强化学习**：这是一个较新的、专门针对需要逻辑推理的任务（如数学、编程）的训练阶段。它通常在经典后训练之后进行，目标是让模型学会“思考”并得出正确答案。
    *   **数据规模**：大约一百万个有明确答案的问题。
    *   **计算成本**：较高，可能需要数周时间和约一百万美元。
    *   **目标**：在具有客观答案的任务上优化模型性能。例如，DeepSeek-R1 就是第一个开源的推理模型。
    *   **关键挑战**：需要设计稳健的奖励机制，防止模型“作弊”或利用系统漏洞来优化奖励，而非真正解决问题。

![](img/059af7517699ed545cde957ec87eab02_8.png)

通常，我们将后两个阶段（经典后训练和推理强化学习）统称为 **后训练**，它们都是在预训练模型的基础上进行的专门化调整。

## 1.2：训练LM时需要考虑的五个方面

上一节我们介绍了训练流程的三个阶段，本节中我们来看看在具体实施训练时，需要统筹考虑的五个关键方面。

在训练一个语言模型时，你需要综合考虑以下五个要素：

1.  **架构**：模型的基础结构，例如 Transformer 或混合专家模型。
2.  **训练算法与损失函数**：如何优化模型参数，例如使用交叉熵损失进行下一个词预测。
3.  **数据与环境**：用什么数据训练，以及如何评估训练进度。
4.  **评估**：如何衡量模型的性能，确保训练朝着正确的方向前进。
5.  **系统与基础设施**：如何高效地利用计算资源进行大规模训练。

在2023年之前，学术研究主要聚焦于前两个方面（架构和算法）。然而，在实践中，真正起决定性作用的是后三个方面：**数据质量、评估方法和可扩展的系统基础设施**。架构虽然重要，但当前变化相对较小，且其影响通常不如高质量数据和高效评估来得关键。

## 1.3：预训练详解

上一节我们概述了训练的关键要素，本节中我们深入探讨第一个也是最重要的阶段：预训练。

### 方法：下一个词预测

预训练的核心方法是 **下一个词预测**。这类似于手机输入法的联想功能。模型的目标是，给定一段文本（如前文），预测最可能出现的下一个词是什么。

![](img/059af7517699ed545cde957ec87eab02_14.png)

**公式表示**：对于一个词序列 `(w1, w2, ..., wt-1)`，模型需要学习预测 `P(wt | w1, w2, ..., wt-1)`。

通过在海量数据上完成这个看似简单的任务，模型被迫学习语法、事实、逻辑关系乃至世界知识，因为它必须理解上下文才能做出准确预测。

### 数据：来自互联网的巨量文本

预训练的数据目标是尽可能多地使用互联网上“干净”的文本。这通常意味着需要处理原始HTML，提取文本，并进行多轮过滤和清洗。

![](img/059af7517699ed545cde957ec87eab02_16.png)

![](img/059af7517699ed545cde957ec87eab02_18.png)

![](img/059af7517699ed545cde957ec87eab02_20.png)

![](img/059af7517699ed545cde957ec87eab02_22.png)

以下是构建预训练数据集的典型流程：

1.  **下载**：使用 Common Crawl 等工具抓取数百亿网页。
2.  **提取**：从HTML中提取纯文本，这是一个计算量很大的步骤。
3.  **过滤**：
    *   移除不良内容（如个人身份信息、不安全内容）。
    *   **去重**：避免在完全相同或高度相似的文本上重复训练，这能显著提升效率。
    *   启发式过滤（如移除过长或过短的文档）。
    *   基于模型的过滤（例如，训练一个分类器来识别类似维基百科风格的高质量文本）。
4.  **数据混合与重加权**：对不同领域（如代码、书籍、新闻）的数据进行平衡，确保模型在各领域都有良好表现。
5.  **中期训练**：在主要预训练之后，使用一小部分已知的极高品质数据（如精选的学术论文、书籍）对模型进行额外训练，以进一步提升其能力。

研究表明，严格的数据过滤和去重能显著提升模型在相同计算预算下的性能。

### 计算：规模定律与扩展

![](img/059af7517699ed545cde957ec87eab02_24.png)

在预训练中，一个核心发现是 **缩放定律**：模型的性能（表现为测试损失）与训练所消耗的计算量之间存在可预测的幂律关系。

**关键启示**：
*   你可以在小规模实验上探索不同的超参数和架构，然后利用缩放定律预测它们在大规模训练时的表现。
*   这允许你将绝大部分计算资源用于最终的大模型训练，而非浪费在大量中等规模的试验上。

**计算最优分配**：著名的 Chinchilla 定律指出，对于给定计算预算，存在一个模型大小和训练数据量之间的最优平衡点（例如，每10亿参数大约需要200亿词元）。但需要注意的是，该定律仅优化训练成本，未考虑模型更大时带来的更高推理成本。

![](img/059af7517699ed545cde957ec87eab02_26.png)

![](img/059af7517699ed545cde957ec87eab02_28.png)

![](img/059af7517699ed545cde957ec87eab02_30.png)

**大数定律的启示**：由于计算能力持续增长（摩尔定律），且更多计算能带来更好性能，因此长期来看，最重要的研究方向是开发能够**有效利用不断增长的计算资源**的简单、可扩展的方法，而非过度优化当前规模下的复杂技巧。

## 1.4：后训练详解

![](img/059af7517699ed545cde957ec87eab02_38.png)

![](img/059af7517699ed545cde957ec87eab02_40.png)

![](img/059af7517699ed545cde957ec87eab02_42.png)

上一节我们深入了解了预训练，本节中我们来看看如何通过后训练让模型变得“有用”和“善解人意”。

![](img/059af7517699ed545cde957ec87eab02_44.png)

### 为什么需要后训练？

![](img/059af7517699ed545cde957ec87eab02_46.png)

![](img/059af7517699ed545cde957ec87eab02_48.png)

一个仅经过预训练的模型只是一个强大的“语言建模器”，它擅长续写文本，但未必能很好地协助用户。例如，当你要求它“向六岁孩子解释登月”时，它可能会开始列举其他类似的问题，而不是直接给出答案。后训练的目的就是**引导模型的行为，使其符合人类的期望和需求**。

![](img/059af7517699ed545cde957ec87eab02_50.png)

后训练主要分为两类：

![](img/059af7517699ed545cde957ec87eab02_52.png)

1.  **经典后训练**：也称为指令微调或对齐。目标是让模型学会遵循指令、以有帮助且安全的方式与人类互动。
2.  **推理强化学习**：专门针对数学、编程等有明确答案的复杂推理任务进行优化，教会模型“思考”的步骤。

![](img/059af7517699ed545cde957ec87eab02_54.png)

### 方法一：监督微调

![](img/059af7517699ed545cde957ec87eab02_56.png)

监督微调 是最直接的后训练方法。它使用与预训练相同的“下一个词预测”目标，但数据换成了人类或高级AI编写的“理想答案”。

![](img/059af7517699ed545cde957ec87eab02_58.png)

**数据来源**：
*   **人类标注**：高质量但昂贵、缓慢。
*   **模型合成**：使用现有强大模型（如 GPT-4）生成答案。这更廉价、快速，但前提是你有一个更强的模型可供学习。
*   **拒绝采样**：当没有明显更强的模型时，可以让当前模型对同一问题生成多个答案，然后通过规则（如代码测试）或验证器挑选出正确的答案，再用这些正确答案进行 SFT。

![](img/059af7517699ed545cde957ec87eab02_60.png)

SFT 可以教会模型指令遵循、特定格式、工具使用以及初步的推理链。它既可以作为最终部署的模型，也可以作为更复杂的强化学习阶段的良好起点。

![](img/059af7517699ed545cde957ec87eab02_62.png)

![](img/059af7517699ed545cde957ec87eab02_63.png)

![](img/059af7517699ed545cde957ec87eab02_64.png)

![](img/059af7517699ed545cde957ec87eab02_66.png)

### 方法二：强化学习

强化学习 的核心思想不是模仿已有的好答案，而是**主动优化一个奖励信号**，鼓励模型产生更受偏好的行为。

**奖励来源**：
*   **基于规则的奖励**：例如，代码是否通过测试用例，数学答案是否与标准答案匹配。
*   **奖励模型**：一个训练好的小型模型，用于预测人类会更喜欢哪个答案。
*   **LLM 即法官**：直接使用一个强大的 LLM 来判断回答的质量。

**典型流程（以 GRPO 算法为例）**：
1.  给定一个问题，让当前策略模型生成多个答案。
2.  使用奖励函数（如代码测试器）为每个答案打分。
3.  计算优势函数，判断每个答案相对于平均表现的好坏程度。
4.  通过策略梯度更新模型参数，增加生成高奖励答案的概率，同时通过 KL 散度惩罚防止模型偏离原始行为太远。

**基础设施挑战**：RL 训练，特别是针对智能体（涉及长序列交互）的训练，对基础设施要求极高。需要高效处理并发推理、环境反馈延迟、以及长序列的暂停与恢复等问题。Kimim 等项目的成功很大程度上得益于其精心设计的系统架构。

### 强化学习从人类反馈

![](img/059af7517699ed545cde957ec87eab02_68.png)

![](img/059af7517699ed545cde957ec87eab02_69.png)

当任务没有明确的对错标准（如创意写作、开放式对话）时，可以使用 RLHF。其经典流程是：
1.  收集人类对模型不同输出的偏好数据。
2.  训练一个**奖励模型**来学习人类的偏好。
3.  使用 RL 算法（如 PPO）优化策略模型，以最大化奖励模型给出的分数。

![](img/059af7517699ed545cde957ec87eab02_71.png)

![](img/059af7517699ed545cde957ec87eab02_73.png)

**挑战与演进**：人类标注成本高、速度慢，且标注者可能过度关注格式而非内容。因此，当前趋势是使用 **LLM 作为法官** 来替代人类进行偏好评估，这种方法成本极低且与人类判断有很高的相关性。

![](img/059af7517699ed545cde957ec87eab02_75.png)

![](img/059af7517699ed545cde957ec87eab02_76.png)

## 1.5：评估与系统优化

![](img/059af7517699ed545cde957ec87eab02_80.png)

![](img/059af7517699ed545cde957ec87eab02_82.png)

上一节我们介绍了后训练的方法，本节中我们来看看如何评估模型性能，以及如何通过系统优化来高效地进行大规模训练。

![](img/059af7517699ed545cde957ec87eab02_84.png)

### 评估：衡量进步的尺子

![](img/059af7517699ed545cde957ec87eab02_86.png)

评估是机器学习中最关键的环节之一，它用于：
1.  **量化进展**：判断训练是否有效，指导超参数调整。
2.  **模型选择**：为特定应用挑选最合适的模型。
3.  **上线决策**：判断模型是否已达到生产环境的要求。

![](img/059af7517699ed545cde957ec87eab02_88.png)

**评估类型**：
*   **封闭式评估**：任务有有限个标准答案（如多项选择题）。可以通过精确匹配或模糊匹配自动计算准确率。优点是自动化、可扩展；缺点是可能对提示词敏感，且存在训练数据污染的风险。
*   **开放式评估**：任务答案开放且冗长（如文章生成、对话）。自动化评估困难。
    *   **人类评估**：如 Chatbot Arena，让人类盲测两个模型的输出并选择更好的一个。结果可靠但成本高。
    *   **LLM 即法官**：使用一个强大的 LLM 来比较两个模型的输出，并计算胜率。这种方法成本极低（如 AlpacaEval），且与人类评估结果高度相关，已成为主流评估方式。

### 系统优化：释放计算潜力

由于缩放定律表明更多计算带来更好性能，而计算资源（GPU）既昂贵又稀缺，因此系统优化的目标就是**最大化每块 GPU 的利用率**。

![](img/059af7517699ed545cde957ec87eab02_90.png)

**GPU 基础**：
*   **核心特点**：大规模并行，专为矩阵乘法优化。
*   **现代瓶颈**：计算单元本身很快，瓶颈在于**将数据快速喂给计算单元**（内存带宽和通信）。

**关键优化技术**：
1.  **低精度计算**：使用 BF16 而非 FP32 进行矩阵运算，减少数据传输量和内存占用，同时基本不影响模型精度。
2.  **算子融合**：将多个连续的操作（如 LayerNorm 的各个步骤）融合成一个内核执行，避免中间结果反复写入和读取全局内存。
3.  **分块计算**：优化矩阵乘法等运算的内存访问模式，使得在从高速缓存中读取一次数据后，能进行尽可能多的计算操作，减少对慢速全局内存的访问。**Flash Attention** 就是成功运用此技术的著名例子。
4.  **并行化**：将模型和计算分布到多个 GPU 上。
    *   **数据并行**：每个 GPU 拥有完整的模型副本，处理不同的数据批次，然后同步梯度。简单，但不减少单卡内存占用。
    *   **模型并行**：
        *   **流水线并行**：将模型的不同层放在不同的 GPU 上。
        *   **张量并行**：将单个层内的矩阵运算拆分到多个 GPU 上。
    *   **零冗余优化器**：将优化器状态、梯度和模型参数本身**分片**存储在不同的 GPU 上，显著降低单卡内存需求，使得训练超大模型成为可能。
5.  **稀疏化**：如混合专家模型，每个输入只激活模型的一部分参数，从而在不增加计算量的情况下极大地增加模型总参数量，并便于分布式计算。

## 1.6：课程总结

![](img/059af7517699ed545cde957ec87eab02_97.png)

本节课中我们一起学习了训练大型语言模型用于AI智能体的完整流程与核心概念。

我们首先将训练流程分解为三个主要阶段：**预训练**（在海量互联网数据上学习世界知识）、**经典后训练**（对齐模型行为以遵循指令并与人类交互）以及**推理强化学习**（在数学、编程等有明确答案的任务上优化模型性能）。

接着，我们探讨了训练中需要统筹的五个方面，并指出在实践中，**数据质量、评估方法和系统基础设施**往往比模型架构和算法的微小创新更为关键。

对于预训练，我们深入了解了其基于**下一个词预测**的核心方法，处理**数万亿词元**级数据的复杂流程，以及**缩放定律**如何指导我们高效分配计算资源。

![](img/059af7517699ed545cde957ec87eab02_99.png)

对于后训练，我们比较了**监督微调**（模仿好答案）和**强化学习**（优化奖励信号）两种方法，并讨论了如何使用**基于规则的奖励**、**奖励模型**或 **LLM 即法官**来提供训练信号。

![](img/059af7517699ed545cde957ec87eab02_101.png)

最后，我们强调了**评估**对于衡量进展和选择模型的重要性，并概述了通过**低精度计算、算子融合、分块技术和多种并行化策略**来进行系统优化，以应对大规模训练在计算和内存上的严峻挑战。

![](img/059af7517699ed545cde957ec87eab02_103.png)

![](img/059af7517699ed545cde957ec87eab02_105.png)

理解这套完整的训练范式，是构建和优化下一代AI智能体的重要基础。
