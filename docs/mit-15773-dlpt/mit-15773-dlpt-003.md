训练神经网络本质上与其他模型训练没有区别。我们有一组参数（即权重和偏置），需要使用数据来找到这些权重的“好”值。“好”通常意味着我们定义了一个衡量模型预测值与真实答案之间差异的指标，然后尝试找到能最小化这个差异的权重。这个差异指标被称为**损失函数**。

![](img/eca06b94614d5e920ad1bd87a9832c01_50.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_52.png)

广义的训练流程是：定义一个网络，它接收输入，经过若干层，产生预测。然后，将预测值与真实值输入损失函数，得到损失分数。接着，将损失传递给优化器，优化器计算损失函数相对于所有参数的梯度，并使用该梯度更新所有权重。这个过程不断重复。

![](img/eca06b94614d5e920ad1bd87a9832c01_54.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_56.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_58.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_60.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_61.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_63.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_65.png)

我们使用的优化算法称为**梯度下降**。在每次迭代中，每个数据点都被用来进行预测、计算损失和梯度。然而，标准的梯度下降并不如**随机梯度下降**高效。

![](img/eca06b94614d5e920ad1bd87a9832c01_67.png)

随机梯度下降不是使用所有数据点，而是随机选择一小部分数据点（一个批次），假设这些是仅有的数据点，进行预测、计算损失和梯度，然后更新权重。

![](img/eca06b94614d5e920ad1bd87a9832c01_69.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_71.png)

## 理解Epoch与Batch

![](img/eca06b94614d5e920ad1bd87a9832c01_73.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_75.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_76.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_77.png)

由于我们每次只处理少量数据点，因此在开始训练模型时需要清楚其中的差异。

![](img/eca06b94614d5e920ad1bd87a9832c01_79.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_81.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_82.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_84.png)

**Epoch** 指的是对训练数据的一次完整遍历。在梯度下降中，一个epoch意味着每个数据点都通过网络一次，我们计算其预测、损失和梯度，然后在epoch结束时进行一次权重更新。

![](img/eca06b94614d5e920ad1bd87a9832c01_86.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_88.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_89.png)

在**随机梯度下降**中，我们以**批次**为单位处理数据。我们将训练数据划分为多个批次（例如batch1, batch2...）。对于每个批次，我们将其中的训练样本通过网络运行，计算梯度并更新参数，然后处理下一个批次。当所有批次都处理完毕时，就完成了一个epoch。

![](img/eca06b94614d5e920ad1bd87a9832c01_91.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_93.png)

因此，在随机梯度下降中，一个epoch内会进行多次权重更新，更新次数等于批次数。批次数由训练集大小除以批次大小并向上取整决定。例如，训练集有194个样本，批次大小为32，则批次数为 `ceil(194/32) = 7`。前6个批次各有32个样本，最后一个批次有2个样本。

![](img/eca06b94614d5e920ad1bd87a9832c01_95.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_97.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_99.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_101.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_103.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_105.png)

这种方法的优势在于，我们使用一小部分数据来近似计算梯度并更新权重，而不是等待所有数据处理完再更新。这样可以在训练过程中更频繁地调整模型，通常能更快地找到好的解。

![](img/eca06b94614d5e920ad1bd87a9832c01_107.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_109.png)

## 过拟合与正则化

现在，我们进入实际启动Colab前的最后一步：过拟合与正则化。

![](img/eca06b94614d5e920ad1bd87a9832c01_111.png)

在机器学习中，当模型变得越来越复杂时，训练误差通常会下降。然而，验证误差在下降到某一点后可能会开始上升，这种现象称为**过拟合**，意味着模型过度拟合了训练数据中的噪声而非一般规律。相反，模型还有很大改进空间时称为**欠拟合**。我们的目标是找到两者之间的“甜蜜点”。

![](img/eca06b94614d5e920ad1bd87a9832c01_113.png)

对于神经网络，层数越多、参数越多，模型越复杂，过拟合的风险也越高。因此，正则化至关重要。

![](img/eca06b94614d5e920ad1bd87a9832c01_115.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_116.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_117.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_118.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_119.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_120.png)

我们将使用两种正则化方法：
1.  **早停法**：将数据分为训练集和验证集。在训练过程中持续监控验证集上的损失（或准确率）。当验证损失停止下降并开始上升时，就停止训练。这是一种简单有效的正则化策略。
2.  **Dropout**：一种在训练过程中随机“丢弃”（即暂时忽略）网络中一部分神经元的技术。这可以防止神经元之间产生过强的依赖，增强模型的泛化能力。我们将在后续课程中详细讨论。

总结一下，构建和训练一个神经网络模型通常遵循以下清单：
1.  准备数据。
2.  设计网络结构（隐藏层数、神经元数等）。
3.  选择合适的输出层。
4.  选择合适的损失函数。
5.  选择优化器（如Adam，它是随机梯度下降的一种高效变体）。
6.  应用正则化策略（如早停法或Dropout）。

## TensorFlow与Keras简介

在进入实践之前，我们先快速了解TensorFlow和Keras。

**张量** 是神经网络中的基本数据单元，它是标量、向量、矩阵向更高维度的推广。一个数字是0阶张量，向量是1阶张量，矩阵是2阶张量，一组矩阵（如彩色图像的RGB通道）是3阶张量，视频可以看作是4阶张量。

![](img/eca06b94614d5e920ad1bd87a9832c01_146.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_148.png)

**TensorFlow** 是一个强大的深度学习库。其名称源于张量在网络中的流动。它主要提供以下功能：
*   **自动微分**：自动计算任意复杂损失函数的梯度，无需手动应用链式法则。
*   **优化器**：提供包括SGD及其变体在内的多种优化算法。
*   **分布式计算**：能够将计算负载自动分配到多台服务器上。
*   **GPU/TPU支持**：自动利用GPU或TPU等并行硬件加速计算。

**Keras** 是一个构建在TensorFlow之上的高级API，它提供了极大的便利性：
*   **层抽象**：提供了Dense（全连接层）、Conv2D（卷积层）等预定义层。
*   **易用性**：简化了模型构建、编译和训练过程。
*   **预处理工具**：提供了方便的数据预处理方法。
*   **评估与监控**：轻松计算和报告训练/验证损失、准确率等指标。
*   **预训练模型库**：提供了大量预训练模型，方便迁移学习。

![](img/eca06b94614d5e920ad1bd87a9832c01_150.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_152.png)

Keras主要有三种API：Sequential（顺序）、Functional（函数式）和Subclassing（子类化）。在本课程中，我们将主要使用功能强大且灵活的**函数式API**。

## 实践：心脏病预测模型

现在，让我们在Colab中实际构建之前设计的心脏病预测模型。模型结构如下：
*   输入层：对应29个特征。
*   隐藏层：一个具有16个神经元、使用ReLU激活函数的全连接层。
*   输出层：一个使用Sigmoid激活函数的神经元（用于二分类）。

以下是训练步骤的分解：

**1. 环境设置与数据加载**
首先，导入必要的库：TensorFlow、Keras、NumPy、Pandas和Matplotlib。为了结果可复现，设置随机种子。然后，加载心脏病数据集（一个Pandas DataFrame），其中包含303个样本和14列（13个特征+1个目标列）。

**2. 数据预处理**
数据包含分类变量和数值变量。
*   **分类变量**：使用独热编码将其转换为数值形式（例如，`pd.get_dummies`）。
*   **数值变量**：需要进行标准化（减去均值，除以标准差），使所有特征处于相近的尺度，这有助于网络训练。

**关键步骤**：在标准化**之前**，先将数据分割为训练集和测试集（例如80%/20%）。这可以防止测试集的信息“泄露”到训练过程中（例如影响均值和标准差的计算）。测试集在最终评估前应一直被“封存”。

![](img/eca06b94614d5e920ad1bd87a9832c01_154.png)

分割并锁定测试集后，用训练集计算均值和标准差，并对训练集和测试集分别进行标准化。

![](img/eca06b94614d5e920ad1bd87a9832c01_156.png)

**3. 数据准备**
将Pandas DataFrame转换为NumPy数组，因为Keras接收数组格式。然后将特征（X）和目标变量（y）分开。目标变量是二进制的（0或1）。

![](img/eca06b94614d5e920ad1bd87a9832c01_158.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_160.png)

**4. 模型构建**
使用Keras函数式API构建模型：
```python
inputs = tf.keras.Input(shape=(29,))
x = layers.Dense(16, activation='relu')(inputs)
outputs = layers.Dense(1, activation='sigmoid')(x)
model = tf.keras.Model(inputs=inputs, outputs=outputs)
```
使用 `model.summary()` 查看模型结构，确认参数数量（应为 `(29*16 + 16) + (16*1 + 1) = 497`）。

**5. 模型编译**
为模型指定优化器、损失函数和评估指标。
*   **损失函数**：因为是二分类问题，使用 `binary_crossentropy`。
*   **优化器**：使用 `Adam`。
*   **评估指标**：添加 `accuracy` 用于监控。
```python
model.compile(optimizer='adam',
              loss='binary_crossentropy',
              metrics=['accuracy'])
```

![](img/eca06b94614d5e920ad1bd87a9832c01_162.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_164.png)

**6. 模型训练**
使用 `model.fit` 进行训练。需要指定：
*   训练数据（X_train, y_train）。
*   训练轮数（epochs，例如300）。
*   批次大小（batch_size，例如32）。
*   验证集比例（validation_split，例如0.2），这将从训练集中再划分一部分作为验证集用于早停法监控。
训练过程中会输出每个epoch的训练损失/准确率和验证损失/准确率。

![](img/eca06b94614d5e920ad1bd87a9832c01_166.png)

**7. 结果分析与早停**
训练完成后，绘制训练损失和验证损失随epoch变化的曲线。观察验证损失何时停止下降并开始上升，那个点就是潜在的早停点。同样，绘制准确率曲线。有时损失和准确率指示的最佳停止点可能不同，通常更关注业务相关的指标（如准确率）。

![](img/eca06b94614d5e920ad1bd87a9832c01_168.png)

**8. 最终评估**
使用从未参与过训练或验证的**测试集**对最终模型进行一次评估，以获得模型在真实场景下性能的无偏估计。
```python
test_loss, test_acc = model.evaluate(X_test, y_test)
```
将这个准确率与一个简单的基线模型（例如，总是预测“无心脏病”，可获得约72.6%的准确率）进行比较，以衡量模型的提升。

![](img/eca06b94614d5e920ad1bd87a9832c01_170.png)

**1. 网络设计挑战**
输入是一张28x28的图像，即一个2阶张量。而之前我们的全连接网络输入是向量。解决方案是**展平**：将28x28的矩阵重塑为一个长度为784的向量。
`flattened_input = input_image.reshape(784)`

![](img/eca06b94614d5e920ad1bd87a9832c01_216.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_218.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_220.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_221.png)

**2. 网络结构**
*   **输入层**：接收展平后的784维向量。
*   **隐藏层**：一个具有128个（示例值）神经元、使用ReLU激活函数的全连接层。
*   **输出层**：这里面临新挑战。我们需要输出10个数字，分别对应10个类别的“得分”。但我们需要的是概率，且这10个概率之和必须为1（因为每张图只属于一个类别）。

![](img/eca06b94614d5e920ad1bd87a9832c01_223.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_225.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_226.png)

**3. Softmax层**
解决方案是使用 **Softmax激活函数**。它接收一个向量（这里是10个得分），对每个得分取指数（确保为正数），然后除以所有得分的指数和。这样就将原始得分转换成了一个概率分布。
`softmax_output = tf.nn.softmax(logits) # logits是10个得分`
Softmax是处理多分类问题输出层的标准选择。

![](img/eca06b94614d5e920ad1bd87a9832c01_228.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_229.png)

![](img/eca06b94614d5e920ad1bd87a9832c01_231.png)

**4. 损失函数**
对于多分类问题，损失函数需要相应调整：
*   如果标签是**独热编码**形式（如[0,0,1,0,...,0]），使用 `categorical_crossentropy`。
*   如果标签是**整数编码**形式（如“2”），使用 `sparse_categorical_crossentropy`。
Fashion-MNIST数据集通常提供整数标签，因此我们使用后者。
