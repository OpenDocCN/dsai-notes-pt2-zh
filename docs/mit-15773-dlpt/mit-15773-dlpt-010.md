## 指令微调详解

指令微调的核心思想是：显式地使用指令数据来训练模型。这个过程主要分为两个关键步骤。

![](img/19a063d68b83933a9f018c21d8f09222_75.png)

### 步骤一：监督式微调

首先，研究人员收集了约12,500个高质量的问题-答案对。例如，对于问题“向一个6岁的孩子解释登月”，人类会撰写一个合适的答案。然后，他们使用这些数据对GPT-3进行进一步的训练，训练方式依然是下一个词预测。这个过程被称为监督式微调，它显著提升了模型遵循指令的能力。

### 步骤二：基于人类反馈的强化学习

![](img/19a063d68b83933a9f018c21d8f09222_77.png)

然而，人工撰写高质量答案成本高昂。一个更高效的方法是让模型自己生成多个答案，然后由人类进行排序。

以下是具体流程：
1.  收集约30,000条指令。
2.  将每条指令输入GPT-3，通过采样生成多个答案。
3.  让人类评估员对这些答案进行排序（例如，给出“点赞”或“点踩”）。
4.  利用这些排序数据训练一个奖励模型。这个模型学习人类的偏好，能够为任何“指令-答案”对输出一个评分。
5.  使用这个奖励模型和强化学习技术来进一步优化GPT-3。模型生成答案后，由奖励模型评分，并使用该评分通过强化学习来调整GPT-3的内部权重，使其倾向于生成能获得高评分的答案。

![](img/19a063d68b83933a9f018c21d8f09222_79.png)

![](img/19a063d68b83933a9f018c21d8f09222_80.png)

![](img/19a063d68b83933a9f018c21d8f09222_82.png)

这个过程被称为基于人类反馈的强化学习。通过这两个步骤，我们得到了GPT-3.5（或称指令GPT）。为了将其转变为能够进行多轮对话的ChatGPT，只需使用对话形式的数据（即一连串的指令和回答）进行类似的训练即可。

### 零样本提示

![](img/19a063d68b83933a9f018c21d8f09222_112.png)

![](img/19a063d68b83933a9f018c21d8f09222_114.png)

![](img/19a063d68b83933a9f018c21d8f09222_116.png)

![](img/19a063d68b83933a9f018c21d8f09222_118.png)

这是最简单的方式，即直接在提示中清晰地向模型提出要求，不提供任何示例。例如，给定一条产品评论，我们可以直接提问：“这条产品评论是否描述了产品缺陷？” 模型通常能给出不错的回答。更好的模型在零样本提示上表现更佳。

提示工程在此非常重要。有时，通过精心设计提示（例如，将复杂问题分解为步骤），可以显著提升模型表现。

![](img/19a063d68b83933a9f018c21d8f09222_129.png)

### 少样本提示

在提示中，除了给出指令，还提供几个输入-输出的示例，这能帮助模型更好地理解任务。例如，要构建一个语法纠正器，可以在提示中给出几个“错误句子 -> 正确句子”的例子，然后再给出需要纠正的新句子。大语言模型这种仅通过少量示例就能学习任务的能力，被称为上下文学习，是GPT-3展现出的新兴能力。

### 检索增强生成

当我们需要模型利用私有知识库（如公司内部文档、FAQ）来回答问题，而这些信息不在其原始训练数据中时，可以使用RAG。

其核心思想是：由于模型有上下文窗口的长度限制，无法将整个知识库都放入提示中。因此，我们需要先根据用户问题，从知识库中检索出最相关的文档片段，再将它们和问题一起放入提示中。

具体步骤如下：
1.  将知识库文档切分成块。
2.  为每个文本块计算上下文嵌入向量。
3.  当用户提问时，计算问题的嵌入向量。
4.  通过计算余弦相似度，找出与问题向量最相似的几个文本块。
5.  将这些最相关的文本块作为“上下文”与用户问题一起构成提示，输入给大语言模型。

RAG是目前业界非常流行的应用模式。

### 微调

以上所有技术都未改变模型内部的权重。而微调则会使用领域特定的输入-输出示例，通过梯度下降更新模型的权重。

![](img/19a063d68b83933a9f018c21d8f09222_133.png)

微调的效果非常显著。例如，要让模型生成更像真实用户的产品评论，而不是营销文案，只需使用几百条“产品描述 -> 真实评论”的示例进行微调，模型的行为就会发生巨大改变。

![](img/19a063d68b83933a9f018c21d8f09222_137.png)

## 参数高效微调

![](img/19a063d68b83933a9f018c21d8f09222_139.png)

![](img/19a063d68b83933a9f018c21d8f09222_140.png)

![](img/19a063d68b83933a9f018c21d8f09222_142.png)

![](img/19a063d68b83933a9f018c21d8f09222_144.png)

![](img/19a063d68b83933a9f018c21d8f09222_145.png)

![](img/19a063d68b83933a9f018c21d8f09222_147.png)

![](img/19a063d68b83933a9f018c21d8f09222_149.png)

![](img/19a063d68b83933a9f018c21d8f09222_151.png)

对于像LLaMA-2 700亿参数这样的大模型，全参数微调需要巨大的计算资源和内存。为了在有限资源（如单个GPU）下进行微调，我们需要参数高效微调技术。

![](img/19a063d68b83933a9f018c21d8f09222_153.png)

![](img/19a063d68b83933a9f018c21d8f09222_155.png)

全参数微调的内存消耗主要来自模型参数、梯度以及优化器状态。对于大模型，这可能需要数百GB内存。

LoRA是一种广泛使用的参数高效微调方法。其核心洞察是：微调带来的权重变化矩阵可能具有较低的“内在秩”。因此，我们可以用两个小得多的矩阵的乘积来近似这个变化矩阵。

![](img/19a063d68b83933a9f018c21d8f09222_157.png)

![](img/19a063d68b83933a9f018c21d8f09222_159.png)

![](img/19a063d68b83933a9f018c21d8f09222_160.png)

![](img/19a063d68b83933a9f018c21d8f09222_162.png)

![](img/19a063d68b83933a9f018c21d8f09222_163.png)

![](img/19a063d68b83933a9f018c21d8f09222_165.png)

![](img/19a063d68b83933a9f018c21d8f09222_166.png)

![](img/19a063d68b83933a9f018c21d8f09222_168.png)

具体操作如下：
1.  冻结原始模型的所有权重。
2.  在原有的权重矩阵（如注意力层中的Key矩阵）旁，添加一对低秩矩阵。
3.  在微调时，只训练这些新添加的低秩矩阵，原始权重保持不变。
4.  在前向传播时，将原始权重矩阵的输出与低秩矩阵的输出相加。

![](img/19a063d68b83933a9f018c21d8f09222_170.png)

![](img/19a063d68b83933a9f018c21d8f09222_172.png)

![](img/19a063d68b83933a9f018c21d8f09222_174.png)

![](img/19a063d68b83933a9f018c21d8f09222_176.png)

![](img/19a063d68b83933a9f018c21d8f09222_178.png)

![](img/19a063d68b83933a9f018c21d8f09222_180.png)

由于只需要训练极少量的参数（可能只有原参数的0.01%），LoRA极大地降低了内存需求和计算成本，使得在消费级GPU上微调大模型成为可能。

*   **零样本/少样本提示**：无需训练，通过精心设计的提示与模型交互。
*   **检索增强生成**：结合外部知识库，增强模型回答的准确性和针对性。
*   **微调**：使用特定数据更新模型权重，使其深度适配特定任务。
*   **参数高效微调**：以LoRA为代表，通过仅训练少量新增参数，在资源受限的情况下实现大模型的高效适配。
