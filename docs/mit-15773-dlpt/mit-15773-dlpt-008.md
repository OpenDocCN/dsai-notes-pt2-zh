## 快速回顾：Transformer编码器

首先，让我们快速回顾Transformer编码器的基本流程。对于一个输入序列（例如“the train left the station”），我们首先获得每个词的独立（无上下文）词嵌入。接着，我们为每个词添加**位置嵌入**，以编码其在序列中的顺序信息。将词嵌入与位置嵌入逐元素相加，就得到了每个词的**位置输入嵌入**。

这个组合后的嵌入序列被送入**Transformer编码器堆栈**。经过堆栈处理后，输出的是**上下文嵌入**，每个词的嵌入都包含了来自序列中其他所有词的信息。

在上一节的槽位填充分类任务中，我们正是利用了这种架构：将自然语言查询输入，获得上下文嵌入，然后通过一个ReLU层和Softmax层对每个词的嵌入进行分类。

## 高效实现自注意力：矩阵运算视角

![](img/c78dec08c8829f89f1d0dd9f607eaee1_27.png)

自注意力机制的核心是计算序列中每个词与其他所有词的相关性。上一节我们以单个词（如“station”）为例，描述了如何通过点积、Softmax和加权平均来计算其上下文嵌入。然而，我们需要对序列中的**每一个词**都执行类似的计算。

为了高效地组织这些计算，我们可以利用矩阵运算。假设我们有一个简单的句子“I love huddle”，其位置编码后的嵌入矩阵为 **X**（形状为 `[序列长度, 嵌入维度]`）。

自注意力计算可以紧凑地表示为以下步骤：
1.  将输入矩阵 **X** 复制三份。
2.  计算 **X** 与其转置 **X^T** 的矩阵乘法，得到所有词对之间的点积矩阵。
3.  对该点积矩阵的每一行应用Softmax函数，得到注意力权重矩阵。
4.  将注意力权重矩阵与原始的 **X** 矩阵相乘，得到最终的上下文嵌入输出。

![](img/c78dec08c8829f89f1d0dd9f607eaee1_29.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_31.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_33.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_35.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_37.png)

整个过程可以用一个公式概括：`输出 = Softmax(X * X^T) * X`。这种表示方式使得整个自注意力层可以通过高度优化的GPU矩阵运算一次性完成，这是Transformer能够高效处理长序列的关键。

## 改进一：可调的自注意力层

![](img/c78dec08c8829f89f1d0dd9f607eaee1_48.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_50.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_52.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_54.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_56.png)

在基础的自注意力中，除了嵌入层，**自注意力层内部没有可学习的参数**。它只是对输入进行固定的点积和加权平均操作。为了增加模型的表达能力，我们引入可学习的权重矩阵。

![](img/c78dec08c8829f89f1d0dd9f607eaee1_58.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_60.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_62.png)

具体做法是，在生成用于计算注意力权重的 **Q** (查询)、**K** (键)、**V** (值) 向量之前，我们让输入 **X** 的三个副本分别通过三个不同的可学习线性变换矩阵 **A_Q**、**A_K**、**A_V**：
*   **K = X * A_K**
*   **Q = X * A_Q**
*   **V = X * A_V**

然后，我们使用变换后的 **Q**、**K**、**V** 来计算自注意力：
`输出 = Softmax(Q * K^T) * V`

![](img/c78dec08c8829f89f1d0dd9f607eaee1_64.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_66.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_68.png)

这样，自注意力层就变成了可学习的。每个注意力头都拥有自己独立的 **A_Q**、**A_K**、**A_V** 矩阵，使得不同的头可以学习关注输入序列中不同类型的模式和关系（例如语义、语法、语气等）。这就是Transformer论文中的核心公式。

![](img/c78dec08c8829f89f1d0dd9f607eaee1_70.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_72.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_74.png)

## 改进二与三：残差连接与层归一化

![](img/c78dec08c8829f89f1d0dd9f607eaee1_75.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_77.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_79.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_81.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_83.png)

为了构建更深、更稳定的网络，Transformer引入了另外两个关键组件。

**残差连接**：在自注意力层（以及后续的前馈神经网络层）中，我们将该层的输入直接加到其输出上，即 `输出 = 层输出 + 输入`。这确保了即使在深层网络中，梯度也能更顺畅地反向传播，缓解了梯度消失问题，并允许网络保留原始的输入信息。

**层归一化**：在残差连接之后，我们对结果进行层归一化处理。这包括计算该层所有神经元输出的均值和方差，并进行标准化，使激活值保持在稳定的范围内。这有助于加速训练并提高模型的泛化能力。

![](img/c78dec08c8829f89f1d0dd9f607eaee1_105.png)

结合以上所有改进，一个完整的Transformer编码器块包含以下步骤：
1.  **多头自注意力层**：输入经过多个可学习的自注意力头处理。
2.  **加与归一化**：将多头注意力的输出与原始输入相加（残差连接），然后进行层归一化。
3.  **前馈神经网络层**：将上一步的结果通过一个全连接前馈网络。
4.  **加与归一化**：再次应用残差连接和层归一化。

由于每个块的输入和输出维度相同，我们可以轻松地将多个这样的块堆叠起来，形成深层的Transformer编码器。

对于像情感分析这样的序列分类任务，我们需要将整个序列的信息汇总成一个向量。一个简单的方法是平均所有词的上下文嵌入，但这可能会丢失信息。

![](img/c78dec08c8829f89f1d0dd9f607eaee1_130.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_131.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_133.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_135.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_137.png)

更优雅的方法是使用**CLS（分类）令牌**。我们在每个输入序列的开头添加一个特殊的 `[CLS]` 令牌。这个令牌会与其他词一起参与Transformer的整个前向传播过程。由于自注意力机制，`[CLS]` 令牌的最终上下文嵌入会聚合整个序列的信息。训练完成后，我们只需取出这个 `[CLS]` 令牌的嵌入，将其通过一个全连接层和Sigmoid函数，即可完成分类。这种方法让模型自己学习如何为分类任务汇总信息。

## 使用Hugging Face快速应用

对于许多标准NLP任务（如文本分类、命名实体识别、问答），我们无需从头训练。**Hugging Face** 平台提供了海量的预训练模型和易用的 `pipeline` API。

以下是使用Hugging Face `pipeline` 的示例：
```python
from transformers import pipeline

![](img/c78dec08c8829f89f1d0dd9f607eaee1_183.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_184.png)

# 1. 情感分析
classifier = pipeline("sentiment-analysis")
result = classifier("I loved the movie! It was incredible.")
print(result)  # 输出: [{'label': 'POSITIVE', 'score': 0.9998}]

![](img/c78dec08c8829f89f1d0dd9f607eaee1_186.png)

# 2. 命名实体识别
ner_pipeline = pipeline("ner", grouped_entities=True)
result = ner_pipeline("Apple is looking at buying U.K. startup for $1 billion.")
print(result)  # 识别出组织、地点、货币等实体
```

## Transformer的多模态扩展

![](img/c78dec08c8829f89f1d0dd9f607eaee1_198.png)

Transformer的强大之处在于其架构与输入模态无关，关键在于如何将输入转化为嵌入。

![](img/c78dec08c8829f89f1d0dd9f607eaee1_200.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_202.png)

![](img/c78dec08c8829f89f1d0dd9f607eaee1_204.png)

*   **视觉Transformer**：将图像分割成多个小块，每个块展平后通过线性投影得到嵌入，并加上位置嵌入。然后像处理文本序列一样，将所有这些“图像块嵌入”送入Transformer。
*   **表格Transformer**：将分类特征转换为嵌入，连续特征保持不变。将所有分类特征的嵌入送入Transformer编码器，输出的上下文嵌入再与连续特征拼接，最后通过全连接层预测。
*   **多模态Transformer**：可以同时处理文本、图像、表格数据。只需为每种模态设计各自的嵌入器，将所有嵌入拼接（或连同 `[CLS]` 令牌一起）送入共享的Transformer编码器，模型会自动学习不同模态间的关系。
