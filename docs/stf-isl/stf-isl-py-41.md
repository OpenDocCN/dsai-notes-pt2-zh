# Python 版 41：逐步选择法 📊

![](img/a21dcc8971f562439889710809decaa4_1.png)

在本节课中，我们将要学习最优子集选择法的两种替代方法：**前向逐步选择**和**后向逐步选择**。这两种方法在计算效率和统计性能上通常优于最优子集选择，尤其是在预测变量数量较多时。

上一节我们介绍了最优子集选择法，它需要评估所有 `2^p` 个可能的模型。当预测变量数量 `p` 较大时（例如超过30或40），这种方法在计算上变得不可行，并且容易导致过拟合。本节中我们来看看如何通过逐步选择法来解决这些问题。

## 逐步选择法的动机 🎯

最优子集选择法存在两个主要问题：
1.  **计算负担**：需要考虑的模型数量为 `2^p`。当 `p=40` 时，这大约是1万亿个模型，计算量巨大。
2.  **统计过拟合**：评估如此多的模型会增加偶然发现一个在训练集上表现极佳但泛化能力差的模型的风险。

因此，即使计算能力允许，评估所有模型也可能不是最佳策略。为了避免“过度拟合”，有意识地只评估模型的一个子集反而可能更有益。

逐步选择法正是基于这一思想。它不评估所有 `2^p` 个模型，而是评估大约 `p^2` 个模型。对于任何合理的 `p` 值，`p^2` 都远小于 `2^p`。

## 前向逐步选择法 ➡️

前向逐步选择法从一个非常简单的模型开始，然后逐步添加预测变量。

以下是前向逐步选择法的具体步骤：

![](img/a21dcc8971f562439889710809decaa4_3.png)

![](img/a21dcc8971f562439889710809decaa4_4.png)

![](img/a21dcc8971f562439889710809decaa4_5.png)

![](img/a21dcc8971f562439889710809decaa4_6.png)

![](img/a21dcc8971f562439889710809decaa4_7.png)

![](img/a21dcc8971f562439889710809decaa4_9.png)

1.  **从零模型开始**：我们从一个不包含任何预测变量的模型开始，记作 `M0`。这个模型只包含截距项，对所有观测的预测值都是响应变量的均值。
2.  **逐步添加变量**：
    *   为了得到包含1个变量的模型 `M1`，我们考虑将 `p` 个预测变量中的每一个单独添加到 `M0` 中。我们选择那个能最大程度降低**残差平方和**或提升 `R^2` 的变量。这样就得到了 `M1`。
    *   为了得到包含2个变量的模型 `M2`，我们考虑将剩下的 `p-1` 个预测变量中的每一个添加到 `M1` 中。同样，选择能带来最大改进的变量。
    *   重复此过程，每一步都基于上一步的模型，添加一个能带来最大改进的新变量，直到所有 `p` 个预测变量都被加入模型。

**核心区别**：与最优子集选择法在每一步评估所有包含 `k` 个变量的模型不同，前向逐步法在每一步只评估那些**在上一步模型基础上增加一个变量**所形成的模型。这意味着我们得到的模型序列是**嵌套**的：`M1` 包含 `M0` 的变量加一个；`M2` 包含 `M1` 的变量加一个，依此类推。

最终，我们会得到 `p+1` 个模型：`M0, M1, M2, ..., Mp`。与最优子集选择一样，我们需要从中选出一个最佳模型。由于这些模型复杂度不同，我们不能直接使用 RSS 或 `R^2` 进行选择，而需要使用交叉验证、AIC 或 BIC 等准则。

### 计算优势

前向逐步选择法的计算量远小于最优子集选择：
*   对于 `M1`，我们评估 `p` 个模型。
*   对于 `M2`，我们评估 `p-1` 个模型。
*   ...
*   总计评估的模型数量约为 `1 + p + (p-1) + ... + 1 ≈ p^2/2`，即 `O(p^2)` 级别。

### 潜在缺点：可能不是全局最优

![](img/a21dcc8971f562439889710809decaa4_24.png)

![](img/a21dcc8971f562439889710809decaa4_26.png)

由于前向逐步法只探索了模型空间的一个子集，它找到的包含 `k` 个变量的模型，其 RSS 可能**高于**最优子集选择法找到的对应最佳模型。

![](img/a21dcc8971f562439889710809decaa4_28.png)

![](img/a21dcc8971f562439889710809decaa4_29.png)

![](img/a21dcc8971f562439889710809decaa4_31.png)

![](img/a21dcc8971f562439889710809decaa4_32.png)

![](img/a21dcc8971f562439889710809decaa4_34.png)

这种情况发生的原因是：**包含 `k` 个变量的最优模型，可能并不是包含 `k-1` 个变量的最优模型的超集**。前向逐步法因为路径依赖（必须基于上一步的模型），可能会错过这个全局最优模型。

![](img/a21dcc8971f562439889710809decaa4_36.png)

![](img/a21dcc8971f562439889710809decaa4_38.png)

![](img/a21dcc8971f562439889710809decaa4_40.png)

**示例**：在信用数据中，当寻找包含4个变量的最佳模型时：
*   **最优子集选择**给出的模型是：`cards`, `income`, `student`, `limit`。
*   **前向逐步选择**（基于其第三步得到的模型 `rating`, `income`, `student`）给出的模型是：`rating`, `income`, `student`, `limit`。

![](img/a21dcc8971f562439889710809decaa4_42.png)

![](img/a21dcc8971f562439889710809decaa4_44.png)

![](img/a21dcc8971f562439889710809decaa4_46.png)

![](img/a21dcc8971f562439889710809decaa4_48.png)

![](img/a21dcc8971f562439889710809decaa4_49.png)

这两个模型不同，且前者的训练集 RSS 更小。这种差异通常源于预测变量之间的相关性。如果所有变量都不相关，两种方法将给出相同的结果。

![](img/a21dcc8971f562439889710809decaa4_51.png)

![](img/a21dcc8971f562439889710809decaa4_53.png)

## 后向逐步选择法 ⬅️

后向逐步选择法的思路与前向法相反。

![](img/a21dcc8971f562439889710809decaa4_61.png)

以下是后向逐步选择法的具体步骤：

![](img/a21dcc8971f562439889710809decaa4_64.png)

![](img/a21dcc8971f562439889710809decaa4_66.png)

![](img/a21dcc8971f562439889710809decaa4_68.png)

1.  **从全模型开始**：我们从一个包含所有 `p` 个预测变量的模型开始，记作 `Mp`。
2.  **逐步删除变量**：
    *   为了得到包含 `p-1` 个变量的模型 `M_{p-1}`，我们尝试从全模型中删除一个变量。我们删除那个对模型拟合质量（如 RSS）损害最小的变量。
    *   接着，基于 `M_{p-1}`，我们删除一个对模型损害最小的变量，得到包含 `p-2` 个变量的模型 `M_{p-2}`。
    *   重复此过程，直到得到一个只包含截距的零模型 `M0`。

![](img/a21dcc8971f562439889710809decaa4_70.png)

![](img/a21dcc8971f562439889710809decaa4_72.png)

![](img/a21dcc8971f562439889710809decaa4_73.png)

后向逐步选择同样会产生一个嵌套的模型序列，并且也只评估大约 `O(p^2)` 个模型。它的限制是：**必须满足样本数量 `n` 大于预测变量数量 `p`**，这样才能拟合初始的全模型。而前向逐步选择即使 `p > n` 也可以从零模型开始。

![](img/a21dcc8971f562439889710809decaa4_76.png)

![](img/a21dcc8971f562439889710809decaa4_77.png)

## 总结 📝

![](img/a21dcc8971f562439889710809decaa4_80.png)

本节课中我们一起学习了两种逐步变量选择方法：
*   **前向逐步选择**：从零模型开始，逐步添加最重要的变量。
*   **后向逐步选择**：从全模型开始，逐步删除最不重要的变量。

![](img/a21dcc8971f562439889710809decaa4_81.png)

![](img/a21dcc8971f562439889710809decaa4_82.png)

![](img/a21dcc8971f562439889710809decaa4_84.png)

![](img/a21dcc8971f562439889710809decaa4_85.png)

![](img/a21dcc8971f562439889710809decaa4_86.png)

这两种方法都是对**最优子集选择法**的近似。它们的主要优势在于**计算效率**，只需评估约 `O(p^2)` 个模型，而非 `O(2^p)` 个。这使得它们能够处理预测变量数量较多的情况。然而，这种效率的提升是以**可能无法找到全局最优模型**为代价的，因为它们只探索了模型空间的一条路径。在实际应用中，当 `p` 较大时，逐步选择法通常是更可行且实用的选择。
