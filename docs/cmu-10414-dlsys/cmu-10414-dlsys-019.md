在PyTorch的LSTM文档中，公式看起来复杂，但实际上，其核心操作可以简化为几个关键矩阵的运算。模型内部主要有两个权重矩阵：`weight_ih` 和 `weight_hh`。

![](img/fedb479405b637e3d017ec260a8164aa_23.png)

*   `weight_hh` 是一个 400x100 的矩阵，对应公式中的 **W_hh**，用于处理上一个时间步的隐藏状态 `h_{t-1}`。
*   `weight_ih` 是一个 400x20 的矩阵，对应公式中的 **W_ih**，用于处理当前时间步的输入 `x_t`。

![](img/fedb479405b637e3d017ec260a8164aa_25.png)

![](img/fedb479405b637e3d017ec260a8164aa_26.png)

![](img/fedb479405b637e3d017ec260a8164aa_27.png)

![](img/fedb479405b637e3d017ec260a8164aa_29.png)

此外，还有一个偏置项 `bias`。这些矩阵共同作用，通过一个大的线性变换和后续的非线性激活函数（sigmoid和tanh）来计算LSTM的各个门控信号和候选值。

```python
def sigmoid(x):
    return 1 / (1 + np.exp(-x))
```

接下来，定义LSTM单元的前向传播函数。该函数接收当前输入`x`、上一个隐藏状态`h`、上一个细胞状态`c`以及权重和偏置。

![](img/fedb479405b637e3d017ec260a8164aa_60.png)

![](img/fedb479405b637e3d017ec260a8164aa_62.png)

![](img/fedb479405b637e3d017ec260a8164aa_64.png)

![](img/fedb479405b637e3d017ec260a8164aa_66.png)

![](img/fedb479405b637e3d017ec260a8164aa_67.png)

![](img/fedb479405b637e3d017ec260a8164aa_69.png)

```python
def lstm_cell(x, h, c, W_hh, W_ih, b):
    # 合并线性变换: W_hh * h + W_ih * x + b
    linear = np.dot(W_hh, h) + np.dot(W_ih, x) + b
    # 将结果分割为输入门(i)、遗忘门(f)、候选值(g)、输出门(o)
    i, f, g, o = np.split(linear, 4)
    # 应用非线性激活函数
    i = sigmoid(i)
    f = sigmoid(f)
    g = np.tanh(g)
    o = sigmoid(o)
    # 计算新的细胞状态和隐藏状态
    c_out = f * c + i * g
    h_out = o * np.tanh(c_out)
    return h_out, c_out
```

## 实现序列LSTM

单步LSTM单元是基础，但通常我们需要处理整个输入序列。本节我们将实现一个处理完整序列的LSTM函数。

![](img/fedb479405b637e3d017ec260a8164aa_110.png)

这个函数将迭代序列中的每个时间步，依次调用`lstm_cell`，并收集所有时间步的隐藏状态。

![](img/fedb479405b637e3d017ec260a8164aa_112.png)

```python
def lstm_sequence(X, h0, c0, W_hh, W_ih, b):
    # X形状: (序列长度, 输入维度)
    # 初始化输出隐藏状态序列
    H = np.zeros((X.shape[0], h0.shape[0]))
    h = h0.copy()
    c = c0.copy()
    for t in range(X.shape[0]):
        h, c = lstm_cell(X[t], h, c, W_hh, W_ih, b)
        H[t] = h
    # 返回所有隐藏状态和最后一个细胞状态（遵循PyTorch惯例）
    return H, c
```

## 批处理实现与内存布局

![](img/fedb479405b637e3d017ec260a8164aa_143.png)

![](img/fedb479405b637e3d017ec260a8164aa_144.png)

![](img/fedb479405b637e3d017ec260a8164aa_146.png)

为了计算效率，我们希望同时处理一个批次（batch）的多个序列。然而，RNN的时序依赖性使得直接批处理变得复杂。关键在于理解并优化内存布局。

![](img/fedb479405b637e3d017ec260a8164aa_148.png)

![](img/fedb479405b637e3d017ec260a8164aa_150.png)

![](img/fedb479405b637e3d017ec260a8164aa_152.png)

标准的批处理维度在前（`[batch, seq_len, features]`）的布局，在按时间步切片时数据在内存中不连续，不利于高效的矩阵乘法。因此，对于RNN，通常采用`[seq_len, batch, features]`的布局，这样每个时间步的所有批次样本在内存中是连续的。

![](img/fedb479405b637e3d017ec260a8164aa_154.png)

![](img/fedb479405b637e3d017ec260a8164aa_155.png)

![](img/fedb479405b637e3d017ec260a8164aa_157.png)

![](img/fedb479405b637e3d017ec260a8164aa_159.png)

![](img/fedb479405b637e3d017ec260a8164aa_161.png)

我们需要修改LSTM单元和序列函数以支持批处理。以下是支持批处理的LSTM单元：

![](img/fedb479405b637e3d017ec260a8164aa_163.png)

![](img/fedb479405b637e3d017ec260a8164aa_164.png)

![](img/fedb479405b637e3d017ec260a8164aa_166.png)

![](img/fedb479405b637e3d017ec260a8164aa_168.png)

```python
def lstm_cell_batch(X_t, H_t_1, C_t_1, W_hh, W_ih, b):
    # X_t形状: (batch_size, input_size)
    # H_t_1形状: (batch_size, hidden_size)
    # 线性变换
    linear = np.dot(H_t_1, W_hh.T) + np.dot(X_t, W_ih.T) + b
    i, f, g, o = np.split(linear, 4, axis=1)
    i = sigmoid(i)
    f = sigmoid(f)
    g = np.tanh(g)
    o = sigmoid(o)
    C_t = f * C_t_1 + i * g
    H_t = o * np.tanh(C_t)
    return H_t, C_t

def lstm_sequence_batch(X, H0, C0, W_hh, W_ih, b):
    # X形状: (seq_len, batch_size, input_size)
    seq_len, batch_size, _ = X.shape
    hidden_size = H0.shape[1]
    H = np.zeros((seq_len, batch_size, hidden_size))
    h = H0.copy()
    c = C0.copy()
    for t in range(seq_len):
        h, c = lstm_cell_batch(X[t], h, c, W_hh, W_ih, b)
        H[t] = h
    return H, c
```

## 训练LSTM与截断反向传播

![](img/fedb479405b637e3d017ec260a8164aa_208.png)

![](img/fedb479405b637e3d017ec260a8164aa_210.png)

有了LSTM的前向实现，在自动微分框架（如Needle或PyTorch）中训练它就变得简单直接：定义损失函数，调用`.backward()`计算梯度，然后用优化器更新权重。

![](img/fedb479405b637e3d017ec260a8164aa_212.png)

然而，当序列非常长时（例如10,000个时间步），存储所有中间状态用于反向传播会消耗巨大内存。解决方案是使用**截断反向传播通过时间**。

其核心思想是将长序列切分成较短的块（例如每块100个时间步），并分别在这些块上执行前向和反向传播。但简单地独立训练每个块会丢失块间的长期依赖信息。

![](img/fedb479405b637e3d017ec260a8164aa_214.png)

为了改善这一点，我们使用**隐藏状态重打包**技术。在训练完一个块后，我们取其最后一个隐藏状态和细胞状态的值（使用`.detach()`将其从计算图中分离，仅复制数值，不保留梯度历史），并将其作为下一个块的初始状态。这样，信息可以在序列中传递，同时又限制了反向传播的计算图长度，控制了内存使用。

![](img/fedb479405b637e3d017ec260a8164aa_216.png)

以下是训练循环的概念性伪代码：

![](img/fedb479405b637e3d017ec260a8164aa_217.png)

```python

block_size = 100
num_blocks = seq_len // block_size

    start = i * block_size
    end = (i+1) * block_size
    X_block = X[start:end] # 获取当前数据块
    # 前向传播
    H_block, (h, c) = lstm(X_block, (h, c))
    # 计算损失...
    loss = compute_loss(H_block, Y_block)
    # 反向传播（仅在本块内）
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()
    # 隐藏状态重打包：分离值，用于下一块
    h = h.detach()
    c = c.detach()
```
