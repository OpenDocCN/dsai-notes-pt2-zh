## 主方法回顾 📊

![](img/e421a4c005a04e895836ae0f2ae0a4f6_7.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_9.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_10.png)

上一节我们介绍了课程概述，本节中我们来看看用于分析分治算法复杂度的有力工具——主方法。它适用于求解特定形式的递归式。

递归式通常具有以下形式：
`T(n) = a * T(n/b) + f(n)`
其中：
*   `a` 是子问题的数量。
*   `n/b` 是每个子问题的规模。
*   `f(n)` 是分解和合并子问题的代价。

主方法的核心思想是比较递归部分的工作量 `n^(log_b(a))` 与分解合并部分的代价 `f(n)` 的增长速度。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_12.png)

以下是主方法的三种情况：

1.  **若 `f(n) = O(n^(log_b(a) - ε))` (ε > 0)**： 则递归代价占主导，解为 `T(n) = Θ(n^(log_b(a)))`。
2.  **若 `f(n) = Θ(n^(log_b(a)) * log^k n)`**： 则递归与分解合并代价相当，解为 `T(n) = Θ(n^(log_b(a)) * log^(k+1) n)`。一个典型例子是 `a = b = 2`，`f(n) = Θ(n)`，得到 `T(n) = Θ(n log n)`。
3.  **若 `f(n) = Ω(n^(log_b(a) + ε))` (ε > 0)**： 则分解合并代价占主导，解为 `T(n) = Θ(f(n))`。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_14.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_16.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_18.png)

直观理解是：算法总复杂度由两者中较大者决定，若两者相当，则增加一个对数因子。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_20.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_22.png)

## 选择问题：中位数的中位数算法 🎯

![](img/e421a4c005a04e895836ae0f2ae0a4f6_26.png)

上一节我们介绍了分析工具，本节中我们来看看一个经典问题：**选择问题**，即在一个未排序的长度为 `n` 的列表中寻找第 `K` 小的元素。这是许多算法（如寻找中位数、分位数、聚类、神经网络梯度选择）的基础模块。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_28.png)

一个简单的思路是借鉴快速排序的“快速选择”算法：随机选取一个枢轴元素，将数据划分为小于和大于枢轴的两部分，然后只在包含目标元素的那一部分递归。但其最坏情况时间复杂度为 `O(n^2)`。

为了保证线性时间复杂度，我们使用 **“中位数的中位数”** 算法。其核心思想是：高效地找到一个“好”的枢轴，确保每次递归都能丢弃足够比例的元素。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_30.png)

以下是算法步骤：

![](img/e421a4c005a04e895836ae0f2ae0a4f6_32.png)

1.  **分组**： 将输入列表划分为 `n/5` 个小组，每组5个元素。
2.  **找各组中位数**： 找出每个5元素小组的中位数。由于小组大小固定，此操作是常数时间。
3.  **找中位数的中位数**： 递归调用选择算法，从这 `n/5` 个中位数中找出其中位数 `M`。`M` 即为我们的枢轴。
4.  **分区**： 用 `M` 将原始列表划分为三个集合：
    *   `A`: 所有小于 `M` 的元素。
    *   `B`: 所有等于 `M` 的元素。
    *   `C`: 所有大于 `M` 的元素。
5.  **递归或返回**：
    *   若 `K ≤ |A|`，则在集合 `A` 中递归寻找第 `K` 小元素。
    *   若 `|A| < K ≤ |A| + |B|`，则 `M` 就是第 `K` 小元素，返回 `M`。
    *   否则，在集合 `C` 中递归寻找第 `K - |A| - |B|` 小的元素。

**关键分析**： 可以证明，通过这种方式选择的枢轴 `M`，能保证集合 `A` 和 `C` 的规模都至少是 `3n/10`。因此，每次递归我们至少丢弃 `3n/10` 个元素，最坏情况下递归的规模不超过 `7n/10`。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_34.png)

递归式为：`T(n) = T(n/5) + T(7n/10) + O(n)`。
应用主方法（或归纳法）可解得 `T(n) = O(n)`，即线性时间复杂度。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_36.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_38.png)

## 并行选择算法 ⚡

![](img/e421a4c005a04e895836ae0f2ae0a4f6_40.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_42.png)

上一节我们学习了线性的串行选择算法，本节中我们将其并行化。一个朴素的想法是让每个处理器计算其本地数据的中位数，然后收集这些中位数并找出全局中位数作为枢轴。但这可能导致负载不均衡，因为数据分布未知，某些处理器可能始终保留大部分数据。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_44.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_46.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_48.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_50.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_51.png)

为此，我们引入 **加权中位数的中位数** 算法。

1.  **本地中位数**： 每个处理器 `i` 计算其本地数据的中位数 `m_i`，并记录其本地数据量 `w_i`（即权重）。
2.  **全局加权中位数**： 收集所有 `(m_i, w_i)` 到某个处理器。**加权中位数** `M` 定义为：当按 `m_i` 排序后，累计权重首次超过总权重一半时所对应的 `m_i`。
3.  **分区**： 每个处理器根据 `M` 将其本地数据划分为小于、等于、大于 `M` 的三部分，并计算各部分大小。
4.  **全局聚合与决策**： 通过归约操作汇总全局小于、等于 `M` 的元素总数。
    *   若目标秩 `K` 落在对应区间，则返回 `M` 或在对应子集中递归。
    *   可以证明，无论数据如何分布，每次递归至少能丢弃全局 `1/4` 的元素。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_53.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_55.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_57.png)

**复杂度分析**：
*   **通信**： 每轮递归需要收集 `P` 个中位数并进行归约，通信成本为 `O(α log P + β P)`。递归深度为 `O(log n)`，总通信成本约为 `O(α log P log n + β P log n)`。可通过在问题规模降至 `O(n/P)` 时聚集到单个处理器来优化，使深度变为 `O(log P)`。
*   **计算**： 最坏情况下，某些处理器的本地数据量可能无法显著减少，因此本地计算成本可能为 `O((n/P) log n)`。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_59.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_61.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_63.png)

此算法控制了通信开销，但计算负载可能不均衡。存在更复杂的随机化算法能提供更好的理论界限。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_65.png)

## 共享内存排序：并行归并排序 🔀

![](img/e421a4c005a04e895836ae0f2ae0a4f6_69.png)

上一节我们探讨了选择问题，本节开始讨论排序。首先看共享内存模型下的并行归并排序。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_71.png)

归并排序是经典的分治排序算法：
1.  递归地将数组分成两半。
2.  分别排序左右两半。
3.  合并两个已排序的子数组。

串行合并两个已排序数组的时间复杂度为 `O(n)`。如果只是简单地并行化递归调用（`spawn`），但合并步骤仍是串行的，那么算法的**跨度**（关键路径长度）为：
`S(n) = S(n/2) + O(n) = O(n)`
而总**工作量** `W(n) = O(n log n)`。这意味着并行度仅为 `W/S = O(log n)`，非常低。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_73.png)

为了提高并行度，必须**并行化合并步骤**。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_75.png)

**并行合并算法**：
要合并两个已排序数组 `A` (大小为 `n`) 和 `B` (大小为 `m`，设 `n >= m`)。
1.  在较大的数组 `A` 中选取其中位数 `A[mid]`。
2.  在数组 `B` 中使用二分查找找到 `A[mid]` 应插入的位置 `pos`。
3.  现在，原合并问题被划分为两个独立的子问题：
    *   合并 `A[0:mid]` 和 `B[0:pos]`。
    *   合并 `A[mid+1:]` 和 `B[pos:]`。
4.  递归地并行合并这两个子数组。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_77.png)

可以证明，每次递归后，待合并的总规模至少减少 `1/4`。因此并行合并的跨度 `S_merge(n) = S_merge(3n/4) + O(log n) = O(log² n)`。

**完整的并行归并排序**：
现在，排序的递归调用和合并调用都可以并行。
*   **工作量**： `W(n) = 2W(n/2) + W_merge(n) = O(n log n)`（与串行算法相同，是工作高效的）。
*   **跨度**： `S(n) = S(n/2) + S_merge(n) = S(n/2) + O(log² n) = O(log³ n)`。

![](img/e421a4c005a04e895836ae0f2ae0a4f6_79.png)

![](img/e421a4c005a04e895836ae0f2ae0a4f6_81.png)

因此，并行归并排序的并行度为 `W/S = O(n / log² n)`，对于大规模数据，这提供了巨大的并行潜力。

## 分布式内存排序：样本排序 📤

![](img/e421a4c005a04e895836ae0f2ae0a4f6_85.png)

上一节我们讨论了共享内存排序，本节转向分布式内存模型。一个简单的想法是**桶排序**：若数据范围已知且分布均匀，可以为每个处理器分配一个值域区间（桶），元素被送到对应桶所在的处理器，然后各处理器本地排序。但这对数据分布要求苛刻。

**样本排序** 通过采样来智能地确定桶的边界（分割器），以适应任意分布的数据。

以下是基于**规则采样**的样本排序算法步骤：

1.  **本地排序与采样**： 每个处理器对其 `n/P` 个本地元素进行排序。然后，从本地有序数组中**规则采样**，每隔 `n/(P²)` 个元素取一个样，共取 `P-1` 个样本。
2.  **收集与选择分割器**： 将所有 `P*(P-1)` 个样本收集到一个处理器。将这些样本排序后，从中**规则采样**出 `P-1` 个**全局分割器**：即选择排序后样本数组中第 `P/2, 3P/2, 5P/2, ...` 位置的元素。
3.  **广播分割器**： 将这 `P-1` 个全局分割器广播给所有处理器。
4.  **本地划分**： 每个处理器根据接收到的分割器，使用二分查找确定其本地每个元素应属于哪个桶（即哪个处理器）。
5.  **全局数据交换**： 每个处理器将其元素发送到对应的目标处理器。可以证明，每个处理器最终接收的元素数量不超过 `2n/P`，实现了良好的负载均衡。
6.  **本地归并排序**： 每个处理器收到所有属于自己区间的元素后，进行本地归并排序。

**算法分析**：
*   **本地排序**： `O((n/P) log(n/P))`
*   **收集样本**： `O(α log P + β P²)` （收集 `P*(P-1)` 个样本）
*   **选择分割器**： `O(P² log P)` （排序样本）
*   **广播分割器**： `O(α log P + β P)`
*   **本地划分**： `O(P log(n/P))` （`P-1` 次二分查找）
*   **全局数据交换**： `O(α log P + β n/P)` （最多发送/接收 `O(n/P)` 数据）
*   **最终本地排序**： `O((n/P) log(n/P))`

总成本由通信项 `β P²` 和 `β n/P` 主导。当 `P` 很大时，收集样本的 `P²` 项可能成为瓶颈。改进方法包括：使用中位数算法递归选择分割器以减少样本量，或在分布式数组中并行选择分割器。
