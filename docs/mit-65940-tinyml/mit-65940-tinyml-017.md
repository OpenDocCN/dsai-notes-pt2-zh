# 017：GAN、视频与点云理解

在本节课中，我们将学习针对特定计算密集型应用的优化技术。我们将重点探讨用于图像生成的生成对抗网络、视频理解以及点云处理。这些应用一方面计算量巨大，另一方面又存在大量冗余，为优化提供了空间。

## 高效GAN 🖼️

上一节我们介绍了课程概述，本节中我们来看看生成对抗网络。GAN由生成器和判别器两部分组成。生成器的目标是生成足以“欺骗”判别器的图像，使其无法区分生成图像与真实图像。判别器的目标则是准确区分两者。

在推理阶段，我们只使用生成器。因此，压缩和优化的重点应放在生成器部分。

### GAN的背景与分类

GAN主要用于生成逼真的图像。它主要分为两类：
*   **无条件GAN**：生成器以随机噪声作为输入，生成输出图像。
*   **条件GAN**：生成器和判别器的输入都附加了条件信息，例如类别标签或分割图。

尽管GAN功能强大，但其模型通常比识别模型（如图像分类模型）大得多，计算成本高昂，这促使我们对GAN模型进行压缩。

### GAN压缩技术

我们可以应用之前学到的知识来压缩GAN模型。我们的目标是通过减少参数量来压缩一个大型生成器。

以下是实现压缩时采用的三种损失函数：
1.  **重建损失**：确保输出图像在像素级别上与目标一致。
2.  **特征匹配损失**：匹配原始大型模型与压缩模型中间层特征图的分布，需要使用卷积投影层来对齐不同通道数的特征。
3.  **原始GAN损失**：即条件GAN的对抗损失。

我们将这三种损失结合起来进行训练。可以采用Once-For-All超网络方法：训练一个支持不同通道数的超网络，然后通过神经架构搜索来选择满足我们延迟要求的通道配置。

**结果**：对于图像转换任务（如“马变斑马”），压缩后的模型可以缩小11倍，同时生成质量与原始模型相当，并且能够在移动GPU上实现实时推理。

### Anycost GAN用于图像编辑 🎨

GAN可用于图像编辑，但生成过程缓慢，难以在笔记本或平板电脑上实现交互式编辑。通常的方法是将图像投影到一个潜在空间，通过调整潜在空间中的向量来控制生成图像的属性（如微笑、发色）。

为了加速预览，我们受启发于快速预览时降低图像采样率的方法，希望使用一个计算量小的子网络进行快速预览，当用户满意时，再使用完整的、质量更高的模型生成最终图像。

为了实现这一目标，我们需要在两个方面支持弹性变化：
1.  **支持不同分辨率**：预览时使用低分辨率，最终生成时使用高分辨率。
2.  **支持不同通道数**：预览时使用少通道数的轻量子网络，最终生成时使用多通道数的完整网络。

**关键技术**：
*   **多分辨率训练**：在训练时，不仅将最高分辨率图像输入判别器，还会随机采样不同分辨率的生成图像输入判别器，使得生成器在所有分辨率下都能生成逼真图像。
*   **生成器条件判别器**：判别器需要知道当前生成器的配置（如通道数、分辨率），以便为不同大小的生成器提供有效的梯度反馈。使用单一的判别器来处理差异巨大的生成器效果不佳。因此，我们将生成器的架构信息也输入判别器，对其进行调节。

**结果**：这种方法可以在不同计算预算下生成视觉上一致且高质量的图像。在图像编辑应用中，它能实现快速的交互式预览和高质量最终输出，并且已开源，可部署在MacBook等设备上。

### 数据高效的GAN训练 📊

训练GAN通常需要大量数据，而数据收集和标注成本高昂。我们尝试用有限的数据（如100张或1000张图像）来训练GAN，但现有技术在数据有限时效果很差。

量化结果显示，随着训练数据减少，生成图像的质量指标（如FID）会显著下降。这是因为在小数据集上，判别器容易过拟合，导致其无法为生成器提供有效的学习信号，进而使生成器性能恶化。

**解决方案：可微数据增强**
传统的图像增强（如旋转、颜色抖动、平移、裁剪）能有效防止判别器过拟合。但关键是如何将其应用于GAN训练。

如果只对真实图像进行增强，判别器会认为增强后的伪影（如平移、颜色变化）是真实图像应有的特性，从而引导生成器也产生这些伪影，这不是我们想要的。

正确的做法是**对真实图像和生成图像都应用相同的增强变换**。并且，在训练生成器时，也需要考虑增强变换，确保梯度能正确回传。这种方法称为可微数据增强。

**结果**：应用可微数据增强后，即使只用100张图像训练，生成质量也得到大幅提升，并且稳定。模型还能在不同图像属性（如季节、昼夜）之间进行平滑插值。

## 高效视频理解 🎥

视频数据量巨大且增长迅速，理解视频对于边缘计算和自动驾驶等应用至关重要。与图像相比，视频理解的关键在于建模时序信息。同时，视频帧之间存在大量时空冗余（如静态背景），这为高效处理提供了机会。

### 2D CNN与3D CNN方法

我们先回顾视频理解的两种主流方法：
*   **基于2D CNN的方法**：对采样出的视频帧分别用2D CNN处理，然后聚合预测结果。其优点是计算高效且能利用图像预训练模型，但无法有效建模帧间时序关系。虽然可以通过加入光流或使用LSTM进行后期融合来引入时序信息，但光流计算成本高，后期融合会丢失低层时序信息。
*   **基于3D CNN的方法**：使用3D卷积核同时在空间和时序维度上进行卷积，能联合建模时空信息。但其参数量和计算量远大于2D CNN。一种改进是I3D，通过“膨胀”预训练的2D卷积核权重来初始化3D卷积核。

3D CNN虽然性能好，但计算成本高。因此，我们自然要问：能否以接近2D CNN的成本，达到3D CNN的性能？

### 时序移位模块

TSM的核心思想是使用**移位操作**来实现高效的时序建模。移位操作零计算、零参数，但能有效地在通道维度上混合不同帧的信息。

**工作原理**：
假设特征图在通道维度上进行分组。对于某一帧的特征，我们将部分通道的数据替换为前一帧或后一帧对应通道的数据。这样，当前帧的特征就包含了邻近帧的信息。后续的2D卷积操作会在通道维度上融合这些来自不同时刻的信息。

TSM可以以即插即用的方式插入到现有的2D CNN架构中，几乎不增加计算开销即可获得时序建模能力。

**离线与在线模式**：
*   **离线模式**：可用于动作识别等任务，可以同时利用过去和未来的帧信息。
*   **在线模式**：用于自动驾驶等流式处理场景，只能利用历史帧信息。通过多层TSM的堆叠，信息可以随时间向后传播，使当前帧能间接获得较长历史的信息。

**性能**：TSM在精度上媲美甚至超过3D CNN方法，同时计算成本仅与2D CNN相当，实现了显著的加速。在移动设备上也能实现高效的实时推理。

**可扩展性**：TSM模型更小，通信开销低，因此在大规模分布式训练中展现出优异的扩展性，能有效利用数百个GPU节点加速训练。

**可解释性与应用**：通过可视化时空注意力，可以理解模型决策的依据。TSM已应用于手势识别等需要时序理解的实时交互场景。

## 高效点云理解 ☁️

点云是3D传感器（如LiDAR）采集的数据，由空间中的一系列（x， y， z）坐标点及其可能附加的特征（如反射强度）组成。点云在自动驾驶、AR/VR中应用广泛。

**挑战**：点云具有**高度稀疏性**和**不规则性**。这与稠密、规则的2D图像网格截然不同，导致传统的卷积神经网络难以直接高效处理，且不规则的存储器访问模式在GPU上效率低下。

### Point-Voxel CNN

PVCNN是一种混合架构，旨在结合体素方法和点方法的优点，它包含两个分支：
1.  **体素分支**：将3D空间划分为规则的体素网格，将落入同一体素的点赋予相同特征。然后使用标准的3D卷积进行处理。其优点是计算规则、高效，适合聚合局部邻域信息。但缺点是会丢失细节，尤其是当体素分辨率较低时，不同点可能被“坍缩”到同一体素中。
2.  **点分支**：直接在原始点云上运行，进行逐点的特征变换（类似于PointNet）。其优点是可以保持高分辨率，区分每个点，但缺乏对局部几何结构的显式建模。

**协同工作**：体素分支擅长处理大范围的、结构化的场景信息（如桌面、墙面），而点分支专注于精细的、局部的细节（如物体边缘、小部件）。两者特征在后期融合，从而在效率和精度之间取得良好平衡。

**性能**：PVCNN在点云分割等任务上，相比纯点方法或纯体素方法，能实现更快的速度和更高的精度，具有优越的内存-精度权衡。

### 进一步优化：稀疏卷积与架构搜索

为了进一步提升体素分支的效率并减少信息损失，可以采用**稀疏卷积**。它只对非空的体素进行计算，避免了在大量空白体素上的无效操作。

此外，可以针对点云任务进行**神经架构搜索**，自动寻找在给定延迟约束下的最优PVCNN架构配置，从而进一步优化效率。

### 多模态融合：鸟瞰图空间

在实际应用中（如自动驾驶），常常需要融合来自不同传感器（如摄像头和LiDAR）的数据。一个关键的挑战是找到一个共享的表示空间。

**解决方案**：使用**鸟瞰图**作为共享空间。鸟瞰图是从正上方俯视的场景2D投影。
*   摄像头图像可以通过透视变换投影到BEV空间。
*   LiDAR点云可以自然地映射到BEV空间（假设物体不会垂直重叠）。

在BEV空间中，来自不同模态的特征可以被对齐和融合，然后用于完成3D物体检测、地图分割等任务。
