# 024：深度压缩自编码器简介 🧠

在本节课中，我们将要学习一种名为“深度压缩自编码器”的技术。这项技术能够显著压缩图像数据，从而大幅降低后续模型（如扩散模型）的计算负载，实现更快的图像生成速度。

## 压缩原理与收益

上一节我们介绍了压缩的基本概念，本节中我们来看看深度压缩的具体实现和效果。

压缩程度越高，计算工作量就越少。具体表现为分辨率降低，需要处理的令牌数量减少。

我们最近发明了一种深度压缩方法。它可以将数据压缩64倍。这意味着在每个维度上压缩8倍，总体令牌数量减少64倍。因为每次压缩都使令牌数量减半，所以最终结果是64倍更少的令牌。

这种压缩对注意力计算的影响是巨大的。注意力计算复杂度与令牌数量的平方相关。将令牌总数减少64倍，意味着计算量减少4096倍。

## 方法对比与挑战

以下是原始方法与深度压缩方法的对比：

*   **原始稳定扩散方法**：使用变分自编码器将分辨率降低8倍，然后传递给扩散模型，速度较慢。
*   **我们提出的方法**：使用深度压缩自编码器，在每个维度上进行更大幅度的压缩，从而使其快得多。

然而，实现这种压缩并非易事。例如，一个普通的自编码器在64倍的压缩率下，生成质量会很差。但使用深度压缩自编码器，质量则好得多。

在质量与压缩率的权衡中，压缩率越高越理想，但也越困难。基于某些方法，当压缩率提高时，FID分数会迅速下降。但使用深度压缩方法，即使在128倍的压缩率下，FID分数也能保持非常稳定。

## 核心技术：残差重编码

那么，我们是如何实现这一点的呢？关键在于降低分辨率时采用的方法。

一个直接的方法是**将信息从空间维度显式地转移到通道维度**。例如，每2x2的四个像素，可以从 `2 x 2 x 1` 的格式转换为 `1 x 1 x 4` 的格式。这就是从空间到通道的显式转换。

我们添加了这个显式的空间到通道分支，并将其与自编码器的主分支相加。我们称之为**残差重编码**。与其让网络权重去学习这种转换，不如使用一个零权重、无需训练的分支作为残差旁路来学习它。我们发现，这种残差重编码结构使学习变得容易得多。例如，没有残差重编码时，FID分数在压缩率提高时会快速恶化；而有了它，即使在64倍的高压缩率下，FID也能保持非常稳定。

## 三阶段高分辨率适配技术

第二个关键技术是**面向高分辨率的三阶段适配**。我们希望处理像4K这样的高分辨率图像生成，但如何让自编码器支持如此高的分辨率，同时又不需要大量的训练计算呢？

我们将训练解耦为三个阶段：

1.  **低分辨率全模型训练**：首先，在较低分辨率下完整训练编码器和解码器。给定一个图像，编码器将其编码到潜在空间，解码器再将其重建回原始图像。
2.  **高分辨率中间层适配**：冻结大部分参数，只对模型的中间层进行高分辨率潜在空间适配。中间层对应于分类器丢弃层之后的部分，主要与高级语义相关。
3.  **解码器末层精调**：最后，添加分类器引导，仅对解码器的最后一层进行精调，因为这仅与低级特征相关。低级特征实际上是在最后阶段被精调的。

我们进行这种**分层稀疏调优**，并非调优所有层，而是稀疏地调优中间层或后期层，以节省计算量，使训练此类自编码器非常快速。

## 应用效果与总结

使用这种自编码器，我们可以极大地加速图像生成模型。

*   使用原始潜在空间（8倍压缩）时，生成速度较慢，每秒仅能生成17张图像。
*   使用我们的深度压缩自编码器（64倍压缩），将分辨率从512降至8x8，我们可以在几乎相同的计算成本下，处理高达4K的分辨率，速度快了19倍。

因此，自编码器至关重要。如果你能应用这种深度压缩自编码器，就可以处理少得多的令牌，从而**平方级地减少模型推理时的计算量**。

另一个好处是，现在我们可以在不使用大量计算的情况下实现高分辨率图像生成。例如，4K图像生成。与1K图像相比，4K图像包含更多的细节。如果不使用深度压缩自编码器这类技术，将很难高效地生成如此高分辨率的图像。

本节课中我们一起学习了深度压缩自编码器的核心原理，包括其通过残差重编码实现高质量高倍率压缩的方法，以及通过三阶段适配高效支持高分辨率图像生成的技术。这些技术能显著降低计算负载，加速图像生成过程。
