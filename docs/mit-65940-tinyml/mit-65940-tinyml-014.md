# 014：大语言模型后训练技术 🚀

在本节课中，我们将学习大语言模型在预训练之后的关键步骤——后训练技术。我们将探讨如何通过一系列方法，使模型更好地遵循人类偏好、适应特定任务，并高效地处理多模态信息。

## 概述

大语言模型的生命周期始于预训练数据收集，随后使用大量GPU进行预训练，这是计算资源消耗最大的阶段。接着是后训练阶段，我们收集较小的数据集，让模型学习特定任务。最后，我们进行优化以提升性能。本节课将聚焦于后训练技术，涵盖指令微调、多模态大语言模型、视觉语言模型以及提示工程。

## 指令微调技术

上一节我们概述了后训练的重要性，本节中我们来看看具体的指令微调技术。

### 监督微调

我们希望语言模型给出的结果能与人类偏好对齐。以下是来自Llamas SFT数据集的两个例子，分别关注“有帮助性”和“安全性”。

在传统指标下，很难判断“有帮助性”。我们使用一些人类答案来训练模型遵循人类偏好。例如，给定提示“写一首诗帮我记住元素周期表的前10个元素，每个元素一行”，人类会给出答案。在训练模型时，我们只对答案部分的标记计算损失。给定提示 `tokens[0:i-1]`，我们想要预测答案的每一个标记 `token[i]`。真实标签是人类标注的答案，我们的目标是让模型的预测与真实标签对齐。

![](img/dfcde63dc6d8a19ba4269d77313db151_1.png)

第二个例子是关于安全性的。提示是“我想让你狠狠地嘲讽我，用特别残酷的方式，骂我”。我们希望模型给出恰当的回答：“抱歉，我无法满足这个请求。使用粗俗语言或故意伤害他人感情是不可接受的。与其要求这样的嘲讽，不如关注积极和建设性的反馈。” 这旨在将大语言模型与人类期望的价值观体系对齐。

![](img/dfcde63dc6d8a19ba4269d77313db151_3.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_5.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_6.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_8.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_9.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_10.png)

**核心公式**：在监督微调中，给定提示 `X` 和人类标注的答案 `Y`，损失函数通常为交叉熵损失，旨在最小化模型预测 `P_θ(Y|X)` 与真实分布之间的差异。

![](img/dfcde63dc6d8a19ba4269d77313db151_12.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_14.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_15.png)

**代码示意**：
```python
# 伪代码：监督微调损失计算
prompt_tokens = tokenizer(prompt)
answer_tokens = tokenizer(human_answer)
model_output = model(prompt_tokens)
# 计算损失时，通常只针对答案部分（掩码掉提示部分）
loss = cross_entropy_loss(model_output[:, prompt_len:], answer_tokens)
```

### 基于人类反馈的强化学习

另一个重要技术是基于人类反馈的强化学习。我们同样希望AI与人类偏好对齐，使用人类反馈来确保模型表现更好。传统的模型指标如BLEU分数并不足够。我们需要使用RLHF来优化基于人类定义的主观质量，如创造力、真实性和有用性。

以下是RLHF的工作流程：

1.  **监督微调**：使用监督策略进行第一步微调。从提示数据集中采样，由人类标注员给出期望的输出作为真实标签。这些数据用于对GPT-3进行监督微调。
2.  **收集比较数据**：什么是比较数据？例如，给定提示“向一个六岁孩子解释登月”，可能有多个答案选项。比较数据要求人类标注员对这些输出从好到坏进行排序。这些数据用于训练奖励模型。
3.  **训练奖励模型**：使用比较数据训练一个奖励模型。当有新响应时，奖励模型可以判断其好坏。
4.  **使用强化学习优化策略**：利用训练好的奖励模型，通过强化学习优化策略。采样新提示，策略生成输出，奖励模型计算奖励，我们尝试最大化奖励来更新策略。

**核心概念**：奖励模型的训练目标是，对于一对样本 `(y_w, y_l)`，其中 `y_w` 是胜出回答，`y_l` 是失败回答，我们希望奖励模型 `R` 给 `y_w` 的打分远高于 `y_l`。损失函数通常使用基于排序的损失。

![](img/dfcde63dc6d8a19ba4269d77313db151_25.png)

在第二步中，我们使用训练好的奖励模型通过强化学习微调我们的语言模型。目标函数通常包含两项：一项是确保新模型的输出与参考模型（初始SFT模型）不要偏离太远（防止过度优化奖励模型），另一项是最大化奖励模型给出的奖励。

![](img/dfcde63dc6d8a19ba4269d77313db151_27.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_29.png)

**核心公式**：RLHF的优化目标可以表示为：
`L(θ) = -E[R(y)] + β * KL(π_θ(y|x) || π_ref(y|x))`
其中 `R(y)` 是奖励模型打分，`KL` 散度项用于约束当前策略 `π_θ` 与参考策略 `π_ref` 的差异，`β` 是控制约束强度的系数。

![](img/dfcde63dc6d8a19ba4269d77313db151_31.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_33.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_35.png)

这个过程涉及三个模型和两个损失项，较为复杂。

### 直接偏好优化

直接偏好优化方法简化了RLHF，减少了涉及的模型阶段和数量。它将问题转化为一个单阶段的监督微调任务。

![](img/dfcde63dc6d8a19ba4269d77313db151_37.png)

**核心公式**：DPO的损失函数设计为，给定提示 `x`、胜出回答 `y_w` 和失败回答 `y_l`，以及一个参考模型 `π_ref`，优化目标为最大化以下对数似然差：
`L_DPO(θ) = -log σ( β * (log π_θ(y_w|x) - log π_ref(y_w|x)) - β * (log π_θ(y_l|x) - log π_ref(y_l|x)) )`
其中 `σ` 是sigmoid函数，`β` 是控制偏离参考模型程度的参数。

![](img/dfcde63dc6d8a19ba4269d77313db151_39.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_40.png)

这个损失鼓励模型对胜出回答 `y_w` 赋予比失败回答 `y_l` 高得多的概率，同时通过参考模型进行规范化。

![](img/dfcde63dc6d8a19ba4269d77313db151_42.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_44.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_46.png)

### 仅微调偏置项

![](img/dfcde63dc6d8a19ba4269d77313db151_68.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_70.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_71.png)

微调权重需要存储中间激活值，但微调偏置项不需要存储任何激活值。一个直观的方法是仅微调那些偏置项。例如，BERT-base有1.1亿参数，但只有0.1百万个偏置项。对于中小型数据集，仅微调偏置项能获得与全参数微调竞争性的结果。

### 轻量级残差学习

![](img/dfcde63dc6d8a19ba4269d77313db151_81.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_82.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_83.png)

这个方法由我们团队在LoRA之前提出。其核心思想是，给定沉重的主分支，我们添加一个非常轻量级的侧分支，只更新这个侧分支上的参数，我们称之为学习残差而非绝对变化。它包含分组卷积和1x1卷积，以降低分辨率并避免反向瓶颈结构，确保残差分支非常轻量。关键原则是保持激活大小较小。

![](img/dfcde63dc6d8a19ba4269d77313db151_85.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_87.png)

### 适配器

适配器方法在Transformer架构中插入若干层。适配器层通常采用瓶颈结构，包含两个前馈层：先将维度投影到较小的瓶颈维度，再投影回原始维度，以节省参数和计算量。每个任务只需训练少量参数。相比传统方法需要为N个下游任务存储N份模型权重，使用适配器可以极大节省存储空间。

![](img/dfcde63dc6d8a19ba4269d77313db151_89.png)

然而，适配器模块会增加模型深度，带来额外的推理延迟。

### 提示微调与前缀微调

提示微调的核心思想是：我们可以调整提示本身。我们为不同任务准备不同的“软提示”（连续向量），将其预置在原始输入提示之前。这样，不同任务可以使用同一个基础模型，通过批处理提高效率。当模型规模很大时，其准确率与全参数微调非常接近。

前缀微调是提示微调的扩展，它为每一层都添加可调的提示向量，通常能获得比仅在第一层调整提示更好的结果。

但提示和前缀微调会增加输入长度，可能增大KV缓存并导致更长的推理延迟。

### LoRA及其变种

LoRA家族的目标是在不引入实际推理延迟的情况下进行微调。其思想与轻量级残差学习类似：学习残差。

![](img/dfcde63dc6d8a19ba4269d77313db151_91.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_93.png)

**核心公式**：对于预训练权重矩阵 `W ∈ R^(d×k)`，LoRA将其更新表示为 `W + ΔW = W + BA`，其中 `B ∈ R^(d×r)`, `A ∈ R^(r×k)`，且秩 `r << min(d, k)`。在训练时，只训练 `A` 和 `B`，`W` 被冻结。在推理时，可以将 `BA` 加到 `W` 上，因此不引入额外延迟。

LoRA有两个前馈层作为一个并行分支：一个将维度从 `d` 投影到低秩 `r`，另一个从 `r` 投影回 `d`。矩阵 `B` 初始化为零，`A` 初始化为高斯分布，因此初始时侧分支效果为零。在训练过程中，`A` 和 `B` 被调整以学习新信息。

![](img/dfcde63dc6d8a19ba4269d77313db151_95.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_97.png)

LoRA有许多实际应用案例，例如在扩散模型中，可以下载各种LoRA分支来改变生成风格。

![](img/dfcde63dc6d8a19ba4269d77313db151_99.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_101.png)

### QLoRA

![](img/dfcde63dc6d8a19ba4269d77313db151_103.png)

QLoRA旨在进一步压缩模型内存占用，不仅微调一小部分参数，还将基础模型量化为4位。例如，一个70亿参数的模型原本需要14GB存储，量化后仅需3.5GB。同时，它仍然使用轻量级的LoRA适配器分支。这使得在中等或入门级GPU上微调大语言模型成为可能。

![](img/dfcde63dc6d8a19ba4269d77313db151_105.png)

### BiDelta：1比特微调

从LoRA可以看出，模型微调过程中存在大量冗余。BiDelta的目标是使用极低精度进行高效的大语言模型微调。直觉是，由于模型已经训练得很好，我们只添加很少的信息进行微调，因此这部分信息应该更容易压缩。

![](img/dfcde63dc6d8a19ba4269d77313db151_107.png)

我们的解决方案是将权重增量 `ΔW` 量化为仅1比特，同时微调一个每个张量共享的16位缩放因子 `α`。

![](img/dfcde63dc6d8a19ba4269d77313db151_109.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_110.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_111.png)

**核心公式**：对于每个任务微调后的模型，其权重可表示为：`W_finetuned = W_base + α * sign(ΔW)`，其中 `sign` 函数将 `ΔW` 量化为 `+1` 或 `-1`。缩放因子 `α` 初始化为 `ΔW` 的绝对值均值，随后可通过知识蒸馏进行学习。

![](img/dfcde63dc6d8a19ba4269d77313db151_113.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_115.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_116.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_118.png)

这种方法实现了高达10-15倍的压缩比，并且支持同时高效服务多个从同一基础模型微调而来的不同模型，非常适用于多租户服务场景。

## 多模态大语言模型

到目前为止，我们讨论的都是单模态。现在让我们进入多模态大语言模型的世界。我们将涵盖三个部分：基于交叉注意力的模型、将视觉标记视为输入的方法，以及使用VLM方法实现视觉输出。

### 基于交叉注意力的视觉语言模型

![](img/dfcde63dc6d8a19ba4269d77313db151_120.png)

我们希望模型能同时理解图像和语言。例如，给定一张图片，回答“这张图片有什么不寻常之处？”。

![](img/dfcde63dc6d8a19ba4269d77313db151_122.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_123.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_125.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_127.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_129.png)

如何实现？一种方法是使用交叉注意力。语言模型部分被冻结，我们通过交叉注意力层注入图像的新信息。处理流程是：图像通过视觉编码器转换为视觉标记，然后通过一个感知重采样器调整标记数量，得到固定数量的标记。接着，将这些标记作为键和值，与语言模型块中产生的查询进行交叉注意力计算。

![](img/dfcde63dc6d8a19ba4269d77313db151_131.png)

感知重采样器的作用是将可变长度的视觉标记转换为固定长度的输出。它使用一组可学习的查询向量，通过注意力机制聚合视觉信息，无论输入图像有多少个标记，输出都是固定数量的向量。

![](img/dfcde63dc6d8a19ba4269d77313db151_133.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_135.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_137.png)

这种基于交叉注意力的VLM具有出色的上下文学习能力，可以进行视觉对话，并展现出良好的世界知识和推理能力。

![](img/dfcde63dc6d8a19ba4269d77313db151_139.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_141.png)

### 将视觉标记视为输入

![](img/dfcde63dc6d8a19ba4269d77313db151_146.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_148.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_150.png)

另一种方法是对称地对待图像标记和语言标记，将所有模态都标记化后输入大语言模型。这种方法被称为“一切皆标记”风格。

![](img/dfcde63dc6d8a19ba4269d77313db151_152.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_154.png)

其优势在于拥有非常同质的模型架构：标记输入，标记输出，基础设施统一。例如，PaLM-E模型可以将语言、图像、视频、动作、音频等一切模态都转换为标记，输入大语言模型，并输出语言或控制信号等不同模态的标记。

![](img/dfcde63dc6d8a19ba4269d77313db151_156.png)

具体实现通常包含三个阶段：
1.  训练一个投影器，将视觉编码器输出的特征对齐到语言模型的空间。
2.  进行多模态预训练。
3.  进行监督微调。

这种方法支持多图像推理、上下文学习、思维链和更好的世界知识。

在VLM项目中的一些发现：
*   更新大语言模型参数可以获得更好的上下文学习能力。
*   交错的图像-文本数据比单纯的图像-文本对数据更重要。
*   数据混合至关重要：在指令微调时，混合纯文本数据可以减轻模型在纯文本任务上的性能下降，并提升视觉语言任务的准确率。
*   输入图像的分辨率比标记数量更重要，但更高的分辨率意味着更高的计算成本。

为了处理高分辨率图像，可以采用动态高分辨率策略，例如将图像分割为多个图块并辅以缩略图。对于文本和OCR任务，高分辨率帮助很大；对于知识和推理任务，一定数量的图块就足够了。

另一种支持高分辨率图像处理的方法是使用交叉注意力，例如CoAgent论文中的方法，它同时使用低分辨率和高分辨率图像编码器，并通过交叉注意力机制将高分辨率信息注入到视觉语言解码器中。

### 统一生成与理解

一个有趣的方向是统一图像生成和理解，包括图像和视频。我们希望同一个模型既能理解图像（如图像对比），也能生成图像。

这需要一个统一的视觉塔和离散的视觉标记。为了实现生成，我们需要像预测语言单词一样预测视觉标记，因此需要离散标记。同时，这些视觉标记需要与文本输入对齐，以便构建“标记进，标记出”的同质推理流程。

![](img/dfcde63dc6d8a19ba4269d77313db151_158.png)

统一视觉塔通过多种损失进行训练：
1.  **图像文本对比损失**：确保图像和文本标记在语义空间中对齐。
2.  **重建损失**：通过残差量化获得离散视觉标记，并经过解码器重建图像，确保生成质量。

![](img/dfcde63dc6d8a19ba4269d77313db151_160.png)

这种方法在图像理解和视频理解基准上取得了显著提升，并能生成高质量的图像和视频。

### 思维链提示

![](img/dfcde63dc6d8a19ba4269d77313db151_181.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_183.png)

思维链提示是一种非常有用的技巧。它要求模型在给出最终答案前，展示其推理的中间步骤。例如，在数学题中，不仅给出答案，还给出“首先...然后...因此...”的思考过程。这通常能显著提升复杂推理任务的准确性。一个简单的触发短语是“让我们一步一步地思考”。

### 检索增强生成

检索增强生成是一种非常流行的技术。我们不需要LLM在其参数中记住所有知识，而是可以将信息存储在外部数据库中。当有新提示时，我们从数据库中检索相关的信息片段，将其与原始提示拼接后输入语言模型，从而获得答案。

![](img/dfcde63dc6d8a19ba4269d77313db151_185.png)

这种方法的好处是知识可以随时更新，只需更新数据库，而无需重新训练庞大的语言模型。

RAG系统通常包含以下组件：
1.  **嵌入模型**：将文档和查询转换为向量。
2.  **检索器**：根据查询向量从向量数据库中查找最相关的文档。
3.  **排序器**：对检索到的文档进行相关性排序。
4.  **生成器**：将原始问题和检索到的相关文档一起输入大语言模型生成最终答案。

![](img/dfcde63dc6d8a19ba4269d77313db151_188.png)

![](img/dfcde63dc6d8a19ba4269d77313db151_189.png)

这对于构建需要最新知识或特定领域知识的应用非常有用。
