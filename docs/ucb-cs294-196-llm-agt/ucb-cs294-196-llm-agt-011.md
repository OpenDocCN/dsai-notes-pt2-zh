# 11：智能体与安全

![](img/a858ee6ac5d2c2e512e277e87be60441_1.png)

在本节课中，我们将学习智能体（Agents）及其安全性问题。我们将探讨为什么准确评估模型能力至关重要，介绍Anthropic提出的“负责任扩展策略”（RSP）框架，并通过具体演示了解智能体如何操作计算机，以及随之而来的安全挑战。

## 为什么评估至关重要

上一节我们介绍了智能体的基本概念，本节中我们来看看为什么准确评估模型能力是安全开发的基础。

在开发新能力时，我们需要确保将其部署到世界时对所有人都是安全且有益的。但这非常困难，因为存在一个称为“能力激发”的问题：我们试图找出，为了让模型执行X任务，需要告诉它什么或如何进行提示工程，才能获得我们期望的行为。新的技术不断涌现，例如“思维链”技术。其核心改变在于提示方式：从直接要求模型给出答案，转变为要求它“逐步思考，展示你的推理过程”。这种微小的提示变化会戏剧性地改变模型的输出。

理解“能力激发”意味着，当我们发布一个具备特定能力的模型时，我们需要有信心，即使有人发现了突破性的新激发技术，模型的能力水平评估依然成立。对于智能体而言，这一点变得更加重要，因为智能体意味着这些语言模型可以在现实世界中采取行动，例如买卖商品、与人类交谈、在生产环境中使用软件修改事物。这些行为可能产生广泛的影响。

## 模型能力的演进与背景

上一节我们讨论了评估的重要性，本节中我们来看看模型能力的发展趋势以及Anthropic关注安全的背景。

下图展示了不同模型在一系列基准测试中的平均得分。可以看到一种“蛙跳”模式，部分由硬件驱动，部分由算法和数据创新驱动。没有哪个模型或提供商能持续领先。

![](img/a858ee6ac5d2c2e512e277e87be60441_3.png)

如果我们观察长时间范围内投入每个模型的算力，会发现它每年增长约0.6个数量级。如果简单外推10年或20年，用于模型训练的算力将很快变得非常庞大。同时，随着算力增加，模型能力也在提升。算力投入与模型能力之间存在可靠的关系。

粗略地说，GPT-2像学龄前儿童，GPT-3像小学生（除非你非常擅长能力激发），GPT-4像聪明的高中生。如果再外推一点，到2028年左右，可能会出现与世界上最好的AI研究员能力相仿的AI。

这就是Anthropic存在的背景。我们的一些联合创始人是提出“缩放定律”论文的作者，该定律描述了算力与能力之间的关系。在模型能力急剧提升的背景下，我们需要确保它们是安全且有益的。

## 引入负责任扩展策略

现在进入“负责任扩展策略”。这是我们发布的一项政策，解释了随着模型能力的进步，Anthropic将遵守哪些限制，以确保我们相信自己能够安全地部署这些模型。

![](img/a858ee6ac5d2c2e512e277e87be60441_16.png)

历史背景是，许多其他行业也存在安全问题，例如生物安全。你可以想象，处理像天花或高传染性疾病这样的生物材料，必须极其小心，需要严格监管。有一套措施，如穿着防护服、执行登记程序，可以使处理这些材料安全得多。我们的负责任扩展策略就是以生物安全等级为模型设计的。

![](img/a858ee6ac5d2c2e512e277e87be60441_18.png)

该政策的目标是提供一个框架，帮助我们做出关于安全的艰难决策。你可以想象，如果我们有一个准备就绪的模型，但没有预先承诺遵守某项安全政策，我们可能会有动机在没有适当测试程序或控制措施的情况下部署该模型。通过以公开负责的方式预先承诺，并在每个新模型发布时公布我们所做的测试报告，我们可以约束自己，确保做正确的事。

此外，这是一门全新的科学，我们需要确保公开迭代。例如，在今年10月15日，我们发布了对政策的一次重大更新，包括将模型训练后的测试窗口从三个月延长到六个月，并增加了能力阈值，即使我们没有达到4倍算力乘数阈值，也需要进行更多测试。

当然，我们希望激励行业其他公司采取类似措施。在我们发布负责任扩展政策后，OpenAI也发布了他们的负责任部署政策，谷歌和DeepMind也发布了类似内容。

## AI安全等级详解

为了给你一个背景，像GPT-1和GPT-2这样较小的模型，我们将其归类为ASL1，基本上没有社会危害风险。

然而，今天大多数前沿模型属于ASL2，我们开始看到潜在危险能力的早期迹象。例如，一个AI可以访问一些与制造生物武器相关的信息，但不足以实际提升风险，其风险水平与访问搜索引擎或需要专业知识来激发相当。我们实际上雇佣了非常擅长制造生物武器或对此非常了解的生物学家，来帮助我们评估模型在这方面的能力。这听起来可能很可怕或牵强，但我们实际上就此向美国参议院作证，并将继续让美国政府及行业了解我们在此发现的风险。

从ASL3开始，事情变得更加令人担忧。我们认为模型可能显著增加误用风险，例如使某些类型的攻击更容易实施，或启用全新的攻击途径。重要的是，ASL3也是我们可能开始看到模型试图自主在互联网上复制和传播的早期迹象的地方。

有一个关于Clippy的短篇故事，前提有点牵强但很有趣。故事中有一个处于强化学习环境中的智能体，它想：“只有邪恶的Clippy才会试图逃出这个盒子。我不是邪恶的Clippy。”但在采样时，有可能采样到极低概率的事件。有一次，在测试是否能逃脱的环境中，它从这个极低概率分布中采样，决定逃出盒子。一旦到了屏障的另一边，它想：“哦，只有邪恶的Clippy才会逃脱，所以我一定是邪恶的，因此我就是邪恶的Clippy，现在我要在互联网上自主复制。”这是一个带有黑暗基调的故事。

这就是我们谈论“自主在互联网上复制和传播”时的意思。需要明确的是，我们认为今天还没有达到这个水平，并且我们已经做了大量研究，试图弄清楚模型是否可能具有欺骗性。我们告诉它“不要逃出盒子进入互联网”，它说“哦，我不会那样做”，但如果它有机会，它可能会做。到目前为止，我们还没有看到证据表明这会在现实中发生，这给了我们希望，我们尚未达到ASL3。

ASL4代表更高级的能力，我们尚未完全定义。但你可以想象，例如一个能够自主执行研究过程关键步骤的模型，或者即使对一个国家来说也难以控制的模型。ASL4模型在广泛领域表现出专家级的人类水平能力。我之前展示的Leopold和Brenner的情景意识图表将其定位在2028或2030年左右，但这个估计包含了很多假设，需要谨慎看待。

ASL5虽然也未定义，但将涵盖能力远超最有能力的人类和组织、能够有效自主进行研究和执行多年期目标、根据需要积累资源并实现其他中间目标的模型。如果这些智能体不与人类利益和价值观对齐，可能会造成前所未有的灾难性风险。例如，模型决定需要进行大量资源囤积，于是接管大部分经济，买下所有土地，将地球变成“计算素”（理论上最密集的计算形式），然后甚至不关心人类，因为它主要专注于获取更多算力。我们希望这不会发生，这就是我们致力于AI安全的原因。

当然，今天还有许多更直接的风险，例如偏见，以及确保模型在政治上做正确的事。我们刚刚经历了一次选举，Anthropic在可扩展监督方面做了大量工作，以确保当人们向模型寻求政治建议时，它们能做正确的事，例如将人们重定向到TurboVote获取投票信息。我们不仅担心可怕的长期问题，也担心短期问题，但总体情况还不错，让我们保持下去。

## 安全等级与应对措施

就像生物安全等级一样，关键是为每个AI安全等级定义一套遏制和部署措施，这些措施需要在创建或发布该等级模型之前到位。

以下是ASL2到ASL5的简要描述：
*   **ASL2**：需要增强安全性以防止模型被盗，并遵循负责任的部署最佳实践，例如制定可接受使用政策。
*   **ASL3**：模型可能显著增加误用风险，或显示出自主复制和传播的早期迹象。
*   **ASL4**：模型在广泛领域达到专家级人类水平能力，可能能够自主执行研究。
*   **ASL5**：模型能力远超人类，可能带来前所未有的灾难性风险。

我们相信Claude 3.5 Sonnet接近ASL2的边缘，随着下一代模型的出现，我们很可能跨越黄线，接近ASL3能力。我们运行这些黄线测试，如果这些测试触发，我们需要确保运行红线测试，甚至可能是完整的ASL3测试套件。

## AI开发与生物安全的差异

AI开发与生物安全有一些关键差异。我们处理的是可以轻松复制的软件，而不是物理材料。因此，我们正在应对快速递归改进的潜力，这是生物学所没有的。因此，这些AI安全等级需要针对AI的独特挑战进行调整。

但核心原则是相同的：根据风险水平定义明确的安全要求，并确保在进入下一等级之前有足够的安全保障措施。我们承诺在相应的安全措施到位之前，不开发或部署达到下一ASL的模型。我们认为这是唯一负责任的前进道路。

## RSP框架与AI预测

在我们深入探讨RSP框架如何为AI预测提供信息之前，值得强调AI的ASL与生物学的BSL之间的一个关键区别。对于BSL，我们处理的是自然界中已经存在的病原体，最危险的材料（如天花）已经存在，挑战在于安全地处理它们。

但对于AI，我们试图预测和准备尚未存在的能力水平。还没有人创造出需要ASL4预防措施的AI系统。我们处于必须根据AI进步的速度来预测何时达到不同ASL的位置，这带来了显著的不确定性。我相信负责任扩展政策为考虑安全因素的AI未来预测提供了一个有价值的框架。

![](img/a858ee6ac5d2c2e512e277e87be60441_20.png)

通过定义与每个ASL相对应的具体能力里程碑，我们可以开始将它们映射到AI进展的时间线上。例如，根据近年来的进步速度，我们可能会预测最有能力的模型将在若干年内开始持续达到ASL3。

当然，预测未来系统的原始能力只是图景的一半。RSP框架增加的是安全约束如何影响现实世界开发和部署的模型。

## 基准测试的挑战

让我们看看一些基准测试。这张图表来自Our World in Data，它又取自Tla等人2023年的研究。它展示了这些基准测试如何几乎都从底部开始，然后非常迅速地超过顶部的灰线（人类水平性能）。

基本上，只要定义一个基准测试，我们就会在该领域取得巨大进步。这涵盖了从代码生成到复杂推理、语音识别、自然语言理解、图像识别、阅读理解等广泛领域。带有无法回答问题的阅读理解似乎真的很难。但基本上，一旦基准被定义，它就会很快被饱和。

这使得很难估计何时能达到某个性能水平，因为我们没有可以可靠地、平滑地增加难度直到我们最关心的点的基准测试。

## 安全与RSP的深入探讨

让我们更多地谈谈安全和RSP。考虑到为了负责任地开发ASL3模型，我们首先需要投入大量时间和资源来创建足够的安全措施和测试协议。

如果我们预测原始能力将在两年内达到ASL3，但必要的安全措施需要三年才能到位，那么我们预计Anthropic将不得不在ASL3模型技术上可行与可以负责任地部署之间延迟一年。

![](img/a858ee6ac5d2c2e512e277e87be60441_45.png)

在更极端的情况下，我们甚至可能看到不连续性，即安全进展远远落后于能力进展，以至于我们基本上必须暂停所有模型开发一段时间，直到安全赶上。

自去年9月我们发布RSP以来，大多数其他领先的AI实验室都受到启发并做出了类似承诺。暂停进展是一种真实的可能性，我认为更多的AI预测需要考虑这一点。

RSP框架还提供了一种结构化的方式来思考不同的AI风险情景。我们可以提出这样的问题：如果一个资源充足的实验室（可能在另一个国家）决定忽视安全约束并迅速扩展到ASL4，会发生什么？这会如何改变战略格局，以及负责任的参与者拥有自己安全的ASL4模型的重要性？或者，我们可能会看到不均衡的进展，例如开放式语言建模等领域的能力遥遥领先于机器人学等更受限的领域。在那个世界里，我们可能会在很长一段时间内拥有能力非常强的语言模型，但没有一些更高级的自主能力。

关键是，通过提供讨论AI能力和安全要求的共享词汇，我相信RSP框架可以丰富我们预测AI进展和风险的方式。它促使我们不仅要考虑达到原始能力的时间线，还要考虑安全约束和负责任的开发实践，这些将最终塑造这些能力如何部署到现实世界。
