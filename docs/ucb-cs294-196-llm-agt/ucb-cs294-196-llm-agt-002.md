## 什么是智能体？

![](img/575d48e69f0fc2ef52c317c2fde13179_8.png)

首先，我们需要理解什么是智能体。在人工智能领域，智能体是一个能够与环境交互的智能系统。环境可以是物理世界（如机器人、自动驾驶汽车），也可以是数字世界（如视频游戏、操作系统），甚至可以是人类（如聊天机器人）。

![](img/575d48e69f0fc2ef52c317c2fde13179_10.png)

**智能体** = 一个能够感知环境并采取行动以实现目标的系统。

![](img/575d48e69f0fc2ef52c317c2fde13179_12.png)

智能体的“智能”定义会随着时间和技术发展而变化。几十年前，一个基于简单规则的聊天机器人可能就被认为是智能的，而今天，即使是像ChatGPT这样的模型，其智能程度也已被视为常态。

![](img/575d48e69f0fc2ef52c317c2fde13179_14.png)

## 什么是大语言模型智能体？

![](img/575d48e69f0fc2ef52c317c2fde13179_16.png)

在定义了智能体和语言模型之后，我们可以将大语言模型智能体分为三个层次：

1.  **文本智能体**：智能体与环境的交互（观察和行动）完全通过文本来进行。
2.  **大语言模型智能体**：使用大语言模型来决策和行动的文本智能体。
3.  **推理智能体**：不仅使用大语言模型来行动，还利用它进行内部思考（推理）的智能体。这是当前最前沿的范式，我们将在后面详细探讨。

![](img/575d48e69f0fc2ef52c317c2fde13179_18.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_20.png)

上一节我们定义了智能体的基本概念，本节中我们来看看在大语言模型出现之前，人们是如何构建文本智能体的。

![](img/575d48e69f0fc2ef52c317c2fde13179_22.png)

## 大语言模型之前的文本智能体

![](img/575d48e69f0fc2ef52c317c2fde13179_24.png)

在语言模型兴起之前，主要有两种构建文本智能体的范式：

![](img/575d48e69f0fc2ef52c317c2fde13179_26.png)

**1. 基于规则的智能体（符号AI）**
这种方法在AI早期（如1960年代）就已出现，例如著名的聊天机器人ELIZA。其核心思想是编写大量的“如果-那么”规则来模拟对话。

```
if user_input contains “mother”:
    response = “Tell me more about your family.”
```

![](img/575d48e69f0fc2ef52c317c2fde13179_28.png)

以下是基于规则方法的局限性：
*   **任务特定**：每条规则都针对特定场景，难以泛化。
*   **扩展性差**：每个新领域（如从聊天转到游戏）都需要重新编写大量规则。
*   **能力有限**：无法处理规则未覆盖的复杂或未知情况。

**2. 基于强化学习的智能体**
在文本游戏等环境中，可以将观察（文本描述）、行动（文本命令）和奖励（任务完成度）建模为一个强化学习问题。智能体通过大量试错来学习最大化奖励的策略。

![](img/575d48e69f0fc2ef52c317c2fde13179_30.png)

```
# 伪代码：强化学习智能体在文本环境中学习
state = env.get_text_observation()
action = policy_network(state) # 例如“往北走”
reward, next_state = env.step(action)
```

![](img/575d48e69f0fc2ef52c317c2fde13179_32.png)

以下是基于强化学习方法的局限性：
*   **需要奖励信号**：许多现实任务难以定义明确的奖励函数。
*   **训练成本高**：每个新任务都需要从头开始进行大量训练。
*   **泛化能力弱**：在一个游戏中学到的策略很难直接迁移到另一个游戏。

基于规则和强化学习的方法都受限于特定领域。接下来，我们将看到大语言模型如何以其强大的泛化能力和上下文学习特性，为构建通用智能体带来革命性的希望。

![](img/575d48e69f0fc2ef52c317c2fde13179_34.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_36.png)

## 大语言模型智能体的发展脉络

![](img/575d48e69f0fc2ef52c317c2fde13179_38.png)

大语言模型的出现改变了游戏规则。它们在海量文本上进行了预训练，但在推理时可以通过提示（Prompting）来解决各种新任务。这种泛化能力是构建通用智能体的关键。

![](img/575d48e69f0fc2ef52c317c2fde13179_40.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_42.png)

智能体的发展大致遵循以下路径：
1.  **大语言模型出现**（如GPT-3），人们开始探索其在各种任务上的应用。
2.  **任务分化**：一些任务侧重于**推理**（如数学问答），另一些任务侧重于**行动**（如游戏、机器人控制）。
3.  **范式融合**：推理和行动的范式开始结合，催生了**推理智能体**。
4.  **拓展与深化**：一方面，智能体被应用于更复杂的领域（如网页交互、软件工程、科学发现）；另一方面，研究方法也在不断进化（如记忆、学习、规划、多智能体协作）。

![](img/575d48e69f0fc2ef52c317c2fde13179_44.png)

为了理解这种融合，让我们先深入探讨“推理”和“行动”这两个核心范式。

## 从问答任务看范式演进

![](img/575d48e69f0fc2ef52c317c2fde13179_46.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_48.png)

问答任务直观地展示了智能体面临的挑战和解决方案的演进。

**挑战**：直接向语言模型提问可能无法获得正确答案，原因包括：
*   **需要复杂推理**：例如，“如果我有7万亿美元，能否买下苹果、英伟达和微软？”需要查询实时股价并计算。
*   **知识过时**：语言模型训练数据截止日期后的信息（如新上任的国家领导人）无法知晓。
*   **计算能力不足**：语言模型不擅长精确的数学运算。

**解决方案的演进**：
1.  **程序生成**：让语言模型生成代码（如Python程序）来执行计算。
    ```python
    # 语言模型生成类似代码
    result = prime_factorization(123456789)
    ```
2.  **检索增强生成**：从外部知识库（如维基百科）检索相关信息，提供给语言模型以生成答案。
3.  **工具调用**：让语言模型学习调用特殊工具（如计算器、搜索API）的格式。
    ```
    思考：我需要计算市场总值。
    行动：搜索[苹果公司当前市值]
    ```
然而，早期的方案往往是分散的、针对特定任务的。我们需要一个更统一、更通用的抽象。

## 推理与行动的融合：ReAct范式

![](img/575d48e69f0fc2ef52c317c2fde13179_50.png)

ReAct范式是智能体发展史上的一个关键突破。它巧妙地将**推理**和**行动**交织在一起。

![](img/575d48e69f0fc2ef52c317c2fde13179_52.png)

*   **推理**：指智能体的内部思考过程，例如规划步骤、分析情况。这通过**思维链**技术实现。
*   **行动**：指智能体与外部环境的交互，例如调用工具、执行命令。这通过**工具调用**等技术实现。

![](img/575d48e69f0fc2ef52c317c2fde13179_54.png)

ReAct的核心思想是：让智能体交替进行**思考**和**行动**。思考指导下一步该做什么行动，而行动的结果又为下一步思考提供新的信息。

![](img/575d48e69f0fc2ef52c317c2fde13179_56.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_58.png)

以下是ReAct的工作流程：
1.  **提示**：给语言模型一个示例，展示如何通过“思考-行动-观察”的循环解决一个任务。
2.  **循环执行**：
    *   **思考**：模型分析当前情况，决定下一步。
    *   **行动**：模型执行一个动作（如搜索）。
    *   **观察**：环境返回结果（如搜索结果）。
    *   将“思考-行动-观察”三元组追加到对话上下文中。
3.  **重复**：直到任务完成（如输出最终答案）。

**示例**：回答“7万亿美元能否买下苹果、英伟达和微软？”
```
思考：我需要找到这三家公司的当前市值，然后加总。
行动：搜索[苹果 英伟达 微软 当前市值]
观察：苹果：2.8T；英伟达：1.1T；微软：3.0T。
思考：总市值为2.8+1.1+3.0=6.9万亿美元。
思考：6.9T < 7T，因此可以买下。
行动：完成[答案：可以]
```
ReAct的优势在于其通用性和鲁棒性。思考帮助智能体在行动受挫时调整计划，而行动则为思考提供实时信息和计算能力。这种模式可以轻松推广到网页浏览、游戏、机器人控制等众多领域。

## 推理智能体的本质：无限的语言行动空间

推理智能体与以往所有智能体有一个根本性区别：它拥有一个名为“推理”的**内部行动空间**。

*   **传统智能体**：行动空间由环境严格定义（如游戏中的上下左右）。行动会直接改变环境状态。
*   **推理智能体**：在环境行动之外，增加了一个“思考”行动。思考的内容是**任意的语言**，它可以是一个词、一句话或一整篇文章。思考**不会直接改变外部世界**，但会改变智能体自身的内部上下文（记忆），从而影响其未来的决策。

![](img/575d48e69f0fc2ef52c317c2fde13179_60.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_62.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_64.png)

这个**无限的语言行动空间**赋予了推理智能体前所未有的灵活性和复杂性，使其能够进行长期规划、自我反思和适应异常情况。

我们已经了解了推理智能体的核心范式。接下来，我们将探讨如何让智能体拥有“记忆”，从而能够从经验中学习并持续进步。

## 长期记忆：从金鱼到学者

![](img/575d48e69f0fc2ef52c317c2fde13179_66.png)

标准的语言模型智能体有一个关键限制：其上下文窗口是**短期记忆**。一旦任务结束或上下文被清除，所有经验都会丢失。就像一个只有三秒记忆的金鱼，即使解决了一个难题，下次遇到时还得从头开始。

![](img/575d48e69f0fc2ef52c317c2fde13179_68.png)

**长期记忆**旨在解决这个问题，它允许智能体存储重要经验、知识或技能，并在未来任务中检索和使用。这是一种新型的“学习”方式。

![](img/575d48e69f0fc2ef52c317c2fde13179_70.png)

**示例：Reflexion（反思）**
在编程任务中，智能体可以：
1.  编写代码并运行测试。
2.  如果测试失败，**反思**错误原因（例如，“我忘记处理空输入的情况”）。
3.  将这次反思作为一个知识条目存入**长期记忆**。
4.  当再次遇到类似任务时，检索这条记忆，从而避免犯同样的错误。

![](img/575d48e69f0fc2ef52c317c2fde13179_72.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_74.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_75.png)

这与传统的强化学习不同：
*   **学习信号**：不是标量奖励，而是丰富的文本反馈（错误信息、测试结果）。
*   **学习方式**：不是通过梯度下降更新网络权重，而是通过更新语言形式的记忆（文本）来改变未来行为。

![](img/575d48e69f0fc2ef52c317c2fde13179_77.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_78.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_80.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_81.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_83.png)

长期记忆可以有多种形式：
*   **技能库**：存储可复用的代码片段或操作流程（如“如何在Minecraft中造剑”）。
*   **事件日志**：记录智能体经历的所有事件，构成一部详细的“日记”。
*   **语义记忆**：从事件中总结出的抽象知识和结论（如“张三是个好奇的人”）。

![](img/575d48e69f0fc2ef52c317c2fde13179_85.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_87.png)

如果将语言模型本身的参数也视为一种“固化”的长期记忆，那么我们就得到了一个统一的智能体抽象：它由**决策机制**、**短期记忆**和**长期记忆**三部分组成。这个框架足以表达几乎所有类型的智能体。

![](img/575d48e69f0fc2ef52c317c2fde13179_89.png)

## 历史视角：语言智能体为何不同？

从更广阔的AI历史来看，智能体范式经历了演变：
1.  **符号AI智能体**：基于手工编写的逻辑规则。
2.  **深度强化学习智能体**：基于神经网络从奖励信号中学习。
3.  **大语言模型智能体**：基于语言进行推理和决策。

![](img/575d48e69f0fc2ef52c317c2fde13179_91.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_93.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_95.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_97.png)

三者的核心区别在于**如何将观察映射到行动**：
*   **符号AI**：将观察映射到**符号状态**，再用逻辑规则推导行动。
*   **深度RL**：将观察映射到**神经向量/表示**，再用网络推导行动。
*   **语言智能体**：将观察映射到**语言**，在语言空间中进行思考，再推导行动。

语言作为中间表示的优势：
*   **便捷性**：利用预训练模型的丰富先验知识，通过提示即可快速构建。
*   **通用性**：同样的“思考”能力可应用于驾驶、游戏、投资等完全不同领域。
*   **可扩展性**：思考空间是无限的，不同于固定维度的符号状态或神经向量。

## 新应用：数字自动化与超越

语言智能体催生了一类全新的应用：**数字自动化**。它旨在让智能体像人类助手一样，处理现实世界中繁琐或复杂的数字任务，例如：
*   填写报销报告。
*   编写、调试代码。
*   查找、总结学术论文。
*   处理客户服务请求。

![](img/575d48e69f0fc2ef52c317c2fde13179_99.png)

为了实现这些应用，我们需要比传统学术基准更**真实、复杂、可扩展**的任务环境。例如：
*   **WebShop**：一个模拟在线购物的环境，智能体需要根据文字指令浏览网页、搜索商品、比较信息并完成购买。
*   **SWE-bench**：一个软件工程环境，智能体需要根据GitHub仓库中的问题描述，阅读代码、编写补丁并解决问题。
*   **科学发现**：智能体可以访问科学数据库和工具，提出假设、设计实验，甚至指导现实世界的实验合成。

这些任务要求智能体理解真实世界的语言、进行长程规划、处理不确定性并与环境（包括人）进行多轮交互。

![](img/575d48e69f0fc2ef52c317c2fde13179_101.png)

## 未来方向与挑战

![](img/575d48e69f0fc2ef52c317c2fde13179_103.png)

![](img/575d48e69f0fc2ef52c317c2fde13179_105.png)

大语言模型智能体领域方兴未艾，有许多激动人心的研究方向：

1.  **训练**：如何为智能体任务专门训练模型？目前模型是在通用文本上预训练的，与智能体所需的决策、推理数据存在差异。一种思路是使用智能体自身产生的“思考-行动”轨迹数据来微调模型，形成模型与智能体能力协同进化的良性循环。

![](img/575d48e69f0fc2ef52c317c2fde13179_107.png)

2.  **界面**：如何为智能体设计更好的人机交互界面？现有的界面（如命令行、图形界面）是为人类设计的，未必适合语言模型。为智能体优化界面（例如，一次性返回多条搜索结果供其分析）可以显著提升其任务表现。

3.  **鲁棒性**：如何确保智能体在现实世界中可靠工作？许多实际应用（如客服）要求接近100%的可靠性，而不是学术中常用的“N次尝试中成功1次”。这需要新的评估指标和训练方法，让智能体行为更稳定、可预测。

![](img/575d48e69f0fc2ef52c317c2fde13179_109.png)

4.  **人机协作**：如何让智能体与人类高效协作？这需要智能体能够理解人类意图、接受反馈、解释自身决策，并在多轮交互中共同完成任务。

5.  **基准测试**：如何构建更能反映现实挑战的评测基准？新的基准需要包含更复杂的任务、模拟的人类交互以及强调鲁棒性的评估标准。

*   语言智能体催生的新应用领域——**数字自动化**，以及相关的复杂任务环境。
*   该领域未来在**训练**、**界面**、**鲁棒性**、**人机协作**和**基准测试**等方面面临的挑战与机遇。
