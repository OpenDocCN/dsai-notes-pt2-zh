# 7：企业环境中的智能体应用 🏢

在本节课中，我们将探讨大语言模型智能体在企业环境中的应用。我们将了解智能体如何帮助员工完成实际工作，分析其适用场景、用户角色以及部署环境。课程内容将涵盖API智能体、Web智能体以及它们在未来工作场所中的变革潜力。

## 背景与定义 🤖

在之前的课程中，大家已经学习了智能体系统的基础构建模块，例如推理和工具使用。本节中，我们将明确本课程讨论的智能体类型，并回顾相关术语。

我们讨论的是**大语言模型驱动的智能体**。这类智能体具有一定程度的自主性：它们能够规划、在环境中执行动作、接收环境反馈，并执行人类用户设定的目标。这个过程通常是多次迭代的。

![](img/9ddc547a45ac114d3402048737228dcd_3.png)

需要明确区分的是，我们讨论的并非早期基于强化学习训练的、范围较窄的智能体（例如擅长玩游戏的智能体）。大语言模型智能体的核心优势在于，其底层模型在互联网海量数据上进行了训练，因此“见过”大量关于日常软件（如PowerPoint、Excel乃至Salesforce等商业软件）的使用说明和文档。智能体的目标就是利用这些知识来执行人们想要完成的任务。

在大语言模型智能体内部，我们可以进一步区分两种类型：
*   **API调用智能体**：拥有一组可以通过某种形式化规范（如API端点描述）来定义的工具集，主要交换文本信息。
*   **Web智能体**：能够像人类一样与网页交互，通过点击按钮、填写表单、页面跳转来收集和复用信息以完成任务。

## 企业工作流与自动化机遇 ⚙️

上一节我们定义了智能体，本节中我们来看看它们能在企业环境中解决什么问题。企业中存在大量需要人工处理的、重复性高但价值相对较低的任务。

![](img/9ddc547a45ac114d3402048737228dcd_9.png)

以员工John无法访问公司内网知识库文章为例。他会向IT服务台提交工单，该工单被分配给Sandy处理。Sandy需要手动检查现有知识库、查找类似历史事件、调整访问权限，最后解决案例并记录。即使有生成式AI帮助总结，整个过程仍包含许多手动步骤。这类日常事务在全球企业中每天重复数百万次。

![](img/9ddc547a45ac114d3402048737228dcd_11.png)

企业自动化的发展经历了从低度自动化（如手动脚本、机器人流程自动化）到需要更高智能度的演进。如今，**智能体工作流** 最具智能性，也最有自动化潜力。

![](img/9ddc547a45ac114d3402048737228dcd_13.png)

![](img/9ddc547a45ac114d3402048737228dcd_15.png)

智能体的机遇在于自动化那些**量大但价值相对较低、且难以用传统技术脚本化的“细沙”任务**。例如：安排会议、更新客户关系管理系统、回复邮件等。这些任务每次发生的方式可能都略有不同，难以用固定流程自动化，但智能体有潜力在此大放异彩。

![](img/9ddc547a45ac114d3402048737228dcd_17.png)

![](img/9ddc547a45ac114d3402048737228dcd_20.png)

这个例子展示了我们希望与智能体进行的交互：它们能够询问澄清性问题，并具备一定的自主规划和重新规划能力。

## API智能体与TAPE框架 🛠️

![](img/9ddc547a45ac114d3402048737228dcd_32.png)

接下来，我们深入探讨API智能体。典型的智能体架构包含以下核心组件：一个能发出动作的LLM智能体、接收动作并返回观察结果的环境、用于存储过去动作和响应的短期记忆、各种规划策略（如ReAct），以及一系列工具。

![](img/9ddc547a45ac114d3402048737228dcd_34.png)

然而，要高效构建智能体，我们需要合适的软件框架。为此，我们介绍一个新开源框架：**TAPE Agents**。它旨在为智能体的全周期开发提供支持，既包括构建和调试工具，也包括基于执行轨迹进行优化的工具。

![](img/9ddc547a45ac114d3402048737228dcd_36.png)

TAPE框架试图融合两方面的优势：
1.  **软件工程框架**：如LangChain、LlamaIndex，用于精细地指定智能体、提供预制组件、并发和流式处理等功能。
2.  **优化框架**：如DSPy，专注于提示优化和模型微调。

![](img/9ddc547a45ac114d3402048737228dcd_38.png)

![](img/9ddc547a45ac114d3402048737228dcd_40.png)

TAPE的核心是一个称为 **`tape`** 的统一抽象。它本质上是一个日志，记录了系统中所有智能体的所有“思考”和“动作”，以及环境调用的结果。关键之处在于，`tape`本身是一个结构化的数据对象，可以被其他智能体或下游算法处理，用于调试、优化或提炼知识。

![](img/9ddc547a45ac114d3402048737228dcd_42.png)

执行模型如下：智能体读取`tape`的完整历史，选择其内部某个“节点”执行。节点负责调用LLM，LLM产生的思考或动作被添加到`tape`中。一个协调器会调用环境来执行未处理的动作，并将结果反馈回循环。这个模型支持多智能体和子智能体委托，非常模块化。

![](img/9ddc547a45ac114d3402048737228dcd_44.png)

通过将`tape`视为数据，我们可以实现：
*   完整的审计和调试。
*   进行提示优化循环。
*   将复杂“教师”智能体的工作提炼出来，训练出更经济高效的“学生”智能体。

![](img/9ddc547a45ac114d3402048737228dcd_46.png)

![](img/9ddc547a45ac114d3402048737228dcd_47.png)

![](img/9ddc547a45ac114d3402048737228dcd_48.png)

![](img/9ddc547a45ac114d3402048737228dcd_49.png)

我们进行了一项案例研究，目标是构建一个能帮助用户填写复杂工作请求表单的高质量对话智能体。我们定义了衡量智能体质量的 **`GREAT`** 属性集（Grounded, Responsive, Accurate, Disciplined, Transparent, Helpful）。

![](img/9ddc547a45ac114d3402048737228dcd_51.png)

方法如下：
1.  使用参数规模巨大、成本高昂的LLM（如Claude 3.5）作为“教师”智能体，生成高质量对话数据，并记录为`tape`。
2.  利用这些`tape`数据，对一个参数规模小得多的模型（如Llama 8B）进行微调。

![](img/9ddc547a45ac114d3402048737228dcd_53.png)

结果非常显著：微调后的8B参数模型达到了与庞大“教师”模型相当的性能水平，但**每百万次对话轮次的成本降低了300多倍**。这展示了TAPE框架在构建高性能、低成本智能体方面的强大潜力。

![](img/9ddc547a45ac114d3402048737228dcd_55.png)

## Web智能体的构建与挑战 🕸️

![](img/9ddc547a45ac114d3402048737228dcd_59.png)

现在，我们将焦点转向Web智能体。当没有现成API可用时，Web智能体承诺通过直接与用户界面交互来提供更通用的自动化能力。

一个基础的Web智能体可以这样构建：
1.  **提示设计**：将任务目标、当前网页的HTML内容（或可访问性树）、允许的动作空间（如点击、填写）组合成一个提示。
2.  **LLM调用**：将提示提交给LLM，请求其输出下一步要执行的动作。
3.  **动作执行**：使用浏览器自动化库（如Playwright、Selenium）来执行LLM输出的动作。

![](img/9ddc547a45ac114d3402048737228dcd_61.png)

例如，一个基于GPT-4和ReAct风格提示的简单智能体，已经可以自动填写费用报告等任务。这令人兴奋，因为它展现了下一代用户界面的雏形。

![](img/9ddc547a45ac114d3402048737228dcd_63.png)

然而，现实是这种基础方法非常脆弱。要投入实际应用，我们需要它足够**稳健、可靠、快速**。目前，智能体速度较慢（视频常被加速播放），且容易受到网页注入攻击。

为了系统评估和推进Web智能体，社区构建了许多基准测试：
*   **基于轨迹的基准**：如MiniWoB、WebLINX，通过比较智能体动作与人类标注的“黄金轨迹”来评估。但可能无法涵盖替代解决方案，且存在被LLM记忆的风险。
*   **实时基准**：如WebArena、VisualWebArena，通过检查最终状态（如数据库状态、问题答案）来评估，允许替代解决方案，更接近真实场景。

![](img/9ddc547a45ac114d3402048737228dcd_65.png)

![](img/9ddc547a45ac114d3402048737228dcd_67.png)

我们提出了 **WorkArena** 基准。它包含600个在ServiceNow产品上执行的工作相关任务，从简单的列表排序到复杂的多步骤工作流（如根据仪表盘信息订购库存最低的物品）。评估通过远程开发者实例进行，前端交互，后端API验证结果。

![](img/9ddc547a45ac114d3402048737228dcd_69.png)

评估结果揭示了当前Web智能体的局限性：在简单任务（L1）上，最佳智能体成功率约42.7%；在复杂的现实工作流（L2, L3）上，成功率急剧下降甚至为零，而人类在这些任务上的平均成功率高达94%。失败原因包括：规划失败、幻觉出不存在控件、动作语法错误等。

![](img/9ddc547a45ac114d3402048737228dcd_71.png)

![](img/9ddc547a45ac114d3402048737228dcd_73.png)

为了统一评估和开发，我们创建了 **BrowserGym**（统一评估平台）和 **AgentLab**（智能体开发工具箱）：
*   **BrowserGym**：提供标准化的观察空间（HTML、可访问性树、截图等）和动作空间，集成了多个主流基准，并支持人工评估。
*   **AgentLab**：提供构建模块、调试工具（如Agent X-Ray可视化实验细节）、大规模实验运行和可复现性工具。

![](img/9ddc547a45ac114d3402048737228dcd_75.png)

![](img/9ddc547a45ac114d3402048737228dcd_76.png)

![](img/9ddc547a45ac114d3402048737228dcd_77.png)

![](img/9ddc547a45ac114d3402048737228dcd_78.png)

![](img/9ddc547a45ac114d3402048737228dcd_79.png)

通过BrowserGym和AgentLab，我们有望实现分布式轨迹收集，利用大量智能体在开放网络上的交互轨迹来进一步微调和改进智能体。

![](img/9ddc547a45ac114d3402048737228dcd_81.png)

Web智能体面临的主要挑战包括：
*   **长上下文理解**：网页内容庞大，相关信息稀疏。
*   **长期规划**：在当前页面难以预见几步之后的状态。
*   **学习与适应性**：需要从观察、演示和错误中学习。
*   **多模态需求**：许多任务需要视觉理解。
*   **成本与效率**：需要加快交互速度，减小上下文。
*   **安全性**：必须对网页上的各种注入攻击具有鲁棒性，这是一个随着多模态模型发展而日益严峻的挑战。

![](img/9ddc547a45ac114d3402048737228dcd_83.png)

## 智能体在工作场所的变革潜力 🚀

最后，让我们退一步，展望智能体技术对工作场所更广泛的影响。毫无疑问，AI智能体将彻底改变工作的性质。

回顾Sandy的例子，在智能体普及的未来，工作流程将变为：
1.  工单提交后，**AI智能体**会先尝试分析并制定解决方案步骤。
2.  **协调器智能体**将计划提交给Sandy审批。
3.  Sandy批准后，**一系列专门的AI智能体**（如用户权限管理智能体）会并行执行各个子任务，并在关键操作前再次请求Sandy确认。
4.  最终自动解决并关闭工单。

这种“人机团队”模式将极大提升效率。研究表明，人机协作主要有两种模式：
*   **百夫长模式**：人类进行高层战略分解，将大块工作委托给AI系统。
*   **赛博格模式**：人类与机器进行非常紧密、细粒度的协作。

![](img/9ddc547a45ac114d3402048737228dcd_85.png)

![](img/9ddc547a45ac114d3402048737228dcd_86.png)

要评估智能体对知识工作的宏观影响，可以结合两种方法：
1.  **自上而下分析**：利用职业数据库（如美国O*NET），评估每个职业中各项任务被AI自动化的潜力。
2.  **自下而上评估**：利用像WorkArena这样的基准，精确追踪当前技术对具体任务的自动化率。

这两种方法都不完美，但结合使用可以帮助决策者预测技术采纳曲线，规划员工再培训，以确保技术进步惠及所有人。

## 总结 📚

本节课中，我们一起学习了：
1.  **企业环境中的智能体**：明确了LLM智能体的定义、分类及其在企业自动化“长尾”任务中的独特价值。
2.  **API智能体与TAPE框架**：了解了智能体架构，并深入探讨了TAPE框架如何通过将执行轨迹（`tape`）数据化，来统一智能体的构建、调试和优化，实现高性能、低成本的智能体部署。
3.  **Web智能体**：学习了基础Web智能体的构建方法，认识了当前在复杂任务上面临的挑战，以及社区通过建立多样化基准测试（如WorkArena）和开发统一工具链（BrowserGym, AgentLab）来推动进步的努力。
4.  **未来展望**：探讨了智能体如何通过“人机团队”模式变革工作流程，以及评估其宏观经济影响的方法。

![](img/9ddc547a45ac114d3402048737228dcd_97.png)

![](img/9ddc547a45ac114d3402048737228dcd_99.png)

智能体技术正在快速发展，它有望将人类从大量重复性工作中解放出来，转向更具创造性和战略性的工作。如何安全、可靠、高效地构建和部署智能体，并管理其带来的社会影响，是未来重要的研究和实践方向。
