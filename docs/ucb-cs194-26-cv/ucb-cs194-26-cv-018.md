# 18：序列模型在文字与像素中的应用 🧠

在本节课中，我们将要学习序列模型，特别是马尔可夫链，如何被应用于文本生成和像素（图像/视频）合成。我们将看到，通过数据驱动的方法，我们可以创造出看似复杂的结果，其核心思想是重用和重组已有的数据片段。

---

## 天气预测：一个简单的起点 🌤️

上一节我们介绍了序列模型的基本概念，本节中我们来看看如何将其应用于一个熟悉的场景：天气预测。

假设我们想预测明天的天气，且天气只有三种状态：晴天、多云或雨天。我们可以收集大量历史数据，计算在已知今天天气的情况下，明天出现各种天气的概率。

以下是构建简单预测模型的步骤：
1.  收集历史天气数据。
2.  对于每一种今天的天气状态（如晴天），统计明天各种天气状态出现的频率。
3.  将这些频率转换为概率。

例如，如果今天是晴天，我们可能得到：明天是晴天的概率为70%，多云为20%，雨天为10%。这构成了一个**马尔可夫链**，其状态和转移概率可以表示为一个**转移概率矩阵**。

```python
# 一个简化的转移概率矩阵示例
# 行：今天状态 [晴天， 多云， 雨天]
# 列：明天状态 [晴天， 多云， 雨天]
transition_matrix = [
    [0.7, 0.2, 0.1],  # 今天晴天
    [0.3, 0.4, 0.3],  # 今天多云
    [0.2, 0.3, 0.5]   # 今天雨天
]
```

如果我们用表示今天天气的向量（如 `[1, 0, 0]` 代表晴天）乘以这个矩阵，就会得到明天天气的概率分布向量。多次相乘后，这个分布会收敛到该马尔可夫链的**稳态分布**（特征值为1的特征向量）。

为了获得更好的预测，我们可以扩展上下文。例如，不仅考虑今天，还考虑昨天的天气。这样状态数会从3个（单日）增加到9个（双日组合），模型能捕捉更长的依赖关系，预测更准确。

## 从天气到文本：语言模型 📝

理解了天气预测后，我们可以将相同的马尔可夫链思想应用于文本生成。克劳德·香农在1948年就提出了这个想法。

其核心是：基于大量文本数据，计算在给定当前一个（或几个）词的情况下，下一个词出现的概率。

以下是构建基础文本生成模型的步骤：
1.  选择一个上下文长度（例如，2个词）。
2.  在语料库中，统计所有给定上下文后，下一个词出现的频率。
3.  从这些概率中采样，生成下一个词。
4.  将新生成的词加入上下文，重复过程。

例如，从“我们需要”开始，模型可能根据概率采样出“食物”。然后上下文变为“需要食物”，再生成下一个词，依此类推。这种方法能生成局部统计特性正确、但整体可能无意义的文本，正如早期程序“Mark V Shaney”所展示的那样。

通过增加上下文长度（如使用三元组、四元组），模型能考虑更多信息，生成更连贯的文本。你手机上的**预测输入**功能，本质上就是基于这种原理。

## 从文本到像素：视频纹理 🎬

既然序列模型能处理文本，我们自然会问：它能处理像素吗？答案是肯定的，这引出了“视频纹理”的工作。

目标是：给定一段短视频，生成一段无限循环、看起来自然流畅的新视频。其关键是将视频帧序列视为一个马尔可夫链。

以下是创建视频纹理的核心步骤：
1.  **构建图**：将视频的每一帧视为一个状态。
2.  **计算相似度**：计算任意两帧之间的视觉相似度（如使用L2距离）。相似度越高，从一帧跳转到另一帧的“合理性”就越高。
3.  **定义转移概率**：将距离转换为概率，通常使用公式 `exp(-距离/σ)`，其中σ控制跳转的活跃度。
4.  **采样生成**：从某一帧开始，根据转移概率随机跳转到下一帧，从而生成新的视频序列。

但这里有一个关键技巧：为了保持运动的连续性，我们不仅要求跳转到的目标帧（J）与当前帧（I）相似，更要求J与I的下一帧（I+1）相似。这样跳转时才不会出现明显的视觉断裂。

然而，仅凭单帧信息（位置）无法区分运动方向（速度）。例如，一个摆锤在左右两端的单帧看起来可能一样。解决方法同样是**扩展上下文**：我们使用一个由多帧组成的“块”或“N元组”作为状态，这样就能捕捉运动信息。

## 图像纹理合成：补丁与搜索 🖼️

视频纹理的思想源于更早的图像纹理合成工作。目标是将一小块纹理样本，扩展生成一大张无缝的纹理图像。

核心挑战是：像素的上下文（周围像素）可能非常大，直接建立概率表不现实。解决方案是使用“搜索”而非“建表”的香农技巧。

以下是图像纹理合成的步骤：
1.  **逐像素生成**：从左上角开始，每次生成一个像素。
2.  **上下文匹配**：对于待生成的像素，取其上方和左侧已生成的像素作为上下文（一个L形邻域）。
3.  **搜索最佳匹配**：拿着这个上下文邻域，到原始纹理样本图像中滑动搜索，找到最相似的邻域位置。
4.  **复制像素**：将找到的最佳匹配位置在原始图像中对应的那个像素，复制到待生成位置。
5.  **重复**：从左到右、从上到下重复此过程。

这种方法只需一个参数：上下文窗口的大小。窗口太小，生成结果杂乱；窗口适中，能捕捉局部结构；窗口足够大，则能生成连贯的纹理。这种方法甚至可以用于图像补全和外绘。

## 超越简单匹配：注意力机制与Transformer 🚀

无论是N元组模型还是补丁搜索，其上下文都是固定且有限的。对于需要长距离依赖的任务（如诗歌押韵），我们需要更大的上下文，但又不能简单地扩大窗口，因为计算会变得不可行。

解决方案是**注意力机制**。它允许模型在处理序列的每个位置时，动态地、有选择地关注输入序列中任何其他部分，而不是仅关注前几个词。

其核心思想是：
1.  将输入词转换为有意义的向量表示（**词嵌入**），使语义相似的词在向量空间中接近。
2.  对于序列中的每个位置，计算一个“注意力权重”向量，该向量表示在生成当前输出时，应对输入序列的每个部分赋予多少关注。
3.  将这些权重应用于输入向量，得到一个汇总了相关信息的“上下文向量”。
4.  将此上下文向量与当前位置的信息一起，输入到一个神经网络中，预测下一个输出（词或视觉标记）。

**Transformer** 模型就是基于这种注意力机制构建的。通过堆叠多个这样的“注意力+前馈网络”层，模型能够在底层捕捉语法和局部结构，在高层捕捉语义和长距离依赖关系。它通过“预测下一个词”这个自监督任务进行端到端训练，无需人工标注。

当这种拥有海量参数的模型在互联网级别的全部文本数据上进行训练后，它就能生成极其逼真和复杂的文本，因为它“见过”并“记住”了各种模式和知识组合。

## 生成图像：视觉Transformer 🎨

最后，我们将语言模型的能力延伸到像素生成。直接让模型输出数百万个RGB像素值是不现实的。因此，我们采用一个两阶段方法：

1.  **将图像“词汇化”**：
    *   将图像分割成许多小补丁（如8x8像素）。
    *   使用聚类等方法，将每个补丁映射到一个“视觉词”标记（一个离散的编号）。
    *   这样，一张图像就被表示为一个视觉词标记的序列。

2.  **训练视觉语言模型**：
    *   使用一个强大的Transformer模型（如GPT架构）在大量“图像-标记”序列数据上进行训练。
    *   训练目标同样是“预测下一个标记”。
    *   由于Transformer的注意力机制，模型在生成图像的每一个局部时，都能“记住”全局的生成目标（例如，“一只松鼠在够坚果”）。

3.  **生成与上采样**：
    *   给定一个文本提示，模型自回归地生成一系列视觉词标记。
    *   将这些标记解码回小补丁，拼接成一张低分辨率图像。
    *   最后，使用一个上采样模型（如图像到图像翻译网络）将低分辨率图像转换为清晰的高分辨率图像。

这种方法展示了，当算法（如Transformer）具备足够的能力来吸收和处理海量数据时，数据本身成为了创造力的核心源泉。模型通过重组其“经验”（训练数据）中的片段，能够生成新颖、多样且符合要求的视觉内容。

## 总结 ✨

本节课中我们一起学习了序列模型从基础到前沿的应用之旅：
1.  我们从**马尔可夫链**和天气预测出发，理解了基于概率的状态转移模型。
2.  将其应用于**文本生成**，看到了如何通过数据统计生成语言。
3.  进一步扩展到**视频纹理**和**图像纹理合成**，学会了如何通过匹配和重用数据片段来生成视觉内容。
4.  最后，我们探讨了现代**注意力机制**和**Transformer**模型如何通过动态关注和海量数据训练，极大地提升了序列建模的能力，并成功应用于**文本到图像的生成**。

贯穿始终的核心思想是**数据驱动**和**重用重组**。无论是早期的简单概率模型，还是现代的巨大神经网络，其魔力很大程度上源于对大量已有数据模式的学习与巧妙组合。
