### 线性搜索

![](img/155d22819dc0cdb343122828e099e8ea_35.png)

线性搜索是一种暴力方法，它简单地遍历列表中的每个元素，直到找到目标或到达列表末尾。这种方法不要求列表有序。

![](img/155d22819dc0cdb343122828e099e8ea_37.png)

以下是线性搜索的代码示例：

![](img/155d22819dc0cdb343122828e099e8ea_39.png)

![](img/155d22819dc0cdb343122828e099e8ea_41.png)

```python
def linear_search(lst, e):
    found = False
    for i in range(len(lst)):
        if lst[i] == e:
            found = True
    return found
```

![](img/155d22819dc0cdb343122828e099e8ea_43.png)

![](img/155d22819dc0cdb343122828e099e8ea_45.png)

**复杂度分析**：在最坏情况下（目标元素不在列表中），我们需要检查列表中的每一个元素。循环部分执行 `n` 次（`n` 为列表长度），循环内的操作是常数时间。因此，线性搜索的**最坏情况复杂度是 O(n)**，即线性复杂度。

![](img/155d22819dc0cdb343122828e099e8ea_47.png)

![](img/155d22819dc0cdb343122828e099e8ea_49.png)

### 二分搜索（折半查找）

![](img/155d22819dc0cdb343122828e099e8ea_51.png)

![](img/155d22819dc0cdb343122828e099e8ea_53.png)

二分搜索要求列表是**有序的**。它通过反复将待搜索区间减半来工作：比较中间元素与目标值，根据比较结果决定搜索左半部分或右半部分。

![](img/155d22819dc0cdb343122828e099e8ea_55.png)

以下是二分搜索的递归实现示例：

![](img/155d22819dc0cdb343122828e099e8ea_57.png)

```python
def bisection_search(lst, e):
    def helper(lst, e, low, high):
        if high == low:
            return lst[low] == e
        mid = (low + high) // 2
        if lst[mid] == e:
            return True
        elif lst[mid] > e:
            if low == mid:
                return False
            else:
                return helper(lst, e, low, mid - 1)
        else:
            return helper(lst, e, mid + 1, high)
    if len(lst) == 0:
        return False
    else:
        return helper(lst, e, 0, len(lst) - 1)
```

![](img/155d22819dc0cdb343122828e099e8ea_59.png)

**复杂度分析**：在每一步，问题规模减半。因此，所需的步骤数（即递归调用次数）是 **O(log n)**。每次递归调用内部是常数时间操作（仅传递指针，不复制列表）。因此，二分搜索的复杂度是**对数级 O(log n)**。

## 排序的价值：何时先排序再搜索？

![](img/155d22819dc0cdb343122828e099e8ea_74.png)

![](img/155d22819dc0cdb343122828e099e8ea_76.png)

![](img/155d22819dc0cdb343122828e099e8ea_78.png)

既然二分搜索如此高效（O(log n)），一个自然的想法是：是否应该先对列表排序，然后再进行搜索？

![](img/155d22819dc0cdb343122828e099e8ea_80.png)

![](img/155d22819dc0cdb343122828e099e8ea_82.png)

设 `Sort` 为排序的成本，`Search` 为搜索的成本。
*   单次线性搜索成本：**O(n)**
*   排序后单次二分搜索成本：**Sort + O(log n)**

![](img/155d22819dc0cdb343122828e099e8ea_84.png)

![](img/155d22819dc0cdb343122828e099e8ea_86.png)

我们希望 `Sort + O(log n) < O(n)`。然而，任何排序算法都至少需要查看列表中的每个元素一次，因此排序的复杂度**至少是 O(n)**。这使得上述不等式在单次搜索的场景下似乎不成立。

![](img/155d22819dc0cdb343122828e099e8ea_88.png)

![](img/155d22819dc0cdb343122828e099e8ea_90.png)

但是，关键在于**摊销成本**。如果我们计划对同一个列表进行 **k 次** 搜索，那么比较就变成了：
*   k 次线性搜索成本：**k * O(n)**
*   一次排序 + k 次二分搜索成本：**Sort + k * O(log n)**

![](img/155d22819dc0cdb343122828e099e8ea_92.png)

![](img/155d22819dc0cdb343122828e099e8ea_94.png)

![](img/155d22819dc0cdb343122828e099e8ea_96.png)

![](img/155d22819dc0cdb343122828e099e8ea_98.png)

对于较大的 k 值，只要排序算法足够高效，先排序再多次搜索的策略就会更具优势。这就引出了我们的下一个主题：如何高效地排序？

![](img/155d22819dc0cdb343122828e099e8ea_102.png)

![](img/155d22819dc0cdb343122828e099e8ea_108.png)

### 1. 冒泡排序

![](img/155d22819dc0cdb343122828e099e8ea_118.png)

![](img/155d22819dc0cdb343122828e099e8ea_120.png)

![](img/155d22819dc0cdb343122828e099e8ea_122.png)

冒泡排序通过重复遍历列表，比较相邻元素，如果顺序错误就交换它们。每一轮遍历都会将当前未排序部分的最大元素“冒泡”到正确位置。

**算法过程**：
1.  从列表开头开始，比较每对相邻元素。
2.  如果前一个元素大于后一个元素，则交换它们。
3.  完成一轮遍历后，最大的元素已位于列表末尾。
4.  重复上述步骤，每次忽略已排序好的末尾部分，直到某一轮遍历中没有发生任何交换。

![](img/155d22819dc0cdb343122828e099e8ea_124.png)

![](img/155d22819dc0cdb343122828e099e8ea_126.png)

![](img/155d22819dc0cdb343122828e099e8ea_128.png)

![](img/155d22819dc0cdb343122828e099e8ea_130.png)

以下是冒泡排序的代码示例：

```python
def bubble_sort(lst):
    swap = False
    while not swap:
        swap = True
        for j in range(1, len(lst)):
            if lst[j-1] > lst[j]:
                swap = False
                lst[j-1], lst[j] = lst[j], lst[j-1]
```

**复杂度分析**：
*   外层 `while` 循环：在最坏情况下（列表完全逆序），需要遍历 `n` 轮。
*   内层 `for` 循环：每轮遍历大约 `n` 次比较/交换。
*   因此，冒泡排序的**最坏情况和平均情况复杂度是 O(n²)**，即二次复杂度。

![](img/155d22819dc0cdb343122828e099e8ea_132.png)

![](img/155d22819dc0cdb343122828e099e8ea_134.png)

![](img/155d22819dc0cdb343122828e099e8ea_136.png)

### 2. 选择排序

![](img/155d22819dc0cdb343122828e099e8ea_138.png)

![](img/155d22819dc0cdb343122828e099e8ea_140.png)

选择排序在每次迭代中找到未排序部分的最小元素，并将其与未排序部分的第一个元素交换。这样，列表的前端逐渐构建起有序序列。

![](img/155d22819dc0cdb343122828e099e8ea_142.png)

**算法过程**：
1.  找到列表中最小的元素，将其与索引 0 位置的元素交换。
2.  在剩余列表（索引 1 到 n-1）中找到最小元素，将其与索引 1 位置的元素交换。
3.  重复此过程，每次将未排序部分的第一个元素与其中的最小元素交换，直到整个列表有序。

![](img/155d22819dc0cdb343122828e099e8ea_144.png)

**循环不变式**：在执行了 `i` 次迭代后，列表的前 `i` 个元素（前缀）是已排序的，并且前缀中的任何元素都不大于后缀（剩余未排序部分）中的最小元素。

![](img/155d22819dc0cdb343122828e099e8ea_146.png)

![](img/155d22819dc0cdb343122828e099e8ea_148.png)

![](img/155d22819dc0cdb343122828e099e8ea_150.png)

以下是选择排序的代码示例：

![](img/155d22819dc0cdb343122828e099e8ea_152.png)

```python
def selection_sort(lst):
    suffix_start = 0
    while suffix_start != len(lst):
        for i in range(suffix_start, len(lst)):
            if lst[i] < lst[suffix_start]:
                lst[suffix_start], lst[i] = lst[i], lst[suffix_start]
        suffix_start += 1
```

![](img/155d22819dc0cdb343122828e099e8ea_154.png)

![](img/155d22819dc0cdb343122828e099e8ea_156.png)

![](img/155d22819dc0cdb343122828e099e8ea_158.png)

**复杂度分析**：
*   外层 `while` 循环：执行 `n` 次。
*   内层 `for` 循环：第一次执行 `n` 次比较，第二次 `n-1` 次，依此类推。
*   总的比较次数约为 n + (n-1) + ... + 1 = n(n+1)/2。
*   因此，选择排序的**复杂度也是 O(n²)**。

![](img/155d22819dc0cdb343122828e099e8ea_160.png)

![](img/155d22819dc0cdb343122828e099e8ea_162.png)

![](img/155d22819dc0cdb343122828e099e8ea_164.png)

### 3. 归并排序

归并排序采用**分治**策略。它将列表递归地分成两半，分别对两半进行排序，然后将两个已排序的子列表合并成一个完整的已排序列表。合并两个已排序列表是一个高效的操作。

![](img/155d22819dc0cdb343122828e099e8ea_166.png)

![](img/155d22819dc0cdb343122828e099e8ea_168.png)

**算法过程**：
1.  **分**：如果列表长度为 0 或 1，则它已经有序，直接返回。否则，找到中点，将列表分成左右两个子列表。
2.  **治**：递归地对左子列表和右子列表调用归并排序。
3.  **合**：将两个已排序的子列表合并成一个新的已排序列表。合并时，比较两个子列表前端的元素，将较小的元素放入结果列表，并移动相应子列表的指针，直到一个子列表为空，然后将另一个子列表的剩余部分全部追加到结果中。

![](img/155d22819dc0cdb343122828e099e8ea_170.png)

以下是归并排序的代码示例：

![](img/155d22819dc0cdb343122828e099e8ea_172.png)

![](img/155d22819dc0cdb343122828e099e8ea_174.png)

```python
def merge(left, right):
    result = []
    i, j = 0, 0
    while i < len(left) and j < len(right):
        if left[i] < right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    while i < len(left):
        result.append(left[i])
        i += 1
    while j < len(right):
        result.append(right[j])
        j += 1
    return result

def merge_sort(lst):
    if len(lst) < 2:
        return lst[:]
    else:
        middle = len(lst) // 2
        left = merge_sort(lst[:middle])
        right = merge_sort(lst[middle:])
        return merge(left, right)
```

**复杂度分析**：
*   **合并操作 `merge`**：需要遍历两个子列表的所有元素各一次。如果两个子列表总长度为 `n`，则合并操作的复杂度为 **O(n)**。
*   **递归树**：归并排序将问题不断对半分割，形成一棵深度为 **log₂ n** 的递归树。
*   **每层工作量**：在递归树的每一层，我们需要合并所有该层的子问题。虽然子问题规模变小，但子问题数量增多。关键点在于，**递归树的每一层需要处理的元素总数都是 `n`**。因此，每一层的总工作量是 O(n)。
*   **总复杂度**：有 O(log n) 层，每层工作量为 O(n)。因此，归并排序的**复杂度是 O(n log n)**，即对数线性复杂度。这是基于比较的排序算法中，最坏情况下的最优复杂度之一。

    *   **冒泡排序**和**选择排序**，它们简单直观，但**复杂度为 O(n²)**，适用于小规模数据。
    *   **归并排序**，它采用分治策略，具有优异的 **O(n log n)** 复杂度，是处理大规模数据的高效选择。
