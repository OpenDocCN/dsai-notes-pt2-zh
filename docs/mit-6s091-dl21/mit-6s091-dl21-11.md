## 概述：为何需要AI进行3D内容创作？

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_9.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_11.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_13.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_15.png)

计算机图形学领域正经历一场革命，新的渲染技术与AI的结合使得实时、高质量的内容创作成为可能。三维内容在建筑、游戏、电影、VR/AR以及机器人仿真等领域无处不在。然而，传统的手工创作方式耗时耗力。例如，艺术家仅为了对齐一张现实世界的照片来重建一个场景，就可能需要花费四个小时。对于像《侠盗猎车手》这样的大型游戏，则需要上千人工作数年。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_17.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_19.png)

因此，我们的目标是利用AI，特别是深度学习，来加速甚至自动化三维内容的创建过程，包括场景布局、资产生成和世界动态模拟。

## 1. 生成场景：从概率语法到可学习模型

上一节我们介绍了AI在3D内容创作中的宏观目标，本节中我们来看看如何具体生成一个三维场景。

在游戏开发中，创建多样化的大型世界通常依赖于**过程化生成**技术，其核心是使用**概率语法**或规则来定义如何组合场景元素（如车道、汽车、树木），使其看起来合理。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_21.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_23.png)

**核心概念**：一个场景可以表示为一个**图（Graph）**，其中节点代表资产类型（如汽车、建筑），属性包括位置、旋转、颜色等。艺术家手动设置这些属性的分布（例如，城市中汽车的数量和类型）。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_25.png)

我们的目标是让AI从真实数据（如驾驶录像）中**自动学习**这些分布，而无需人工标注。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_27.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_29.png)

### 方法：可微渲染与图神经网络

我们提出了一种名为 **Meta-Sim** 的方法。其核心思想是：
1.  从一个初始的、手工设定的概率语法生成**场景图**。
2.  使用一个**图神经网络（GNN）** 来修改图中每个节点的属性（如旋转、颜色）。
3.  通过一个**可微渲染器**将修改后的场景图渲染成图像。
4.  计算渲染图像与真实世界图像分布之间的差异（使用如**最大均值差异，MMD** 的度量）。
5.  利用这个差异，通过**强化学习**和数值梯度，反向传播以更新图神经网络，使其生成的场景更接近真实数据。

以下是该流程的简化表示：
```
初始场景图 --(GNN修改属性)--> 新场景图 --(可微渲染器)--> 合成图像
                                            |
                                        损失计算 (MMD)
                                            |
                                        更新 GNN
```

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_31.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_33.png)

### 学习场景图的结构

上述方法优化了属性，但场景图的结构（如有几条车道、多少辆车）仍然是手工定义的。更进一步，我们可以让AI也学习生成场景图的结构。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_35.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_37.png)

我们使用**概率上下文无关语法（PCFG）** 来建模场景生成过程。网络学习在给定当前已生成图部分（上下文）的情况下，预测下一个应扩展的符号（如“增加一条车道”）的概率。

**结果**：经过训练，模型能够从初始化很差的语法（生成几乎空荡的场景）进化到生成与真实城市数据分布高度匹配的多样化场景。例如，生成场景中的汽车数量分布与真实数据集（如KITTI）的分布非常接近。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_39.png)

**评估**：通过从生成模型中合成大量带标注的图像，并用于训练目标检测器等模型，然后在真实数据上测试其性能提升，可以证明生成数据的有效性。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_43.png)

## 2. 创建三维资产：从单张图像到3D模型

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_49.png)

上一节我们探讨了如何生成整个场景的布局，本节中我们来看看如何创建场景中具体的物体（资产），例如汽车、建筑。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_51.png)

理想情况下，我们希望仅通过互联网上广泛存在的**单张物体图片**，就能自动生成其带纹理的**三维网格模型**。这被称为“逆渲染”问题。

**核心思路**：利用**可微渲染**将计算机视觉（从图像到3D）和计算机图形学（从3D到图像）连接起来。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_53.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_55.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_57.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_59.png)

### 可微渲染管线

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_61.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_63.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_65.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_67.png)

基本流程如下：
1.  一个神经网络接收输入图像，并预测三维属性，如**网格顶点、纹理、材质、光照**。
2.  将这些预测输入一个**可微渲染器**，合成出一张新的图像。
3.  计算合成图像与输入图像之间的损失（如像素差异、感知损失）。
4.  由于渲染器可微，损失可以反向传播，从而更新神经网络，使其预测的三维属性能更准确地重建输入图像。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_69.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_71.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_73.png)

公式化表示如下：
```
输入图像 --> 神经网络 --> (网格, 纹理, 光照...) --> 可微渲染器 --> 合成图像
                                                              |
                                                          与输入图像计算损失
                                                              |
                                                          更新神经网络
```

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_75.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_77.png)

### 数据获取的挑战与巧思

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_79.png)

大多数现有方法需要同一物体的**多视角图像**进行训练，而这在网络数据中很难获得。一个巧妙的解决方案是利用**生成对抗网络（GAN）**，如StyleGAN。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_81.png)

研究发现，在GAN的潜在空间中，存在一些维度专门控制**视点**，而其他维度控制**物体内容**。通过固定内容代码并遍历视点代码，我们可以为“虚拟”的物体生成多个视角的图片，从而自动创建出训练所需的多视图数据。

**结果**：使用这种“合成”的多视图数据训练上述可微渲染管线后，模型能够从单张汽车图片预测出其三维形状、纹理和材质（如金属、玻璃）。预测出的模型可以放入游戏引擎或模拟器中，甚至能进一步进行物理动画（如让车轮转动）。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_83.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_85.png)

## 3. 模拟动态世界：神经模拟器

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_87.png)

我们不仅需要静态的场景和物体，还需要让世界“活”起来，具有动态和行为。传统上，这是游戏引擎的职责。本节我们探索能否用神经网络来模拟世界动态，即构建一个**神经模拟器**。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_89.png)

**目标**：用神经网络替代或模仿游戏引擎。用户输入动作（如方向盘、油门），神经网络根据当前状态预测出下一帧的画面，从而模拟出世界的物理变化和视觉表现。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_91.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_93.png)

### 架构：神经渲染游戏引擎

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_95.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_97.png)

我们设计了一个模块化架构，主要包括：
- **动态引擎**：一个神经网络，负责根据当前状态和用户动作，更新世界的内部状态（潜在表示）。
- **渲染引擎**：一个神经网络，负责将更新后的内部状态解码，渲染成下一帧图像。
- **记忆模块**：帮助模型保持长期一致性，避免游戏逻辑混乱。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_99.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_101.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_103.png)

### 应用实例

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_105.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_106.png)

1.  **吃豆人（Pac-Man）**：模型需要学会游戏规则（吃豆子、躲避幽灵、吃能量丸等），并合成出连贯的游戏画面。这是一个验证概念的有趣起点。
2.  **驾驶模拟**：在更复杂的3D环境（如CARLA模拟器）甚至真实驾驶视频上训练。模型接收转向和速度控制，尝试合成出后续的驾驶帧。结果显示，模型能够学习到相当一致的世界表示和动态，尽管在细节上还不完美。

这项工作展示了用纯数据驱动的方式学习世界模拟的潜力，为构建无需精确3D模型即可进行仿真的系统提供了新思路。

## 4. 社区与工具：Kaolin库

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_121.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_123.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_125.png)

解决如此宏大的问题需要社区共同努力。为此，我们发布了一个名为 **Kaolin** 的三维深度学习工具库，旨在加速该领域的研究。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_127.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_129.png)

**Kaolin 提供**：
- 一个 **PyTorch 库**，包含优化的3D数据处理操作（网格、点云、体素）。
- 一个 **交互式可视化工具**，方便在训练过程中实时查看3D预测结果。
- 灵活的**可微渲染器**模块，帮助研究者轻松搭建“逆渲染”等应用。
- 支持在网页浏览器中远程可视化训练进度。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_131.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_133.png)

## 总结与建议

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_153.png)

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_155.png)

本节课中我们一起学习了利用AI进行3D内容创作的几个关键方向：

1.  **场景生成**：通过结合概率语法、图神经网络和可微渲染，从真实数据中学习并生成符合现实分布的复杂场景。
2.  **资产创建**：利用可微渲染和生成模型（如GAN）提供的数据，实现从单张图像到带纹理、材质的三维模型的自动重建。
3.  **世界模拟**：探索使用神经网络作为模拟器，学习世界的动态规则并合成连续帧，为仿真和游戏提供新范式。
4.  **工具生态**：介绍了Kaolin库，它为3D深度学习研究提供了必要的工具支持。

![](img/9a5a4bf0fb96963fc1cb5e68edbadc3b_157.png)

最后，给研究者的建议：**保持热情，专注于解决真正的问题而非论文数量，积极寻找跨领域的合作者，并享受研究过程中的乐趣**。三维内容创作的AI自动化是一个充满挑战但前景广阔的领域，还有许多问题等待探索和解决。
