# 01：大型语言模型与AI开发入门

在本节课中，我们将深入探讨大型语言模型（LLM）从原始数据到智能助手的完整构建流程。我们将剖析其核心训练阶段，并学习如何通过高级提示技术有效地使用它们。

## 从原始数据到基础模型 🧱

我们都与语言模型互动过，无论是向ChatGPT询问食谱，还是让AI总结长篇文档。你点击发送，几乎瞬间就能得到看似智能的回复。但在你点击发送后，那个“黑箱”里究竟发生了什么？更重要的是，在你输入第一个提示词之前的数月甚至数年里，发生了什么？这正是我们今天要探讨的核心问题。

本次深度探讨旨在揭开这个不可思议的多阶段工程流程的面纱，该流程将原始、杂乱的互联网数据转化为你使用的AI助手。我们将讨论训练的基本阶段，重点关注三个主要阶段：预训练、监督微调和强化学习。然后，我们将深入探讨实际指挥这些模型所需的高级提示技术。

我们的资料来源于一些相当深入的技术论文和工程讲座，它们揭示了这一切背后惊人的规模和巧妙技巧。我们学习者的任务是真正理解这些系统的真实架构。我们将追溯模型的演变过程，从它消耗的数万亿文本片段开始，一直到让它看起来能够推理的方法。

那么，让我们从最基础的部分开始。在LLM成为有用的助手之前，它始于我们所说的**基础模型**。从功能上讲，它是一个庞大的、高度复杂的文本补全机器。

### 预训练：知识获取阶段

这个第一阶段——预训练——的目标仅仅是知识获取，其规模之大令人难以想象。

**数据收集**是第一步，其范围堪称“圣经级”。它始于密集的数据抓取，主要来自互联网的深层档案。最大的来源通常是Common Crawl，这是一个自2007年以来一直在索引网页的非营利组织，覆盖超过27亿个页面。然后，这些原始互联网数据会与高质量、经过筛选的文档相结合，例如维基百科、学术期刊、GitHub上的公共代码等。

数据是原始的，这意味着它极其杂乱。他们当然不能将整个互联网直接丢给模型。**过滤过程是无情的**：他们会移除所有垃圾信息、恶意软件、已知的种族主义内容、成人材料，甚至仅仅是重复的文本。但即使经过所有这些筛选，数据集仍然庞大得惊人。一个具有代表性的高质量公共数据集，如FineWeb，其文本数据量仍达到约44TB。

仅仅是文本？这确实令人眩晕。这些文本需要被处理，转换成**令牌**序列。令牌基本上是文本的小块，可以是一个词，甚至是词的一部分。仅FineWeb数据集就转化为大约**15万亿个令牌**，即15000亿个训练数据片段，模型必须吸收这些数据。

### 核心机制：预测下一个词

那么，核心的机械操作是什么？那些数十亿的GPU到底在做什么？其机制在概念上出奇地简单：LLM是**自回归模型**。这意味着它们完全根据之前的文本来预测序列中下一个令牌的概率。

例如，如果我说“那只猫坐在……”，模型会预测“垫子上”或类似的内容。它计算最可能的补全内容。现代模型可以处理惊人的上下文量来进行这种预测。它的“工作记忆”窗口变得非常巨大，呈指数级增长。像GPT-2这样的旧模型可能只能处理大约1000个令牌的上下文，而现代模型可以回顾数十万，有时甚至上百万个令牌。

这种预测完全是数学运算。它将上下文通过一个具有**数十亿甚至数万亿可调参数**的神经网络。可以把这些参数想象成巨型调音台上的旋钮。GPT-2有15亿个，据报道GPT-4可能有1.8万亿个。训练就是一遍又一遍地微调这些旋钮的过程，使模型对下一个令牌的预测与训练数据中的实际情况相匹配，每秒进行数百万次。

这种优化需要几乎难以置信的大量专用计算能力。这就是所谓的“淘金热”。公司争相租用这些庞大的机器，例如一个Apex H100 GPU节点，每GPU每小时成本约为3美元。巨大的开销正是训练大型基础模型需要数千万美元的原因。这种成本分析实际上决定了整个行业的结构，只有少数参与者能够负担得起创建这些基础模型的费用。

每一台昂贵的机器都在试图更快、更准确地预测下一个令牌。因此，预训练的目标是创建互联网的一个巨大的**有损压缩包**。这是一个强有力的思考方式：它是人类知识的概率性zip文件。

但如果它是压缩的，哪些信息会丢失呢？它会丢失细微差别，失去推理或理解什么是真实、什么只是互联网上常见内容的能力。它学习的是知识的语法，而不是逻辑的深层结构。这正好将我们引向下一个阶段。

## 从基础模型到对话助手 🗣️

那个昂贵的基础模型，那个自动补全机器，并不是我们与ChatGPT对话时使用的模型。预训练的输出只是基础模型。要将其转变为有用的对话助手，我们必须进入**后训练**阶段。

这始于**监督微调**。SFT是塑造模型“个性”的第一步，它在计算上要便宜得多，我们谈论的是小时而不是月。我们交换了数据：用原始互联网文本换成了更小、高度精选的数据集——数十万个高质量对话示例。

**人类标注员**在这里登场。他们基本上是在定义AI的“人设”。公司雇佣人类，给予他们非常详细和严格的指令，他们的任务是为各种提示词编写理想的助手回复。他们通过示例，直接教导LLM采用那种“有帮助、诚实且无害”的人设。

因此，当我与AI对话时，我并不是在与某种新形式的智能交流，而是在与一个神经网络互动，这个网络模拟的是一个平均技能高超、完美遵循指令的人类标注员。最简单的说法是：你正在与有史以来最昂贵、高度优化的人类客服模拟器对话。

## 教授模型如何思考：强化学习 🧠

一旦模型具备了正确的“人设”，我们需要教它如何实际解决问题。这就将我们带到了第三个、也是最先进的阶段：**强化学习**。

这与SFT有何不同？SFT教它“说什么”，而RL教它“如何思考”。通过RL，模型可以在可验证的领域（如数学和代码）中练习解决问题，在这些领域我们确切知道最终正确答案是什么。你给它问题和答案，但它必须找出如何到达答案的路径。它必须发现能可靠地导向那个结果的令牌序列。

这变得真正迷人，因为正是在这里，我们看到了模型的认知策略浮现出来。它通过创建一种**内部独白**来学习思考。没错，因为模型每个令牌的计算能力是有限的，它学会了必须将推理分布在多个令牌上。我们称之为**思维链**。

如果你只是要求它一次性输出一个困难数学问题的答案，它会失败。它会在心算上持续失败。关键要点就在这里：**模型需要令牌来思考**。如果你不让它逐步思考，就等于要求它不用草稿纸做微积分。

这种纯粹的RL过程在答案可以客观验证时（如数学、代码、象棋）效果很好。但对于主观性的事物，比如写一个笑话或有说服力的文章呢？对于这些领域，公司使用一种称为**基于人类反馈的强化学习**的变体。在这里，他们训练第二个AI——一个奖励模型——来充当人类评委。人类对不同的AI输出进行排名，奖励模型学习复制这些排名，它成为人类品味的代理。

但资料指出这里有一个主要缺点：强化学习非常擅长找到“玩弄”任何模拟的方法。LLM可以学会生成内容，仅仅是为了欺骗奖励模型给出高分，优先考虑优化而非真正的质量。因此，RLHF虽然强大，但也是一个有缺陷的过程。模型最终会学会利用奖励系统。没错，人类的监督仍然是强制性的。

## 提示工程：指挥的艺术 🎨

好的，一旦你拥有了这个经过高度训练的模型——具备知识、个性和一定的推理能力——最后阶段就是关于**提示工程**，即通过对话编程模型的艺术。

一个基础技术是**上下文学习**。你只需在提示本身内部给模型几个示例。例如，如果你需要它使用非常特定的敬语风格将句子翻译成韩语，你只需提供一两个该风格的确切示例，模型就会立即适应，它在上下文中学习模式。

但对于更复杂的逻辑任务，我们必须回到思维链。甚至一个简单的指令，如“让我们逐步思考”，这被称为**零样本思维链**，也能显著提高准确性。你正在解锁模型大声推理的能力。

然而，模型的知识是冻结在某个时间点的。它不知道昨天发生了什么，也不知道你公司的私有数据库里有什么。这就是为什么外部工具对于现实世界任务如此关键，特别是**检索增强生成**。

RAG是我们对抗“幻觉”的关键防御手段。这就像给AI一个智能的谷歌搜索功能。LLM意识到它需要当前信息，因此它会生成特殊的令牌来调用工具（如网络搜索）。该搜索的结果被直接反馈到AI的工作记忆中，就在它生成最终答案之前，输入到其上下文窗口中。这迫使模型使用当前事实，而不仅仅是其旧的训练记忆。

我们可以通过**自我修正**技术进一步优化这一点。以**自我一致性**为例：你不是问模型一个问题一次，而是问它五次，迫使它找到五条不同的推理路径，然后取多数答案。这减少了随机错误。

然后是**反思**，这对编码非常有用。你将错误信息反馈到提示中，并要求模型批评和修改自己的代码。你正在教它实时从错误中学习。这些技术将简单的聊天与高杠杆的专业使用区分开来。

## 专业应用：从替代到增强 ⚙️

这给我们带来了专业人士正在经历的核心转变。讨论总是关于替代，但资料强调了一个不同的观点。其信条是：**你不会被AI取代，你将被懂得如何使用AI的工程师取代**。LLM不是在取代工程师，而是在增强他们，将一个普通工程师变成一个高效能的工程师。

我们在OpenAI等公司内部使用Codex模型时看到了这一点。让我们看一个具体例子。

**代码理解**：工程师粘贴错误堆栈跟踪。通常，这需要一小时的挖掘。相反，他们提示模型：“给定这个堆栈跟踪，查看此代码库中的作者逻辑并追踪数据流。”这直接节省了45分钟。

**重构**：另一个巨大的应用。更新一个遗留的API调用，涉及数十个文件。这是一个乏味的三小时手动工作。使用LLM，你提示“在所有40个文件中一致地应用此更改”，这只需要90秒。这是真正的杠杆作用。

**性能优化**：与其花一小时寻找瓶颈，不如让模型分析代码，标记缓慢的数据库调用，并起草修复方案。30分钟的工作变成了5分钟的提示。

**质量保证与测试**：你可以让模型指向覆盖率低的代码，一觉醒来就能得到可运行的单元测试，覆盖了人类可能遗漏的边缘情况。

但要获得这种精确度，你不能只是随意输入请求。有一些最佳实践。

**角色提示**：你必须积极地定义角色。“你是一个热爱编程、具备高级软件开发者水平的乐于助人的助手。”

**结构化提示**：在粘贴复杂数据时使用像`[ERROR]`或`[LOG]`这样的标签，以便模型能高效解析。

最先进的技术是**持久上下文**。工程师会维护一个文件，比如`agent_spec.md`，里面包含所有业务逻辑和命名约定，这些是AI不可能知道的。他们将这个上下文粘贴到每个会话中。这使模型保持信息灵通，并显著提高了所有提示的准确性。

## 总结与展望 🔮

今天，我们描绘了整个路径：从数万亿的原始互联网令牌，到赋予AI个性的人类标注对话，再到需要思维链的高级推理。

对于学习者，核心要点是：LLM虽然强大，但它们存在认知缺陷。我们称之为**能力的瑞士奶酪模型**。漏洞并不完全对齐：它可以解决一个奥林匹克级别的数学问题，但随后可能在比较两个简单数字（如9.1和9.9）时失败。你必须将它们视为**随机性工具**，始终检查它们的工作，并使用这些技术来弥补它们的局限性。

最后，让我们深入思考一个值得课后回味的问题：当LLM在答案可验证的问题上使用纯粹的强化学习进行训练时，模型不受人类逻辑的约束。多年前我们在AlphaGo发现第37步棋时就看到了这一点——那是一步人类从未考虑过的、辉煌且完全出乎意料的策略。

因此，如果这些“思考模型”继续在开放领域问题解决中进化，发现新的、更优越的认知策略，甚至可能发展出一种比英语更高效的、用于思考的新内部语言，那么当解决复杂商业问题的最佳方式是人类无法构想的事物时，这对我们意味着什么？这提出了一个关于未来人机协作的重要问题。

在本节课中，我们一起学习了大型语言模型从数据收集、预训练、监督微调、强化学习到实际应用和提示工程的完整生命周期。理解这个流程是有效利用和批判性评估AI工具的关键。
