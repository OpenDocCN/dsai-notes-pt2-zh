## 扩散模型核心概念 🔍

简而言之，扩散模型的名字来源于扩散过程。例如，布朗运动，你取一个粒子并随时间不断向其添加高斯噪声。你可以看到，如果从 0 开始，这个过程会随时间演变，有点像随机游走。当然，这是一维的情况。你可以对更高维的数据（如图像）进行此操作，即向每个像素逐个添加独立噪声。这被称为前向过程，通过向数据添加噪声，你将数据（这里是图像）转化为纯噪声。

扩散模型试图学习反向过程，试图从噪声中恢复数据。例如，从完全噪声的图像开始，尝试预测一个噪声稍少的版本，然后以迭代方式持续进行，直到得到干净的图像。这是扩散模型工作的高级概述。

为了更具体，让我们看看扩散模型究竟是如何训练的。

## 如何训练扩散模型 🏋️

你从数据流形（可能是图像、分子等任何东西）中的一个样本开始，我们称之为 **x0**。你取这个干净数据，选择一个噪声水平（我们稍后会讨论如何选择），然后选择一个噪声向量（可以将其视为一个方向）。你将噪声添加到干净数据中，从而得到数据的噪声版本。扩散模型试图做的是，给定噪声输入以及噪声水平，尝试预测噪声的方向。这非常重要，它同时接收噪声输入和噪声水平，并尝试预测 **ε**（即噪声方向），通过最小化一些平方损失来实现。你可以将其视为一个监督学习问题，即给定噪声图像或噪声数据，尝试预测噪声方向。

那么如何采样这三个量呢？**x0** 通常是从训练数据中均匀随机采样，或者通过打乱数据并分批获取。**σ** 是噪声调度，我们稍后会详细讨论，训练时的噪声水平是从噪声调度中采样的。噪声方向则简单地采样为各向同性高斯分布，每个分量独立同分布。

有任何关于如何训练的问题吗？到目前为止有任何问题吗？如果没有，我将更详细地介绍这些量是如何选择的，并向你展示训练一个简单扩散模型的例子。

## 噪声调度与模型架构 🎛️

首先，我们讨论噪声调度，即如何选择添加到干净数据中的噪声水平。在实践中，这些噪声水平可以从非常小到非常大，小到 0.01，大到 100。与数据量级（大约为 1）相比，噪声可以比数据量级小得多，也可以大得多。这些噪声水平通常是从离散的噪声调度中采样的，噪声调度基本上是一个不同噪声水平的列表。它们通常绘制在图表上，y 轴通常是对数尺度，这是因为我们拥有的噪声水平范围非常大，通常在对数尺度上查看。我们使用坐标 **λ**，它基本上是噪声水平的对数，可以将其视为信噪比的对数。这个采样过程实际上可以可视化为从这个 **λ**（信噪比）分布中采样。我们也将其视为从预定义分布中采样，这些被称为不同的训练噪声调度。如果你注意到，这个分布往往更侧重于某些噪声水平，特别是那些不太大也不太小的噪声水平。这是有意设计的，通常这些噪声调度是根据你拥有的数据类型选择的。如果你想在非常不同的数据模态上训练扩散模型，可能需要更改噪声调度。

为了让事情更具体，让我们尝试在一个基本玩具模型上训练扩散模型。

我们从这个 2D 螺旋数据集开始。我们的数据仅由螺旋上的点组成。它是一个二维数据，即这些点的 x 和 y 坐标。你可以添加不同水平的噪声，这会将螺旋扭曲成基本均匀的高斯分布。这将是模型的输入之一。对于模型的另一个输入，你需要添加噪声水平，模型需要知道我们添加的噪声水平。为此，我们基本上将噪声水平嵌入到另一个 2D 向量中。我们将这两个向量连接起来作为模型的输入。这里的模型是一个玩具模型，它基本上是一个非常简单的深度神经网络，称为多层感知机，由线性层和非线性激活函数组成。它将预测添加到这些数据点上的噪声。在 Python 代码中，它基本上是这样的，你可以稍后查看，但这基本上只是实现了图中所示的架构。

这将是我们用于训练 2D 螺旋数据集的模型。

这里我们展示了一个非常简单的架构。但是，如果你想超越这个玩具模型，进入更复杂的图像数据集，还有哪些更常用的架构呢？

这里我只展示两种在实践中常用的架构。第一种基于卷积层，称为卷积 U-Net。这可能看起来非常复杂，但我们需要关注这里的轮廓。基本上，输入经过一系列层，这些层减少了输入的物理维度（例如从 64x64 到 8x8），但当然也增加了通道数。然后有一个解码器部分，它进行相反的操作，增加物理维度但减少通道数。这些层通过跳跃连接相连，并受当前噪声水平条件控制。这通常被称为 U-Net，最初是为图像分割开发的，但人们已将其重新用于扩散模型，这是图像模型最常用的架构。

另一种不同的、更新的架构基于 Transformer（如果你听说过，这是大型语言模型中使用的）。但它被应用于图像。你取一张图像，将其分解为不同的图像块，然后将这些图像块输入神经网络以获得不同的图像块嵌入，然后你取一个 Transformer 块来处理这些图像块，并持续通过 Transformer 块，同时受噪声水平条件控制。

这两种架构具有相同的输入和输出，即噪声图像和噪声水平，并尝试预测图像中的噪声方向。它们各有优缺点，我现在不会深入讨论。只是让你知道这些是一些更常用的架构。

## 训练目标与理想去噪器 🎯

问题是，为什么我们对噪声使用平方损失，而不是重建损失？这是一个很好的问题，因为你可以预测多种东西。你可以预测噪声，也可以预测干净图像。实际上这两种方法都有效，你甚至可以预测噪声和干净图像之间的差异，如果你这样做，你会得到人们称之为流匹配的东西。所以，有多种不同的方式来修改损失函数，目前我将专注于人们首先尝试的方法，即预测噪声。但稍后我们会更多地讨论预测不同的量。

让我们回到玩具模型，回想一下我们有一个玩具数据集和一个非常简单的多层感知机。如果我们训练这个玩具模型会发生什么？这是训练代码，基本上实现了我们刚才描述的损失函数。你可以看到，这只是噪声数据和干净数据之间的损失函数。你可以训练几分钟，然后损失函数会下降，我们实际得到了什么？我们尝试使用学习到的去噪器来预测不同噪声水平下的干净数据。这里，这是一个非常小的噪声水平，这是一个中等噪声水平，这是噪声水平较大的情况。我们从二维网格中的不同点开始，要求预测干净数据，我们得到了什么？这个箭头对应于不同点（例如 **x_σ**）的预测 **x0**。你可以立即看到，它在不同噪声水平下预测出非常不同的东西。在大的噪声水平下，预测都趋向于数据的中心，它倾向于预测数据的均值。在较小的噪声水平下，你可以看到螺旋形状开始形成。在非常小的噪声水平下，它预测螺旋本身上的点，尽管不完全是，因为如果你看这些中间的点，它实际上并没有精确预测螺旋上的点。你必须非常接近螺旋，预测才会准确。这是训练模型时的经验行为。

现在的问题是，模型实际学到了什么？我们如何从这个玩具数据集中重建或生成样本？因为我们看到不能直接使用模型来预测数据。例如，在高噪声水平下，你基本上只得到数据的均值。你必须以不同的方式更多地采样。但首先，让我们谈谈模型学到了什么。

我将简要介绍理想去噪器的概念，这是一个可以通过解卷积推导出去噪器。通过使用贝叶斯定理和其他计算，你能够基本上写出这个去噪器的闭式解。就我们目前的目的而言，只需说这个理想去噪器是如果你精确最小化这个损失函数会得到的结果。让我们看看理想去噪器的另一种解释，以直观理解发生了什么。之前，为什么学习到的模型在不同噪声水平下预测不同的东西？

我们将更多地讨论距离函数。首先，我们定义一个距离函数，基本上是到这些数据集的距离。有趣的是，距离函数的梯度基本上投影到数据集上。如果你在数据集（这里是黑色轮廓）外取一个点，并计算距离函数的梯度，它会投影到表面上。这很好，但有一个问题是距离函数不平滑。有许多方法可以平滑距离函数，其中之一是使用软最小值而不是最小值。如果你以这种方式移动距离函数，结果发现这与理想去噪器之间有非常好的联系。事实上，这个平滑距离函数的梯度正是理想去噪器预测的东西。所以你可以认为理想去噪器（即扩散模型试图学习的东西）基本上是在尝试预测平滑
